# ⚙️ Project Mecha

> **"유저에게 생각할 시간을 주어서는 안된다. 하지만, 매 순간 유저는 최선의 생각을 해야한다."**

**Project Mecha**는 고철 부품을 주워 나만의 메카를 조립하고, 적과 맞서 싸우는 2D 픽셀 아트 턴제 메카 빌더 로그라이크 게임입니다. Godot 4.x 엔진의 유연한 기능을 활용하여 전략적인 재미와 깊이 있는 커스터마이징을 목표로 합니다.

---

## 🎮 게임 개요

- **장르:** 2D 메카 빌더 로그라이크 (Pixel Art)
- **엔진:** Godot 4.6
- **런 길이:** 약 15분
- **핵심 루프:** 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투

## ✨ 핵심 특징

- **부품 조립 시스템:** 코어를 중심으로 팔(2), 등(1), 다리(1) 슬롯에 다양한 부품을 장착하여 자신만의 메카를 구성합니다.
- **턴제 전략 전투:** 내 턴에는 스킬을 선택해 공격하고, 적의 턴에는 예고된 행동에 맞춰 대응하는 전략적 판단이 중요합니다.
- **로그라이크 요소:** 10층으로 구성된 던전을 탐험하며 무작위 부품, 상자, 조우 이벤트를 통해 매번 새로운 도전을 경험합니다.
- **2D 픽셀 아트:** 클래식하고 세밀한 픽셀 아트를 통해 메카의 매력을 극대화합니다.

## 🏗️ 아키텍처 설계 원칙

본 프로젝트는 확장성과 유지보수를 위해 다음 원칙을 따릅니다:

1.  **조합 중심 (Composition):** 상속보다는 독립적인 기능을 가진 노드(컴포넌트)를 조합하여 엔티티를 구성합니다.
2.  **데이터 기반 (Data-Driven):** 메카 부품, 스탯 등은 `Resource (.tres)` 파일을 통해 관리하여 밸런스 조정을 용이하게 합니다.
3.  **신호 기반 통신 (Signals):** 시스템 간 느슨한 결합을 위해 Signal과 전역 `EventBus`를 적극 활용합니다.

## 📁 프로젝트 구조

- `Asset/`: 스프라이트, 텍스처, 오디오 등 리소스 에셋
- `Docs/`: GDD 및 기술 설계 문서
- `Resources/`: `.tres` 데이터 및 리소스 스크립트
- `Scenes/`: 게임 핵심 씬 (Game, UI, Levels)
- `Scripts/`: 전역 유틸리티 및 Autoload(싱글톤)
