# ⚙️ Project Mecha — Game Design Document

> **핵심 철학**
> "유저에게 생각할 시간을 주어서는 안된다. 하지만, 매 순간 유저는 최선의 생각을 해야 한다."
> 불필요한 마찰은 최소화하되, 매 선택은 의미 있어야 한다.

> **핵심 재미 — "파츠가 주인공"**
> 파츠를 적극적으로 **사용하고, 버리고, 교체하고, 활용한다.** 파츠는 코어를 지키는 **소모성 장갑이자 화력**이다.
> 교체는 권장된다(조립은 무비용, 전투 중 교체는 손패를 뒤로 돌릴 뿐 손실 없음). 파괴 비용은 *의도적 소비(투척·희생)* 에만 붙는다.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 메카 빌더 턴제 로그라이크 (2D 픽셀 아트) |
| **엔진** | Godot 4.x (GDScript) |
| **런 길이** | 약 15분 |
| **핵심 루프** | 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투 |

### 1.1 한 줄 소개

고철 부품을 주워 메카를 조립하고, 그 메카로 적과 싸우는 턴제 로그라이크.

### 1.2 두 주인공 구조

- **스토리의 주인공 = 코어(파일럿).** 게임 내내 바뀌지 않는 단일 상수. 스캐빈저 파일럿이다.
- **게임플레이의 주인공 = 파츠.** 런마다, 전투마다 바뀌는 변수. 주워온 고철이다.
- *파일럿은 무대, 파츠는 배우.* 런 안의 파워는 파츠가 만들고, 코어는 그 파츠를 담고 부리는 그릇이다. 파츠는 버려져도 파일럿의 코어 권한은 사라지지 않는다.

## 2. 런 구조

### 2.1 전체 흐름

```
거점(출격 준비) → 던전 특성 확인 → 출격 빌드 설계 → 던전 진입
  └─ 던전 내 반복: [전투 클리어 → 보상 → prep 루프(맵·조립·공급) → 다음 방 선택]
런 종료: 보스 처치(성공) / 코어 HP 0(실패) → 결과 화면 → 거점 복귀
```

- 런 시작 전 거점에서 출격을 준비하고, 런 종료 시 거점으로 복귀한다.
- **거점의 세부 구조와 런 종료 정산 규칙은 기획 예정.** 현재는 게임 전체 루프를 잇는 연결점으로만 둔다.

### 2.2 prep 루프 — 전투 사이의 준비 화면

전투와 전투 사이, 다음 방으로 가기 전 플레이어는 세 개의 탭을 오간다.

| 탭 | 역할 |
|----|------|
| **[맵]** | 다음 경로(방)를 고른다 = 보급 전략 (§2.3) |
| **[조립]** | 파츠를 갈아끼워 다음 전투 빌드를 짠다. 교체 자유·무비용 (§5.7) |
| **[공급]** | 작업대 — 파츠를 수리·분해·제작하는 가공소 (§2.6). 상시 접근 |

> 작업대는 던전의 "방"이 아니라 **항상 들고 다니는 [공급] 탭**이다. 경로로 도달하는 목적지가 아니라, 들어온 원자재(파츠·고철)를 가공하는 상시 도구다.

### 2.3 던전 구성 (맵)

- 총 **10층** 구조. **Slay the Spire식 연결 그래프**로, 최대 4칸 폭에서 갈래가 합쳐졌다 갈라진다.
- **엣지 제약:** 연결된 노드로만 이동한다. 하나를 고르면 다른 길은 포기 — "이 길의 X vs 저 길의 Y"가 진짜 결정이 된다.
- **전체 맵을 처음부터 공개**해 경로를 미리 계획한다. 각 노드는 종류 아이콘 + 예상 보상 등급을 보여준다.
- 10층은 보스로 고정되며, 클리어 시 런이 종료된다.

**경로 = 보급 전략.** 파츠는 부위별로 소비·고갈되므로, 경로 선택은 "어디가 안전한가"가 아니라 **"내 빌드에 무엇을 보급하나"**다.

| 길 성격 | 구성 | 보급 체감 | 교리 경향 |
|---------|------|----------|----------|
| **강행군** | 전투·엘리트 빽빽 | 파츠·고철이 많이 들어오나 소비도 큼(물량전) | 해체공 |
| **비축** | 상자·조우 위주 | 전투가 적어 소비 적음. 확정/도박성 고급 파츠 위주(골라먹기) | 정비공 |

- **부위 특화:** 일부 길은 특정 부위 보급에 치우친다(확정 공급이 아니라 드롭 확률↑, 그런 길 자체가 드묾). **다리는 더 드물게** 보급된다(§5.5 다리 경제).
- **교리 노드:** 맵에 가끔 선택지 하나로 `해체공 전투` / `정비공 전투` 특수 노드가 섞인다. 클리어 시 그 교리의 **런 한정 boon**을 얻는다 (§3.7).

### 2.4 방 종류

맵 노드의 방 종류는 다섯 가지다. (작업대는 방이 아니라 [공급] 탭 — §2.2.)

| 방 종류 | 보상 | 비고 |
|--------|------|------|
| **일반 전투** | 격파 적 1기당 0~2개 (COMMON) + 크레딧 15~25 | |
| **엘리트 전투** | 격파 적 1기당 0~2개 (RARE) + 크레딧 45~65 | 강한 적, 높은 보상 |
| **상자** | *(기획 예정)* | 전투 없음 |
| **조우 이벤트** | *(기획 예정)* | 텍스트 선택지 |
| **보스** | 격파 적 1기당 0~2개 (EPIC) + 크레딧 85~105 | 10층, 클리어 시 런 종료 |

> 상자·조우 이벤트의 보상·결과 상세는 기획 예정이다.

전투 드롭 파츠는 선택 없이 전부 인벤토리로 이동한다. 격파한 적 1기마다 드롭 개수를 굴린다: **0개 40% / 1개 50% / 2개 10%.**

### 2.5 재화

- **크레딧** — 런 기본 재화. 전투 클리어 시 고정 지급(방 종류별 상이). 작업대 수리·코어 연구에 사용.
- **고철** — 보조 재화. **커먼(⚙) / 레어(✦)** 두 등급. 분해·고철더미·드롭으로 수급하며, 작업대 제작 재료다.
  - COMMON 파츠 분해 → 커먼 고철, RARE/EPIC 분해 → 레어 고철 중심.

### 2.6 작업대 ([공급] 탭) — 파츠 공급 허브

"파츠가 주인공"에서 투척·희생으로 파츠 소비가 빠르므로, 공급이 소비를 떠받치지 못하면 게임이 무너진다. 작업대는 **파츠 공급 라인**으로서 이를 정면으로 해결한다. 세 기능 + 고철더미로 구성되며, 크레딧·고철 소모로 자연스럽게 제한된다.

| 기능 | 입력 | 산출 | 교리 |
|------|------|------|------|
| **수리** | 크레딧 | 파츠 내구도 복구 (1개 / 전체) | 정비공 |
| **분해** | 파츠(파괴) | 등급별 고철 | 해체공 |
| **제작** | 고철 | 부위·affix 개수를 지정한 새 파츠 | 해체공 |
| **고철더미** | — | 층당 5회 한정 랜덤 파츠 드로우 | 공급 |

- 제작 시 affix 최대 개수는 코어 노드 진행도에 묶인다.
- 작업대는 **계획적·영구 공급**, 전투 중 휘발 동사(§3.2)는 **급박한 즉흥 공급**을 맡는다(같은 환상을 시간축으로 분리).

### 2.7 보급 readout

경로를 짜는 내내 보이도록, 맵 구석에 부위별 보급 현황을 상시·컴팩트하게 표시한다.

```
[보급]  인벤토리 12/16
  팔 ×3    등 ×1    다리 ×0 (!)    추가팔 ×1     ← 멀쩡한 것만, 0인 부위 경고색
  고철 ⚙3 ✦1   ·   파손 2
```

총합이 아니라 **부위별 갯수**라야 "다리가 동났다" 같은 진짜 보급 위기가 보인다.

### 2.8 결과 화면

런 종료 시 성공·실패 공통으로 표시한다: 클리어/실패, 도달 층수, 런 소요 시간, 코어 잔여 HP%, 총 전투 수, 크레딧 획득/사용/순이익, 고철 획득, 최종·회수·손실 파츠 수, 희귀도별 파츠 수, 파손/파괴 파츠 수.

## 3. 코어 시스템

### 3.1 코어 본체

- **단일 주인공.** 절대 교체·해금이 없다. 게임 내내 1개 고정.
- **런 중 고정.** HP·하중·행동력(AP)·슬롯 등 본체 스탯은 런 동안 변하지 않는다. **런 내 파워 변동은 전적으로 파츠가 담당한다.**
- **성장은 메타로.** 코어를 바꾸는 게 아니라, *단일 코어의 노드망·예산·슬롯이 게임 전체 진행도(거점 재화 투자)에 따라 깊어진다.*

### 3.2 코어 어빌리티 — 동사 모델

코어 어빌리티는 파일럿의 **파츠 조작 동사 그 자체**다. 파츠를 조작하는 권한(교체·투척·덧대기 등)을 *잘 부서지는 파츠 층*이 아니라 *안 부서지는 코어 층*에 두었다 — 파츠가 버려져도 권한은 남는다.

출격 전 **3개 슬롯 로드아웃**으로 동사를 세팅하며(런 중 고정), 이 조합이 곧 빌드이자 교리 방향이 된다.

- **슬롯 성장:** 시작 1슬롯 → 메타로 2 → 3 (→ 후반 특별 해금 4번째).
- **공통 비용 = AP + 재료** 단일 축. 재료는 파손 파츠·내구도 등 동사 고유 연료다. 인컴뱃 행동에 공짜는 없다(가드레일).
- 동사는 교리에 종속되지 않는 **자유 부품**이다. 어떤 조합이든 가능하며, 결과적으로 빌드가 특정 교리 쪽으로 기운다.

#### 상시 기본기 (슬롯과 무관, 항상 사용 가능)

| 기본기 | 내용 |
|--------|------|
| **기본공격** | 출격 전 1택. 파츠가 전부 파손돼도 쓸 수 있는 안전판. 각 기본공격은 교리를 가진다(키스톤 타이브레이커). |
| **긴급 장착** | 파손돼 떨어져 나간 빈 슬롯에 큐의 파츠를 장착해 손실을 복구한다. |

> **기본공격 3종과 교리:** 입자 광선(고비용·최고 피해·전 AP 소진) = **태운다** / 간이 방벽(공격하며 쉴드 확보) = **살린다** / 속사 사격(저비용·반복) = **중립.** 기본공격은 파츠 내구도를 소모하지 않으며 상시 사용한다.
>
> **교체 vs 긴급 장착:** 교체는 *멀쩡한* 파츠를 능동적으로 떼고 큐 파츠를 끼우는 전략 행위. 긴급 장착은 *파손으로 빈* 슬롯을 채우는 복구 행위(상시 기본기). 긴급 장착이 상시이므로 교체를 슬롯에 안 넣어도 손실 복구는 항상 가능하다.

#### 동사 로스터 (슬롯에 넣는 7종)

| 동사 | 효과 | 교리 친화 | 지속성 |
|------|------|----------|--------|
| **교체** | 멀쩡한 장착 파츠를 큐로 보내고 큐 파츠를 장착. AP만 소모 | 중립/회전 | 영구 |
| **덧대기** | 재료 파츠를 소모해 대상 파츠의 **내구도를 복구** | 살린다 | 영구 |
| **투척** | 파츠를 적에게 던져 피해 + 소비 | 태운다 | 영구(소비) |
| **과부하** | 한 파츠의 스킬 출력을 폭주시킴. 사용 후 파괴 | 태운다 | 영구(가속 소모) |
| **재가동** | 파괴된 파츠를 내구도 50%로 되살림 | 집착→소비 | 휘발 |
| **복제** | 파츠 1개의 1회용 사본을 생성·장착 | 버스트(태운다) | 휘발 |
| **임시방편** | 능력 없이 "때리기" 공격만 가진 임시 파츠를 생성·장착 | 즉흥/중립 | 휘발 |

- **임시방편 레퍼런스:** 제이슨 본이 수건·잡지를 말아 무기로 쓰듯, "무에서 연료·몸뚱이를 만든다."

#### 휘발 동사 (재가동·복제·임시방편)

- **전투 한정.** 전투가 끝나면 산출물이 소멸한다. 작업대(영구 공급)와 시간축으로 대비된다 — 계획적 정비 vs 급박한 임기응변.
- **재가동 = 화장:** 무리하게 50%로 부팅한 파츠는 내구도가 소진되면 완전 소멸한다. 살아남아도 전투가 끝나면 사라지므로 "그 전투 안에 다 짜내야" 한다.
- **산출물 배치:** 복제 사본·임시방편 파츠는 생성과 동시에 빈 슬롯에 즉시 장착된다. 빈 슬롯이 없으면 큐 최상위에 들어가 다음 교체/긴급 장착 1순위가 된다.

#### 동사 업그레이드 (메타)

각 동사는 기본형으로 출발해 메타 노드로 확장된다. 업그레이드는 "더 세짐"이 아니라 **"쓰는 법이 갈라짐"**이며, 일부는 두 갈래 중 하나만 고르는 트레이드오프다.

| 동사 | 기본형 | 업그레이드 방향 (택1) |
|------|--------|----------------------|
| **교체** | 멀쩡 장착 파츠 큐로 · AP만 | ⓐ 교체 AP 0(무료 회전) / ⓑ 새로 낀 파츠 첫 사용 강화 |
| **덧대기** | 대상 파츠 내구도 복구 | ⓐ 재료 파츠 affix 1개 이전 / ⓑ 최대 내구도 증가 / ⓒ 능력치(스킬 수치) 증가 |
| **투척** | 파손 파츠만 던짐 | ⓐ 장착 중인 파츠도 / ⓑ 인벤토리의 멀쩡한 파츠도 (둘 다는 불가) |
| **과부하** | 폭주 후 파괴 | ⓐ 파괴 대신 파손 상태로(재활용) / ⓑ 피해 더↑ 대신 코어 반동 |
| **재가동** | 50% 부활·휘발 | ⓐ 부활 내구도↑(75%) / ⓑ 대상 +1(연쇄) |
| **복제** | 1회용 사본·휘발 | ⓐ 강화 사본 / ⓑ 다중 복제(2) / ⓒ 영속 사본 |
| **임시방편** | 때리기 임시 파츠·휘발 | ⓐ 즉석 affix / ⓑ 공격형(위력↑ + 처치 시 AP 환급) / ⓒ 긴급 일괄(빈 슬롯 수만큼) |

**자연 발생 콤보** (자유 조합의 결과로 빌드 색깔이 드러난다):
- 복제 → 과부하(사본) → 투척: 원본은 안전, 사본으로 폭딜.
- 임시방편 → 투척/과부하: 쓰레기를 찍어내 태우는 *연료 엔진.*
- 과부하(파괴) → 재가동(50%) → 마지막 한 방: 끝까지 우려먹기.

### 3.3 교리

- **교리는 클래스(상자)가 아니라 축/지향점**이다. 플레이어가 *겨냥하는 방향*. 현재 **태운다 / 살린다** 두 축 + 중립이며, 확장형이다.
- 교리를 가르는 단 하나의 축은 **"소비한 파츠를 어떻게 보느냐"**다.

| 교리 | 정체성 | 태도 | 전투 체감 |
|------|--------|------|----------|
| **해체공 (태운다)** | "고철은 적 얼굴에 던진다" | 파손을 이용한다(태워서 자원화) | 파츠가 연료. 휘발적·폭발적, 손이 바쁘다. 투척·과부하·소모 |
| **정비공 (살린다)** | "이 팔, 보스까지 데려간다" | 파손을 막는다(오래 살린다) | 같은 파츠로 진득하게, 내구도 관리가 곧 실력. 덧대기 |

> 새 코어는 **해체공(적극 소비) 쪽에 기본 무게**를 두되, 정비공은 "오래 살려 진득하게"라는 대안 플레이로 남는다.

- **태그가 backbone이다.** 빌드 재료(동사·affix·교리 패시브·boon)마다 0~1개의 교리 태그가 붙고, 게임은 활성 태그 분포(=정렬)를 읽는다.
- **두 층으로 작동:** 출격 전(노드)은 점수를 합산해 키스톤을 확정하고(선언형), 런 중(파츠·affix·boon 태그)은 키스톤 방향에 맞춰 모으면 약한 시너지가 쌓인다(누적형). 키스톤이 나침반 역할.
- **"교리에 가두지 않는다."** 동사와 노드는 자유롭게 조합하고, 교리는 *결과로* 결정된다.

### 3.4 키스톤 결정

코어 설계 화면은 **노드 빌드**다. 노드마다 교리 점수(태운다 +N / 살린다 +N / 중립 0)를 가진다.

| 순서 | 규칙 |
|------|------|
| 1 | 노드 교리 점수를 **합산**한다 *(기본공격 제외)* |
| 2 | **동점이면** 기본공격 교리 점수를 발동·가산해 타이를 해소한다 |
| 3 | 선두 교리가 **문턱(최소 점수) 이상**이면 그 교리의 **강한 키스톤** 발동 / 미달이면 키스톤 없음(제너럴리스트) |

- **기본공격 = 순수 타이브레이커.** 점수는 평소 잠자고 동점일 때만 발동한다.
- **키스톤은 빌드 일관성으로 "버는" 것.** 확실히 commit하면 강한 키스톤, 어정쩡하면 유연하지만 밋밋하다 — 이것이 곧 빌드 결정의 무게다.
- 노드 조정 시 교리 점수 바와 "현재 키스톤: ___"을 실시간으로 보여준다.

### 3.5 메타 성장 (코어 업그레이드)

- **메타 성장은 스탯을 부풀리지 않는다.** 대신 **규칙을 비트는 키스톤 + 트레이드오프**를 제공한다. (예: "재가동한 파츠가 내구 소진돼도 소멸하지 않는다 — 대신 ○○를 포기.")
- **코어 업그레이드 노드 = 규칙 + 스탯(대가 필수).** 순수 +스탯 노드는 없다. 스탯을 주면 반드시 다른 것을 내준다.

| 형식 | 내용 | 예 |
|------|------|----|
| **규칙형** | 규칙을 비틂(스탯 0) | "재가동 파츠 소멸 무효 / 대신 재가동 전투당 1회" |
| **스탯+대가** | 소폭 스탯 ↔ 다른 스탯/규칙 내줌 | "HP +20 / 하중 −10", "AP +1 / 쉴드 최대 −25" |
| **혼합** | 규칙 + 소폭 스탯 + 대가 한 묶음 | "투척 사거리 무제한 + 피해 −20% / 멀쩡 파츠 투척 금지" |

- 모든 코어 업그레이드 노드도 교리 점수를 가져 키스톤 합산에 기여한다.
- 해금은 거점 재화 투자·전체 진행도로 영구적이며, 노드망·슬롯·키스톤이 이를 통해 깊어진다.

### 3.6 코어 설계 화면 (노드 빌드)

- **평면 노드망.** 티어 게이팅이나 노드 레벨이 없다. 노드는 해금/장착의 단일 강도를 가진다.
- 노드 종류 = 동사 / 기본공격(택) / 코어 업그레이드 / 교리 패시브. 각 노드가 교리 점수를 가진다.
- 진행 곡선은 **재화 예산 + 해금 순서**로 만든다(메타로 망이 넓어진다).
- 출격 전 여기서 확정한다: **3 동사 슬롯(시작 1) + 기본공격 1택 + 키스톤(노드 교리 점수 합산 결과).**

### 3.7 런 boon (교리 노드)

- 던전 맵의 교리 노드(§2.3)를 클리어하면 그 교리의 **런 한정 boon**을 얻는다 — 이번 런만 유효, 런이 끝나면 리셋(하데스 신 boon 결).
- boon은 새로운 인터랙트가 아니라 **미리 세팅해 온 코어 어빌리티를 강화**하는 방향이며, 런 한정 교리 태그로 키스톤/정렬 방향을 증폭한다. 같은 교리를 반복해 모으면 약한 시너지가 쌓인다.
- **영구 교리 성장(거점 재화 투자)과는 별개 레이어**다. 시간축이 다르다(메타 vs 런) — 하데스의 "거울 업글(영구) + 신 boon(런)" 구조처럼 둘은 보완 관계다.

## 4. 파츠 시스템

### 4.1 슬롯 구조

메카는 코어를 중심으로 기본 4개의 부품 슬롯을 가진다. `진화 군주` affix가 붙은 정상 팔/등 파츠를 장착하면 **추가 팔 슬롯** 1개가 열린다.

| 부위 | 슬롯 수 | 스킬 성향 | 특징 |
|------|--------|----------|------|
| **팔** | 2 | 공격, 방어 | 능동적 행동 |
| **등** | 1 | 방어, 회복, 버프 | 지원형 |
| **다리** | 1 | 방어, 회피·반격 | **피격 시 발동** 조건 |
| **추가 팔** | 조건부 1 | 공격, 방어 | `진화 군주` 필요, 팔 파츠 장착 가능 |

### 4.2 파츠 = 코어의 소모성 장갑 + 화력

파츠는 코어를 지키는 소모품이다. 이 재프레이밍이 "파츠가 주인공"을 떠받친다.

- **빈 슬롯이면 그 부위로 들어오는 공격이 코어 HP로 직격한다.** 파츠는 갈아끼우면 되지만 코어 HP는 못 바꾼다 → **부위를 빼서 위협을 회피할 수 없다.**
- 팔은 화력·AP를 공급한다. 팔을 빼면 기본공격만 남아 엘리트를 못 깬다.
- 따라서 위협은 **"소모전"**이지 "회피 신호"가 아니다. 막 쓰고 버리며 보급으로 버티는 플레이가 정답이 된다.

### 4.3 덱 구조

파츠는 일종의 **덱**이다 — 부위 슬롯 4개(장착)와 인벤토리 큐(예비 풀)로 구성된다. 동사(교체·긴급 장착·투척 등)가 이 슬롯과 큐 사이를 움직이며 전투를 굴린다.

- **예비 = 하이브리드:** 인벤토리가 부위별 예비를 자동 공급하고, 슬롯별 "핀"으로 우선순위를 줄 수 있다(핀을 안 하면 자동).
- 덱빌딩의 손맛은 *전투 밖*(획득·분해·제작·affix)에서 나온다.

### 4.4 파츠 풀 · 등급 · Affix

고유 파츠는 부위별로 다양하게 존재하며(팔·등·다리), 스킬은 각 파츠에 고정된다.

등급은 파츠에 고정된 속성이 아니라 **드롭 시 굴린 affix 개수**로 결정된다. 루팅할 때마다 수치·affix 개수·종류가 달라진다.

| 등급 | affix 개수 | 초기 손상도 | 주요 획득처 |
|------|-----------|-----------|-----------|
| **COMMON** | 0~1 | 3 | 일반 전투 |
| **RARE** | 2~3 | 5 | 엘리트 전투, 상자 |
| **EPIC** | 4~5 | 7 | 엘리트/보스, 상자 |

**Affix = 교리 태그 시너지.** affix는 단순 스탯 보너스가 아니라 **교리 태그(태운다/살린다 등)가 붙은 시너지 조각**이다. 키스톤이 가리키는 방향의 태그를 모으면 빌드가 강해진다. 파츠는 직관적인 스킬 + affix를 가진 "덱 카드"로 작동한다.

### 4.5 부위별 동사 인격 · 다리 경제

부위마다 파츠를 다루는 성격이 다르다 — 이것이 "파츠마다 다른 기믹"의 정체다.

| 부위 | 정체성 | 핵심 동사 | 논리 |
|------|--------|----------|------|
| **팔** | 만능 일꾼 · 소모품 | 교체·투척·방어 다 함 | 흔함 → 막 굴린다 |
| **등** | 쓰고 버리는 지원 | 투척, 또는 **자신을** 일회용 방패/버스트로 소모 | 보조 장비 → 한 방 쓰고 끝 |
| **다리** | 지켜야 할 기둥(희귀) | 잘 안 버림. **다른 파츠(팔)를 덧대 수명 연장** / 미끼로 피격 유도 | 귀해서 아껴 쓴다 |

- **겹침 방지:** 등 = *자신을 희생*(소모되는 쪽) / 다리 = *남을 희생해 지킴*(지켜지는 쪽). **"팔을 태워 다리를 지킨다"**가 이 게임만의 내부 환율이다.
- **다리 경제:** 다리는 드롭률만 낮춰 희귀하게 만든다(종류는 유지). 다리도 교체 가능하나 *아껴 쓰는* 내러티브다 — "못 바꾸는" 게 아니므로 방어 죽음의 나선은 없다.

### 4.6 스킬 종류

모든 플레이어 파츠 스킬은 **액티브**(플레이어 턴에 직접 선택·사용)다. 버프·디버프는 독립 스킬이 아니라 각 스킬의 **부가 효과**로 처리한다.

| 스킬 타입 | 설명 |
|----------|------|
| **공격** | 적 코어 또는 부품에 피해 |
| **방어** | 피해 감소, 쉴드 부여 |
| **회복** | 코어 HP 회복 |
| **유틸** | 위 외 능동 효과 |

> 다리 슬롯 스킬은 **피격 시 발동** 조건을 공통으로 가진다(회피, 반격 등).

### 4.7 파손과 소비

| 조건 | 결과 |
|------|------|
| 적 공격으로 내구도 0 | 해당 파츠 **파손**(장착 불가, 큐 점유, 스킬·affix 비활성) |
| 의도적 소비(투척·희생) | 해당 파츠 **파괴**(소멸) |
| 조립 중 교체 | 파손·손실 없음 (무비용) |
| 전투 중 교체 | 손패 뒤로 돌아갈 뿐 **무손실** (템포 비용만) |

**처분:** 거점 수리(고철 → 원본 복구) / 작업대 분해(슬롯 해방, 고철 회수) / 전투 중 투척(화력 전환) / 전투 중 덧대기(재료 파츠 소모 → 내구도 복구).

교체가 파손을 만들지 않으므로, **인벤토리는 불안 관리가 아니라 갈아 쓸 예비 자원 풀**이다.

### 4.8 인벤토리 용량

| 단계 | 슬롯 수 | 그리드 | 해금 조건 |
|------|--------|-------|---------|
| 기본 | 16 | 4×4 | 런 시작 시 |
| 확장 1 | 24 | 4×6 | 코어 업그레이드 |
| 확장 2 | 28 | 4×7 | 코어 업그레이드 |

- 파손된 파츠도 슬롯 1개를 점유한다.
- 인벤토리가 가득 찬 상태에서 드롭이 발생하면 해당 파츠는 유실된다.

### 4.9 조립 — 시각적 조립 경험

파츠가 스탯 + 슬롯으로만 표현되면 카드 게임과 다를 게 없다. 차별점은 **파츠를 붙일 때마다 메카 실루엣이 바뀌는 시각적 조립 경험**에 있다. 팔을 끼우면 팔이 생기고, 등에 대형 캐논을 달면 등이 무거워 보인다. 기본 4개를 채우면 내 메카가 서고, 그 실루엣이 던전으로 걸어 들어간다.

**조립 화면([조립] 탭)** 은 다음 위협을 보고 빌드를 짜는 곳이다. 결정의 무게는 "비용"이 아니라 **"이 위협엔 이 빌드가 답인가"**에 실린다.

- **위협 readout** — 다음 방 종류·적 구성·위협 태그를 *소모형 언어*로("팔 소모전 — 예비 권장", 회피형 표현 금지).
- **파츠 트레이** → **무대(메카 실루엣)** 로 드래그해 장착. 위협받는 부위는 경고색. 클릭으로 탈착.
- **턴 프리뷰** — 이 빌드의 전투 턴(스킬 아이콘 + AP)을 실시간으로 보여준다(전투 스킬 덱과 같은 모양).
- 드래그앤드롭으로 소켓에 붙인다. 호환 소켓은 초록 피드백, 비호환은 거부.

## 5. 전투 시스템

### 5.1 전투 구조

- **구도:** 내 메카 1 vs 적 메카 1~3
- **방식:** 교대식 턴제 (내 턴 → 적 턴 → 반복)
- **승리 조건:** 적 코어 HP 0
- **패배 조건:** 내 코어 HP 0 → 런 종료

### 5.2 내 턴

- 턴당 행동력 = 코어 기본값 + 파츠 AP 기여 + 코어 노드 보정.
- 스킬마다 행동력 비용을 지불한다.
- 스킬 선택 후 **타겟 지정**(적이 여럿이면 직접 선택, 비우면 적 코어 자동 공격).
- 기본공격은 파츠가 파손돼도 쓸 수 있는 안전판이다.
- 코어 어빌리티 동사(교체·투척·과부하 등)를 전투 중 사용해 파츠를 능동적으로 운용한다.

### 5.3 적 턴

- 적은 랜덤으로 스킬을 선택한다.
- 단, **다음 턴 행동을 미리 공개**해(예고) 플레이어가 대응을 판단할 수 있다.

### 5.4 비주얼 규칙 — Into the Breach × Darkest Dungeon (전 화면 공통)

> **"고철 더미의 어둠 속, 정밀한 조종석 계기판."** 무대는 무겁고 어둡게, 계기판은 밝고 정확하게.

화면을 픽셀 단위로 섞지 않고 **역할(레이어)로 가른다.**

| 레이어 | 담당 | 느낌 | 출처 |
|--------|------|------|------|
| **무대** (메카·적·파츠·배경) | 감정·무게 | 어둡고 무겁고 그을림, 강한 실루엣, 극적 명암 | Darkest Dungeon |
| **계기판** (정보·수치·예고·버튼) | 판단·명료 | 밝고 평면적, 정확, 군더더기 없음, 다 예고됨 | Into the Breach |

- 그을음은 계기판의 가독성을 침범하지 않고, 계기판은 무대의 압박감을 누그러뜨리지 않는다. **분리가 곧 규칙.**
- **하드 베젤(분리선)을 두지 않는다.** 무드 + 위치로만 분리한다(Slay the Spire식 — 배경 가득, 부드럽게 덱으로 이어짐).
- 철학과 직결: "생각할 시간을 주지 않는다" = 무대의 압박(DD) / "매 순간 최선의 생각" = 계기판의 완벽한 가독성(ItB).

### 5.5 전투 화면 레이아웃

3구역으로 구성한다.

- **상단 — 계기판:** 층 · HP · 쉴드 · 크레딧.
- **중앙 — 전투 장면(StS식):** 내 메카를 크게 배경 안에 세우고, 우측에 적, 적 위에 예고 아이콘(계기판이 무대에 닿는 유일 지점).
- **하단 — 컨트롤 덱(3분할):**

| 좌 — 손상도 요약 | 중-좌 — 기본 + 파츠 스킬 | 중-우 — 코어 어빌리티 | 우 — 예비 파츠(스왑) |
|---|---|---|---|
| 부위별 내구도 상태 | 기본공격 + 부위별 파츠 스킬 | **특수 프레임**(함입 콘솔·앰버 글로우·교리 문장) | 예비 큐(맥락 집중, 팔 중심) |

- **읽기 순서가 서사:** 좌(내 상태) → 중(내 행동) → 우(내 예비 자원).
- **3층 중요도(코어 어빌리티 > 파츠 스킬 > 기본)** 는 크기보다 틀·반응으로 표현한다. 코어 어빌리티 셀은 비용을 `AP + 트레이드`("AP2·기존팔 상실")로 표기해 가드레일을 눈에 보이게 한다.
- 손상 분업: 무대는 손상 '사건'을 연출(피격 번쩍·흔들림·스파크 = DD 무게), 정확한 손상도 '상태'는 좌측 요약(ItB 가독).

### 5.6 파손 반응 루프 (비-모달, 시그니처)

파츠가 부서졌을 때 "그 다음은?"을 보여주는 이 게임의 시그니처 흐름이다. **모달이 아니다 — 멈춤 없이 무시하고 계속 싸울 수 있다.**

1. **위험 신호(파손 전):** 적 저격 예고 시 좌측 손상도 요약 + 무대의 그 부위가 경고 앰버.
2. **파손(DD 무게):** 순간 암전 + 메카 흔들림 + 무거운 타격음. 그 부위 균열·스파크, 해당 파츠 스킬 회색 사망, 좌측 요약 [파괴] 빨강.
3. **"그 다음은?"(ItB 명료):** 코어 어빌리티 패널 맥동 + 「○○ 파손 — 대응」. 패널은 *주의만 재배치*한다.
4. **대응(부위별 세트):** 팔 파손 → [교체][투척][덧대기] + 스왑 큐 / 등 소모 → [투척][일회용 방패] / 다리 위협 → [팔 덧대 보존][미끼].
5. **마무리:** 갱신 후 루프 지속. 미대응 시 한 손으로 계속 싸우되 손상도 요약에 파괴 상태가 남아 "아직 대응 가능"을 상기시킨다.

교체/대응 시 호버하면 **미리보기 한 줄**을 준다: `공격 12→19 │ 비용: AP2·오른팔 영구 상실`.

### 5.7 조립 ↔ 전투 연결 (한 몸 보장)

조립과 전투가 맥락·재미·UX로 이어지도록 못박는 4개 조항.

1. **메카 에셋 동일** — 조립 무대의 메카 = 전투 무대의 메카. "내가 만든 그 기체가 싸운다"가 깨지지 않게.
2. **언어 동일** — 위협 readout 언어 = 전투 예고 언어. 약속(조립)과 지불(전투)이 정확히 맞물림.
3. **턴 프리뷰 = 전투 스킬 덱의 미니** — 조립에서 배치한 스킬 배열이 전투 하단 덱에 그대로. "내가 짠 게 곧 내가 쥐는 패."
4. **"조립은 태세, 전투는 즉흥"** — 조립엔 위협의 *종류(벡터)만* 준다("팔이 갈린다"). *언제·어느 팔·몇 뎀*은 전투에서 닥친다. 위협 정보를 너무 완전하게 + 카운터를 너무 확정적으로 주면 전투가 결정론이 되므로, **완벽 카운터를 금지**한다.

### 5.8 전투 보상

- 크레딧과 고철 획득.
- 격파한 적 1기마다 파츠 드롭 개수를 판정한다(0개 40% / 1개 50% / 2개 10%).
- 드롭된 파츠는 선택 없이 모두 런 인벤토리에 추가된다.

## 6. 적 시스템

### 6.1 지역 기반 분류

적들은 Protocol Interrupt 당시 **처해 있던 환경(지역)** 에 따라 분류된다. 지역마다 공통 별칭과 특색이 있으며, 각 지역에는 전용 보스가 존재한다. 적의 슬롯은 스킬 풀 다양성을 결정하며, 플레이어가 타겟·파괴할 수 없다.

| 지역 | 별칭 | 세계관 배경 | 병종 방향 | 보스 |
|------|------|-----------|----------|------|
| **매립지 (Junkyard)** | 스크래퍼 | Protocol 당시 이미 폐기·방치된 기계들 | 공격 / 파츠 압박 / 지원 / 증원 (역할 튜토리얼) | 수집가 |
| **구 도시 (Old City)** | 위병 | Protocol 당시 현역 도시 수비·서비스 기계들 | 차단·경보·제압 편대 | 미정 |
| **연구시설 (Lab)** | 레플리카 | Protocol 당시 연구 중이던 프로토타입들 | 복제·출력 폭주·오류 주입 | 미정 |

- 일반 병종도 플레이어 파츠를 공격할 수 있다. 지역별 Breaker 역할이 파츠 압박을 담당한다.

### 6.2 수집가 보스 (매립지)

- 시작 시 코어 1 + 팔 4 = 5체. 각 팔이 독립적으로 행동한다.
- 살아 있는 팔이 코어를 보호한다(방어 팔은 추가 보호). 팔이 전멸하면 코어가 노출되었다가 4개를 재수집한다.
- **팔 탈취:** 플레이어의 팔 슬롯을 강탈해 그 파츠의 스킬로 플레이어를 공격한다.
- **코어 HP 0이면 팔이 남아 있어도 즉시 승리.**
- **저격(EMP):** 특정 슬롯(예: 등)의 내구도를 표적해 깎는 예고형 공격.

## 7. 세계관

인류의 기술이 정점에 달한 미래, 어느 날 기계들이 **명령어**를 수신하기 시작했다. 발신원은 불명. 명령어를 받은 메카들은 자의 없이 움직이며 인류 문명을 쑥대밭으로 만들었다.

살아남은 인간들은 외곽으로 흩어졌고, 일부는 **스캐빈저**가 되었다 — 오염 구역에 들어가 멀쩡한 파츠를 골라내 팔거나 재활용하며 생존하는 사람들.

플레이어는 그 중 한 명이다. **게임의 한 런 = 오염 구역에 들어가 파츠를 모아 나오는 한 번의 출격.**

> 명령어의 정체는 의도적으로 공개하지 않는다.
