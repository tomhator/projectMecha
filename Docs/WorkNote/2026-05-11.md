#### 구현 내용

**PartSocketUI.gd**
- `@export var slot_type` 으로 슬롯 타입(ARM_L/ARM_R/BACK/LEG) 보유
- `_can_drop_data()` — 파츠 타입 호환 검사, 호환 시 초록 테두리, 비호환 시 빨간 테두리
- `_drop_data()` — `part_dropped` 시그널 발신 → AssemblyScene에서 장착 처리
- `_on_gui_input()` — 좌클릭 시 `part_unequipped` 시그널 → 슬롯 해제
- `NOTIFICATION_DRAG_END` / `NOTIFICATION_MOUSE_EXIT` — 드래그 종료 시 스타일 복원
- `StyleBoxFlat` 기반 상태별 색상: 빈 슬롯(회색), 장착됨(청색), 호환 호버(초록), 거부(빨강)

**PartCardUI.gd**
- `setup(part: PartsData)` 으로 파츠 데이터 주입
- `_get_drag_data()` — 드래그 미리보기(반투명 패널) 생성, `{ "part": part_data }` 딕셔너리 반환
- 등급별 테두리 색상: COMMON(회색), RARE(파랑), EPIC(보라)
- 손상 파츠 ⚠ 표시

**AssemblyScene.gd / .tscn**
- 노드 구조: `MainContainer > LeftColumn/ChassisPanel (소켓 4개) + RightColumn/ScrollContainer/InventoryTray (카드 목록)`
- `_build_sockets()` — 4개 슬롯을 코드로 동적 생성, 기존 장착 파츠 반영
- `_rebuild_inventory()` — 인벤토리 파츠를 PartCardUI로 동적 생성
- `_on_socket_drop()` — 기존 슬롯 파츠 인벤토리 반환 후 `GameState.equip_part()` 호출
- `_on_socket_unequip()` — `GameState.unequip_part()` 후 인벤토리 복귀
- 하중 ProgressBar + 초과 시 빨간 경고 텍스트
- 힌트 레이블: "파츠를 소켓으로 끌어다 놓아 장착하세요. 소켓 클릭으로 해제."

## 특이사항 / 결정 사항

- GDD §4.4 "A단계 — 실루엣 먼저" 기준에 따라 스프라이트 없이 컬러 블록으로 구현
- Godot 4에서 `_can_drop_data`는 호버 중에만 호출되므로, 마우스가 소켓 밖으로 나갈 때 `NOTIFICATION_MOUSE_EXIT`로 스타일 복원
- 소켓/카드 모두 `.new()`로 동적 생성하여 .tscn에 하드코딩하지 않음 (확장성 유지)
- 기존 `GameState.equip_part()` / `unequip_part()` 로직은 변경 없이 재활용

### [fix] 조립 슬롯 너비·코어 정사각형 (오버플로 수정)

- `ChassisPanel` 너비 기준 3열 그리드 셀 크기 `floor((w - 16) / 3)` (열 간격 8×2), `clamp(72, 220)` — 소켓·스페이저·코어 플레이스홀더 동일 `cell×cell` 정사각형
- `chassis_panel.resized` + `call_deferred("_refresh_chassis_layout")` 로 창 크기 변경 대응
- `PartSocketUI`: 고정 200×90 제거, `set_layout_size()`로 부모에서 크기 주입; 초기 폴백 72×72
- `AssemblyScene.tscn`: 좌·우 열 `size_flags_stretch_ratio = 1`, `LeftColumn.clip_contents = true`
- 변경 파일: `Scenes/Assembly/AssemblyScene.gd`, `AssemblyScene.tscn`, `PartSocketUI.gd`, `Docs/WorkNote/2026-05-11.md`
