# 작업 일지 — 2026-05-26

## 작업 내용

### [design] 어빌리티 트리 5레벨 보너스 밸런싱
- 15개 어빌리티 트리 노드의 `level_five_bonus_text`에서 임시 "예정" 문구를 제거하고, 5레벨 완성 역할과 누적 수치 기준을 확정.
- 행동력 보너스는 레벨별 선형 증가 대신 5레벨 도달 시 +1만 적용하도록 `AbilityTreeNode.action_bonus_at_level()`을 보정.
- 어빌리티 트리 리소스 검증에서 5레벨 보너스 문구 누락과 임시 문구 잔존을 실패 처리하도록 강화.
- 참조한 문서: [[AbilityTreeSystem]], [[TODO/TODO-NEXT]]
- 변경된 파일: `Docs/AbilityTreeSystem.md`, `Resources/AbilityTreeNode.gd`, `Resources/AbilityTree/*.tres`, `Scripts/Validation/check_resource_integrity.gd`, `Docs/TODO/TODO-NEXT.md`

### [design] 코어 외형 슬롯 부위명 개편
- 참고 이미지의 탑승형 중장비 메카 형태를 반영해 코어 외형 슬롯을 `상부 센서 마스트`, `조종석 장갑`, `견부 프레임`, `후방 장비팩`, `전면 장갑판`으로 재정의.
- 사람이 들어가는 중앙 조종석을 코어 구조의 기준으로 문서화하고, 연구/출격 UI에서 raw `visual_slot` key 대신 한국어 부위명이 보이도록 수정.
- `Resources/AbilityTree/*.tres`의 `visual_slot`을 새 슬롯 key로 갱신하고, 리소스 무결성 검사에서 허용 슬롯 목록을 검증하도록 보강.
- 참조한 문서: [[AbilityTreeSystem]]
- 변경된 파일: `Docs/AbilityTreeSystem.md`, `Scenes/CoreSelect/CoreSelectScene.gd`, `Scenes/UI/CombatUI.gd`, `Resources/AbilityTree/*.tres`, `Scripts/Validation/check_resource_integrity.gd`

## 특이사항 / 결정 사항

- 5레벨 보너스는 데모 안정성을 위해 조건부 전투 패시브로 확장하지 않고, v1에서는 누적 스탯과 운용 역할을 명확히 하는 방식으로 고정했다.
- 행동력 노드 2개를 모두 장착해도 최대 +2까지만 증가하도록 제한해 턴당 행동 폭이 과도하게 커지지 않게 했다.
- 코어 외형 슬롯은 팔·등·다리 파츠와 별개인 코어 동체 부위로 유지하되, 조종석 장갑을 독립 슬롯으로 두어 탑승형 메카 인상을 명확히 했다.
- 작업 전부터 존재한 `Resources/Cores/core_base.tres`, `.codex/`, 검증 `.uid` 미추적 파일은 이번 작업에서 수정하지 않았다.
