# 작업 일지 — 2026-05-27

## 작업 내용

### [design] 거점 시스템 기획
- 거점을 오염 구역 밖의 `외곽 은신처`로 정의하고, 다키스트 던전식 건물형 메인 화면을 기준으로 기획했다.
- 은신처 마당, 격납고, 코어 연구대, 작전 단말, 출격 게이트, 시스템 콘솔의 역할을 정리했다.
- 영구 파츠 창고와 런 인벤토리를 분리하고, 출격 시 장착 파츠 4개 + 런 인벤토리 16칸을 구성하는 흐름을 확정했다.
- 재화는 크레딧과 고철 2종으로 나누고, 크레딧은 코어 연구/서비스, 고철은 파츠 수리/제작/확장 중심으로 사용하도록 정리했다.
- 성공 시 장착/인벤토리 파츠와 재화 100% 회수, 실패 시 장착 파츠 영구 손실 + 런 인벤토리 파츠 회수 + 재화 50% 회수 규칙을 확정했다.
- 기존 문서의 코어 3종/경량 코어/인벤토리 5×6 잔재를 단일 기본 코어, 4×4 런 인벤토리 기준으로 정리했다.

- 변경된 파일: `Docs/BaseSystem.md`, `Docs/GameDesignDocument.md`, `Docs/PartsSystem.md`, `Docs/UI/HUD.md`, `Docs/TODO/TODO-NEXT.md`, `ARCHITECTURE.md`, `Docs/WorkNote/2026-05-27.md`

### [feat] 거점 MVP 구현
- `HubScene`을 프로젝트 메인 씬으로 추가하고, AI 생성 bitmap 배경 `Asset/UI/hub_outer_hideout.png`와 건물형 버튼 5개를 연결했다.
- `HangarScene`을 추가해 영구 창고, 출격 장착 슬롯 4개, 런 인벤토리 16칸, 수리/분해/되돌리기 서비스를 구현했다.
- `GameState`에 `meta_scrap`, `storage_parts`, `sortie_inventory`, `sortie_equipped_parts`, `run_scrap`, `last_run_summary`와 저장/복원/정산 API를 추가했다.
- `RewardScene`에 방 종류별 고철 보상을 추가하고, `RunEndScene`은 성공/실패 정산 요약 후 Hub로 복귀하도록 변경했다.
- `CoreSelectScene`에 은신처 복귀 버튼과 연구/출격 초기 탭 분기를 추가했다.
- `check_scene_smoke.gd`에 Hub/Hangar를 추가하고, `check_base_state.gd`로 저장/정산/수리/분해/16칸 제한 계약을 검증했다.
- 참조한 문서: [[BaseSystem]], [[PartsSystem]], [[GameDesignDocument]]
- 변경된 파일: `Scenes/Base/HubScene.gd`, `Scenes/Base/HubScene.tscn`, `Scenes/Base/HangarScene.gd`, `Scenes/Base/HangarScene.tscn`, `Scripts/Autoload/GameState.gd`, `Scripts/Autoload/EventBus.gd`, `Scripts/Autoload/PartsFactory.gd`, `Resources/PartsData.gd`, `Scenes/CoreSelect/CoreSelectScene.gd`, `Scenes/Dungeon/RewardScene.gd`, `Scenes/Dungeon/RunEndScene.gd`, `Scripts/Validation/check_base_state.gd`
