## 특이사항 / 결정 사항

- macOS 파일시스템에서 `scripts/`가 기존 `Scripts/`와 충돌하므로 검증 스크립트는 `Scripts/Validation/`에 배치.
- `gdparse`, `godot`, `godot4`가 PATH에 없으면 로컬 후보 경로를 먼저 찾고, 없으면 실패 대신 `SKIP`으로 처리.
- P0 검증은 수동 플레이테스트 대신 재현 가능한 Godot headless 스크립트로 우선 완료 처리. 실제 UI 눈검수는 이후 데모 플레이 세션에서 별도 확인 가능.
- 5티어 노드의 5레벨 보너스는 데이터에 효과 축만 기록했다. 파츠 운영 패시브의 정확한 동작과 수치는 후속 밸런스 작업으로 남김.
- 아이콘·스프라이트 UI는 후속 아트 연결을 위해 데이터 필드를 먼저 열고, 현재는 자동 생성 placeholder 비주얼로 조립/전투 레이아웃을 고정했다.
- 기존 적 상세 기획과 적 `.tres`는 역할 기반 병종표와 충돌하므로 삭제했다. 매립지 일반 병종과 수집가부터 새 역할 기준으로 다시 채웠다.
