## 작업 1. `get_available_skills()` — 파괴 파츠 스킬 제외

**파일:** `Scenes/Entities/MechaEntity.gd`

### 현재 코드 (line 20~23)

```gdscript
func get_available_skills() -> Array[SkillData]:
    return available_skills.filter(func(s: SkillData) -> bool:
        return s.skill_type != SkillData.SkillType.PASSIVE
    )
```

### 문제

- PASSIVE 타입만 걸러내고 파괴 여부는 무시함
- `TurnManager._get_usable_skills()`가 이 함수를 호출하므로,
  파괴 파츠 스킬이 `usable` 목록에 남아 `usable.is_empty()` 판정을 통과 못 함
  → 모든 스킬이 파괴 파츠 소속일 때 자동 턴 종료가 안 되는 버그

### 수정 방법

```gdscript
func get_available_skills() -> Array[SkillData]:
    return available_skills.filter(func(s: SkillData) -> bool:
        if s.skill_type == SkillData.SkillType.PASSIVE:
            return false
        var part: PartsData = _skill_to_part.get(s)
        if part != null and part.is_broken():
            return false
        return true
    )
```

**핵심:** `_skill_to_part`는 `setup()`에서 파츠 스킬만 등록하고 코어 스킬은 등록하지 않으므로,
`_skill_to_part.get(s)` 가 `null`이면 코어 스킬 → 파괴 체크 생략이 자동으로 됨.

### 영향 범위

이 수정 하나로 아래 3곳이 동시에 올바르게 동작함:

1. `TurnManager._get_usable_skills()` — 파괴 스킬 자동 제외, 턴 종료 판정 정상화
2. `TurnManager.on_skill_selected()` — 파괴 스킬로 공격 요청 자체가 불가능해짐
3. `CombatUI._rebuild_skill_buttons()` — 기존 `_broken_skills` 로직과 중복되나 충돌 없음
   (UI 레이어에서도 막고, 로직 레이어에서도 막는 이중 방어)

## 작업 2. `_affix_bonus_sum()` — 파괴 파츠 Affix 무효화

**파일:** `Scenes/Entities/MechaEntity.gd`

### 현재 코드 (line 107)

```gdscript
func _affix_bonus_sum(part: PartsData, target: Node, consume_temp: bool) -> float:
    var sum: float = 0.0
    if _has_affix(part, "mindless"):
        sum -= 0.10
    # ... 이하 affix 체크
```

### 문제

파츠가 파괴(`durability == 0`)돼도 Affix 보너스가 그대로 계산에 반영됨.

### 수정 방법

함수 시작 부분에 조기 반환 한 줄 추가:

```gdscript
func _affix_bonus_sum(part: PartsData, target: Node, consume_temp: bool) -> float:
    # 파괴된 파츠는 Affix 효과 없음 (zombie_process 예외는 추후 구현)
    if part != null and part.is_broken():
        return 0.0
    var sum: float = 0.0
    # ... 이하 기존 코드 그대로
```

> **참고 — `zombie_process` affix:**
> 기획상 "파괴 후 1턴 더 작동"이 있으나 현재 구현 안 됨. 이 조기 반환을 넣어도
> zombie_process 미구현 상태에서는 동작 변화 없음. 나중에 구현할 때
> `if part.is_broken() and not _has_affix(part, "zombie_process"):` 로 교체하면 됨.

## 작업 3. 적 파츠 저격 스킬 3종 `.tres` 생성

**저장 위치:** `Resources/Skills/`

기존 스킬 `.tres`를 참고해 아래 3개를 새로 만든다.
`SkillData.TargetSlot` enum 값: `ARM_L=0, ARM_R=1, BACK=2, LEG=3`

### 3-1. `skill_snipe_arm.tres` — 팔 저격

```
skill_name = "팔 저격"
skill_type = ATTACK (0)
skill_target = ENEMY (1)
skill_damage = 18.0
skill_action_cost = 1
target_slot = ARM_L (0)   ← 또는 ARM_R(1), 적 설정에 맞게 조정
```

### 3-2. `skill_overload_attack.tres` — 다리 과부하 공격

```
skill_name = "다리 과부하 공격"
skill_type = ATTACK (0)
skill_target = ENEMY (1)
skill_damage = 14.0
skill_action_cost = 1
target_slot = LEG (3)
```

### 3-3. `skill_emp_shock.tres` — 등 EMP 충격

```
skill_name = "등 EMP 충격"
skill_type = ATTACK (0)
skill_target = ENEMY (1)
skill_damage = 10.0
skill_action_cost = 1
target_slot = BACK (2)
```

> **밸런싱 기준:** 일반 공격(20~25 데미지)보다 약하되 슬롯 내구도 추가 감소가 보상.

일반 적에는 넣지 않는 것을 권장 (난이도 밸런스).

## 작업 4. `EnemyEntity.execute_actions()` — 슬롯 내구도 감소

**파일:** `Scenes/Entities/EnemyEntity.gd`

### 현재 코드 (line 71~82)

```gdscript
func execute_actions(target: Node) -> void:
    for action: SkillData in next_actions:
        print("[%s] '%s' 사용" % [enemy_name, action.skill_name])
        if action.skill_damage > 0.0:
            target.take_damage(action.skill_damage * attack_multiplier)
        if action.skill_defense > 0.0:
            _heal_shield(action.skill_defense)
        if action.skill_heal > 0.0:
            _heal_hp(action.skill_heal)
            print("  > [%s] HP +%.0f" % [enemy_name, action.skill_heal])
        EventBus.skill_used.emit(self, action)
    decide_next_actions()
```

### 문제

`target_slot != NONE`인 저격 스킬을 써도 슬롯 내구도에 아무 영향 없음.

### 수정 방법

`target.take_damage()` 호출 직후에 슬롯 내구도 감소 블록 추가:

```gdscript
func execute_actions(target: Node) -> void:
    for action: SkillData in next_actions:
        print("[%s] '%s' 사용" % [enemy_name, action.skill_name])
        if action.skill_damage > 0.0:
            target.take_damage(action.skill_damage * attack_multiplier)
            # 파츠 저격 스킬: 해당 슬롯 내구도 추가 감소
            if action.target_slot != SkillData.TargetSlot.NONE:
                _apply_snipe_to_slot(target, action.target_slot)
        if action.skill_defense > 0.0:
            _heal_shield(action.skill_defense)
        if action.skill_heal > 0.0:
            _heal_hp(action.skill_heal)
            print("  > [%s] HP +%.0f" % [enemy_name, action.skill_heal])
        EventBus.skill_used.emit(self, action)
    decide_next_actions()


func _apply_snipe_to_slot(target: Node, slot_index: int) -> void:
    if not target.has_method("get_part_at_slot"):
        return
    var part: PartsData = target.get_part_at_slot(slot_index)
    if part == null or part.is_broken():
        return
    part.durability = maxi(part.durability - 1, 0)
    EventBus.part_durability_changed.emit(part)
    print("  > [저격] %s 슬롯 내구도 감소 → %d" % [part.parts_name, part.durability])
    if part.is_broken():
        print("  > [파츠 파괴] %s" % part.parts_name)
```

### `MechaEntity`에 `get_part_at_slot()` 추가 필요

**파일:** `Scenes/Entities/MechaEntity.gd` (함수 추가)

```gdscript
func get_part_at_slot(slot_index: int) -> PartsData:
    for slot: CoreData.CoreSlot in GameState.equipped_parts:
        if int(slot) == slot_index:
            return GameState.equipped_parts[slot]
    return null
```

> `CoreData.CoreSlot` enum 값이 `SkillData.TargetSlot` enum 값과 순서가 같은지 확인 필요.
> `PartsData.PartsType`(ARM_L=0, ARM_R=1, BACK=2, LEG=3)과도 맞춰서 일관성 유지.

## 주의 사항

1. **`CoreData.CoreSlot` vs `SkillData.TargetSlot` 정수값 일치 여부**
   `_apply_snipe_to_slot()`에서 정수 비교를 쓰므로, 두 enum의 순서가 다르면 잘못된 슬롯을 타격함.
   `CoreData.gd`의 `CoreSlot` enum을 먼저 열어서 ARM_L/ARM_R/BACK/LEG 순서 확인할 것.

2. **`zombie_process` affix**
   작업 2에서 조기 반환 추가 후, `zombie_process`를 가진 파츠가 파괴돼도 즉시 Affix가 꺼짐.
   기획 의도와 다르지만 현재 미구현 상태이므로 TODO 주석을 남기고 일단 넘어가도 됨.

3. **CombatUI `_broken_skills`와 중복**
   작업 1 완료 후에도 `CombatUI._rebuild_skill_buttons()`의 `_broken_skills` 로직을
   지울 필요는 없음. UI 레이어 방어막으로 남겨두면 됨.
