# 작업 일지 (2026-05-10)

## 작업 내용

### [feat] 전투 로직 고도화 및 던전 시스템 개선

- **전투 로직 및 UI 개선**
  - `TurnManager.gd`: 플레이어 턴 시작 시 쿨다운 정보를 `CombatUI`에 전달하도록 `player_action_required` 시그널 파라미터 수정.
  - `CombatUI.gd`: 스킬 쿨다운 정보를 받아 쿨다운 중인 스킬 버튼을 비활성화하고 남은 턴 수를 표시하도록 수정. 패시브 스킬은 버튼으로 표시되지 않도록 예외 처리.
  - `MechaEntity.gd`: 공격 스킬 사용 시 `GameState.attack_multiplier`를 곱하여 최종 데미지를 계산하도록 수정.
  - `GameState.gd`: 공격 배율(`attack_multiplier`) 변수 추가 및 런 시작 시 초기화 로직 추가.

- **던전 씬 및 이벤트 구현**
  - `DungeonMapScene.gd` / `.tscn`: 현재 크레딧을 표시하는 `CreditsLabel` 추가 및 업데이트 로직 구현.
  - `EncounterScene.gd` / `.tscn`: 랜덤 인카운터 이벤트 목록(5종) 및 선택지(진행, 무시)에 따른 결과 처리 로직 구현. (HP 증감, 크레딧 획득 등)
  - `WorkshopScene.gd` / `.tscn`: 크레딧을 소모하여 HP나 쉴드를 회복하는 서비스 목록(3종) 구현. 현재 상태(HP, 쉴드, 크레딧)를 표시하는 UI 추가.

- **보상 시스템 고도화**
  - `RewardManager.gd`: 하드코딩된 파츠 경로 배열 대신 `DirAccess`를 사용하여 `Resources/Parts/common`, `rare`, `epic` 폴더 내의 파츠 리소스를 동적으로 수집하도록 로직 변경.
  - 기존 `Resources/Parts` 하위의 개별 파츠 리소스 파일들을 등급별 폴더(`common`, `rare`, `epic`)로 이동 및 재구성.

- **기타 수정**
  - `RewardScene.gd`: 보상 선택 후 인벤토리에 추가하고 던전 매니저를 통해 진행을 재개하도록 로직 수정.
