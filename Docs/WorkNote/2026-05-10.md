## 특이사항 / 결정 사항

- **조우이벤트 B 디버프 방식**: `PartsData.is_damaged` 플래그 + `MechaEntity._skill_to_part` 매핑 방식 채택. Resource 객체는 런 내에서 동일 참조이므로 플래그 변경이 전 세션에 반영됨. 런 종료 시 `start_run()`에서 인벤토리가 초기화되므로 세션 간 오염 없음.
- **작업대 업그레이드**: `skill_damage *= 1.2`는 Resource 객체를 직접 변경. 런 내에서는 의도한 동작이나, 같은 런 내 동일 스킬을 여러 번 강화하는 것이 가능함. 현재는 스코프 내에서 허용.
