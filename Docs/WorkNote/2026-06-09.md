### [design] boon 방향 합의 + 코어 어빌리티 현황 진단

어제(06-08) 맵 재설계에서 "내일"로 미뤄둔 **교리 노드 런-한정 boon** 방향을 대화로 잡았다. boon 본 설계는 다음 세션으로 이월하기로 하고, 오늘 합의·진단 내용을 세션 컨텍스트로 보존했다.

**합의된 boon 방향 (사용자 확정):**
- 하데스 풀카피 아님. 맵 갈림길(최대 4) 중 일부가 **교리 패시브 부여**. 교리 노드 = 전투/조우/상자.
- 콤보는 약한 시너지만 — **같은 교리 반복 선택 시 약한 시너지**.
- boon = **미리 세팅해온 코어 어빌리티 강화** (던전 전 빌드 세팅을 런 중 교리 노드가 강화). **새 인터랙트 추가 안 함.**
- 강화 대상은 노드 종류(전투/조우/상자)별로 다름 — **미정**.

**코어 어빌리티 현황 진단 (코드+문서 조사):**
- 출격 빌드 3칸 = 기본공격 택1 / 어빌리티 트리 티어별 1개 / 파츠 활용 어빌리티 택1.
- 5동사 교리 매핑: 정비공=수리·덧대 / 해체공=투척·교체·소모.
- **발견된 갭:** ① **2교리 vs 3축 미정합**(boon=2교리, 트리=공/방/유 3축) ② **교리가 코드에 0건**(문서뿐, grep 전무) ③ `core_skill_role` 3종뿐·**수리 enum 누락** ④ **던전 전 빌드 세팅 화면 부재**(전투 UI는 스킬 평면 나열).

**결정:**
- boon은 단독 불가 — **갭①(2교리 코어 어빌리티 정의)이 선행 의존성**. 다음 세션에서 **boon + 교리 코어 어빌리티 통합 설계**로 진행.
- 동결 게이트(nV)는 구현만 막고 설계는 자유 → 통합 설계 문서 작성 가능.

### [design] 코어 시스템·코어 어빌리티·교리 재기획 (저녁 세션)

아침 boon 진단에서 드러난 갭(2교리 vs 3축, 코어 어빌리티 분류 얇음)을 풀기 위해, "코어 성장과 파츠 재기획" 중 **코어 기둥**을 대화로 전면 재기획했다. 산출물 = [[Docs/superpowers/specs/2026-06-09-코어시스템-재기획-design]].

**확정 사항:**
- **코어** = 단일 주인공(스토리 상수)·런 중 고정·메타 노드 트리 성장. 이중 주인공(스토리=코어 / 게임플레이=파츠).
- **코어 어빌리티** = 파츠 조작 동사 그 자체. 출격 전 3슬롯 로드아웃. 상시 기본기 = 기본공격(노드 택)·긴급 장착.
- **동사 8종 → 3택:** 교체·수리·투척·덧대·과부하·재가동·복제·임시방편. 휘발 3종(재가동=화장·복제·임시방편)=전투 한정 ↔ 작업대=영구.
- **교리** = 형식 태그축(태운다/살린다, 확장형). (C) 노드선언 + 런누적.
- **키스톤** = 노드 교리 점수 합산 → 문턱 + 기본공격 타이브레이커. 미달 시 제너럴리스트.
- **메타** = 스탯 뻥튀기 아닌 "규칙 비트는 키스톤 + 트레이드오프".

**boon 의존성 해소:** 아침 진단의 **갭①(2교리 vs 3축)·갭③(분류 얇음·수리 누락)을 설계로 해소.** 교리=형식 2교리 태그축, 동사 8종에 수리 포함. boon 남은 선행 = affix 교리 태그(파츠 기둥).

**후속 패스 — 동사 디테일 확정:** 저장소 최신화(origin/main rebase) 후 코어 어빌리티를 더 진행. **공통 비용 축 = AP + 재료(단일축)**, **시작 1슬롯**, **투척**(기본=파손 파츠만 → 메타 두 갈래 택1: 장착/멀쩡), **수리**(파손 소모 → 업글로 장착·인벤 멀쩡 소모), **휘발 산출물**(즉시 장착→빈슬롯 없으면 큐 최상위) 확정. **통일 패턴: 동사=기본형+메타 업그레이드(일부 두 갈래 택1)** = §5 메타와 동형. 스펙 §2.1·§2.3.1·§2.4·§5·§8 갱신.
