### TODO 1. 던전 게임 루프 (`GameDesignDocument.md` v1.2)

- §2.1 런 구조: 첫 런(빈손 시작) / 이후 런(사전 조립 화면) + 10층 루프
- §2.2 층 구성: 1~3 일반 / 4~5 분기+작업대 / 6~8 일반+엘리트 / 9 작업대 / 10 보스
- §2.3 방 타입: 드롭 0~2개 **전부 획득** (선택 없음), 인벤 풀 시 유실
- §2.4 크레딧 범위: 일반 15~25 / 엘리트 45~65 / 보스 85~105
- §2.5 결과 화면 12항목: 총 전투 수·크레딧 수지·파츠 희귀도별 획득 수·파손/파괴 수 등
- §4.4~4.6: 매 전투 후 재조립 가능(교체 시 파손), 인벤토리 16→24→28 (코어 업그레이드)
- §7.3 작업대 비용: 50% 41cr / 100% 79cr 고정, 부품 강화 `(PARTS_VALUE × 0.7) + 15`

**핵심 논의 내용**:
- 드롭 방식: 당초 선택식이었으나 "전부 가져간다"로 변경 (선택 피로 제거)
- 작업대 위치: 매 전투 후 재조립은 유지. 작업대 노출 빈도 고민 → 4~5층·9층 고정 배치로 확정
- 인벤토리: 30슬롯(5×6)에서 기획 재조정 → 16슬롯(4×4) 시작, 코어 업그레이드로 확장
- 수리 비용: 코어 종류 무관 고정 비용. 이벤트·스킬로만 변동 가능

### TODO 3. 전투화면 메카 표시 기획 + 구현 (`CombatScreen.md` + 코드)

**기획 변경 과정 (중요)**:

초기 설계는 **블록형 십자 레이아웃** (조립씬 미니어처)이었으나, 세션 중 방향을 전면 재설계.

> "전투화면에서는 실제로 메카가 조립된 모습을 보여주고 싶은거야."  
> — 스케치 공유: 둥글고 뭉툭한 SD 캐릭터 형태의 로봇

재설계 결정 사항:
1. **MechIllustration** (PlayerColumn): 파츠별 이미지 레이어를 합성한 SD 캐릭터 일러스트. 손상 tint 없이 항상 깨끗하게 표시.
2. **PartsStatusHUD** (화면 좌하단): 별도 십자형 아이콘으로 손상 상태 표시 (디아블로식 장비 내구도 UI).

HUD 아이콘 배치: 가로 나열형 / 십자형 / 세로 목록형 논의 → **십자형** 확정 (조립씬 배치감과 일치).

**코드 구현 (플레이스홀더)**:
- `CombatUi.tscn`: `PartsLabels VBoxContainer` → `MechIllustration Control` (140×170px)
- `CombatUI.gd`:
  - `_build_mech_layers()`: 5개 `ColorRect` 레이어 생성 (BACK z0, LEG z1, CORE z2, ARM_R z3, ARM_L z4)
  - `_build_parts_hud()`: 3×3 `GridContainer` 십자형 HUD 생성, `BottomHBox` 맨 앞 삽입
  - `_refresh_hud_icon(slot)`: `StyleBoxFlat`으로 정상/손상/파괴 상태 스타일 적용
  - `_on_part_durability_changed()`: 변경된 슬롯만 HUD 갱신 (전체 재빌드 → 슬롯 단위 최적화)
