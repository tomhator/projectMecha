### [design] 동사 업그레이드 묶음 + 밸런스 초안 (후속 패스)

[[Docs/superpowers/specs/2026-06-09-코어시스템-재기획-design]] §2.3.1 "나머지 동사 후속 패스"를 이어, 동사별 **기본형 + 업그레이드 옵션 묶음(유저 1택)** 을 정리했다. 산출물 = [[Docs/superpowers/specs/2026-06-10-동사-업그레이드-묶음-밸런스초안-design]].

**⚠ 정정 (세션 후반 — "6종 표"는 정본 아님):**
직전(유실) 세션의 "6종 표"(복구·개조·연성·방출…)를 git 이력·문서와 대조한 결과 정본이 아님이 확인됐다. **8종(§2.3) 기준으로 재정리.**
- **복구** = 수리 계열 → 수리가 덧대기에 흡수되므로 **삭제**.
- **개조** = 사용자 발언 기록 0(AI 명명). 실제 의도 = 덧대기 시 재료 affix 이전 → **덧대기 업그레이드로 복원**.
- **연성** = 근거 0(옛 git 매치는 "유연성" 부분문자열) → **폐기**.
- **방출** = 8종 대응 없음 → **폐기**.

**확정 (✅):**
- **수리 → 덧대기 흡수**, **투척 = 투척**, 교체 = §2.2 상시 기본기. 과부하·재가동·복제·임시방편 불변.
- **덧대기 업글** — 재료 파츠 affix 중 1개를 **0~15% 확률로 결과 파츠에 이전.**
- **투척 업글(§2.3.1)** — 두 갈래 택1(장착 파츠 ↔ 인벤토리 멀쩡 파츠 던지기, 둘 다 ✕).
- **복제 업글** — 강화/다중(2)/영속. *공명 제외.* (이전 세션 합의·재확인 권장.)
- **임시방편 업글** — 즉석 affix / 공격형(위력↑+처치 AP환급) / 긴급 일괄(빈 슬롯 수만큼). (이전 세션 합의·재확인 권장.)
