## P0.5 — 핵심 재미 검증 (플레이테스트)

> 기준 문서: [[superpowers/specs/2026-05-31-검증기준-온보딩-안할것-design]]
> 핵심 재미 = "교체의 긴장". 아래는 전달력·재미 검증(층위 2·3). 기능 검증(층위 1)은 `validate.sh`가 커버.

- [ ] **V1 교체 수락률 30~70%** — 점유 슬롯 교체 제안 대비 수락 비율. 플레이테스트 5런 집계. 밴드 벗어나면 파손 비용·드롭 등급 곡선 재조정(GDD §4.5/§2.3), 연속 2회 실패 시 교체 비용 모델 재검토.
- [ ] **V2 첫 교체 딜레마 3분 이내(2층)** — 신규 플레이어 관찰. (구현: 온보딩 보장)
- [ ] **V3 교체 결과 인지 80%+** — 교체 후 강해졌나/약해졌나 정답률. 실패 시 조립/교체 UI 비교 표시 개선, 3회 실패 시 "교체 전후 수치 1줄 요약" 강제 표시.
- [ ] **V4 한 화면 정보로 첫 교체 판단** — 추가 설명 없이 진행. 신규 플레이어 관찰.
- [x] **V5 런당 교체 딜레마 3회+ / 4층 연속 0회 없음** — headless 측정(`check_swap_dilemma_frequency.gd`, Godot 4.6.2) 평균 9.15회/런(목표 ≥3.0), 0회 런 0/200 → PASS (2026-06-01). (사람 플레이테스트 5런 집계는 V1·V6과 함께 별도 수행 권장)
- [ ] **V6 "한 판 더?" 과반(3/5) yes** — 사후 질문. 실패 시 코어루프 기획 재진단. **이 통과 전까지 NOT DOING 목록 해제 금지.**
