## 1. 개요

적들은 기계신(Machine God)의 **Protocol** Interrupt 이후, 각자가 처해 있던 **환경**에 의해 다른 형태로 변이한 기계들이다.

Protocol이 닿았을 때 그 기계가 **어디에 있었는가**가 행동 방식과 외형을 결정한다.  
이를 기준으로 적들을 **지역별 체계**로 분류하며, 각 지역의 기계들은 공통된 별칭으로 불린다.

인간 세력은 별도 진영으로, 이 체계와 독립적인 자체 단계 시스템을 가진다. (데모 이후 기획 예정)

## 3. 매립지 (Junkyard) — "스크래퍼" ★데모 구현

### 3.1 지역 설정

Protocol이 닿았을 때 이미 폐기되거나 방치된 기계들.  
제대로 된 코어도, 정상 회로도 없이 버려진 채 Protocol 신호를 수신했다.  
Signal을 해석할 능력조차 없어 잡음(Noise)만 반복하며 움직인다.

조잡하게 붙은 고철 덩어리들이 뒤엉킨 외형. 균형도 없고 형태도 제각각.  
하지만 수가 많고 예측 불가하다는 점에서 초보 스캐빈저에게 충분히 위협적이다.

**공통 특성**
- HP 낮음 / 쉴드 없거나 매우 낮음
- AP 낮음 (1~2)
- 스킬 풀 단순 (2~3종)
- 슬롯 0~1개 (팔 or 등 중 랜덤, 다리 없음)
- AI: 완전 랜덤

### 3.3 지역 보스 — 수집가 (The Collector)

매립지 어딘가에서 오랫동안 고철을 모아온 존재.  
Protocol이 닿기 이전부터 파츠를 모으는 행동을 반복했고, Protocol 이후에도 그 행동이 멈추지 않았다.  
현재는 6개의 팔을 뻗어 무엇이든 집어넣고, 잃으면 다시 채운다.

**보스의 핵심 정체성**: 파괴할수록 다시 채워넣는다. 팔을 부수는 게 답이 아니다. 결국 코어를 노려야 한다.

|--------|----|------|------------|------|

#### 팔 탈취 (Arm Theft) — 수집가 전용 특수 스킬

> **핵심 콘셉트**: 수집가는 팔이 파괴되면 그 자리를 플레이어의 팔로 채운다.  
> 플레이어 입장에서 "내가 없앤 적 팔이 내 팔로 대체되는" 역설적 위협.

**발동 조건**
- 수집가의 현재 활성 팔 수 < 4 (팔이 1개라도 파괴된 상태)
- 이 조건이 충족될 때만 `decide_next_actions()`의 스킬 후보 풀에 팔 탈취가 포함됨

**효과 순서**

```
1. 플레이어의 ARM_L 또는 ARM_R 중 장착된 슬롯을 랜덤 선택
   └ 두 슬롯 모두 비어있으면 → 팔 탈취 실패, 단순 타격(10 데미지)만 적용
2. 선택된 슬롯의 파츠를 강탈 → 해당 슬롯이 빈 슬롯이 됨
3. 강탈한 파츠의 스킬을 사용하는 새 보스 팔 엔티티 생성
   └ 새 팔 HP: 45 (기존 보스 팔과 동일)
   └ 스킬: 강탈한 파츠의 primary 스킬 1종 그대로 사용
   └ 대미지 배율: 수집가 공격 배율(1.3) 적용
4. 수집가의 활성 팔 수 +1
```

**주요 전략 포인트**

| 상황 | 결과 |
|------|------|
| 플레이어가 강력한 ARM 장착 중일 때 탈취 → | 그 스킬이 수집가의 무기가 됨 |
| 모든 ARM 슬롯이 빈 상태 → | 탈취 불발, 소데미지만 적용 |
| 탈취 후 수집가 팔 4개 복구됨 → | 다음 턴 탈취 조건 불충족, 탈취 불가 |
| 탈취된 팔을 다시 파괴하면 → | 파츠는 소멸 (반환 없음) |

> [!warning] 구현 복잡도
> 팔 탈취는 다음 추가 구현이 필요:
> - `BossCollectorEntity`: 플레이어 파츠 데이터에 접근해 강탈 처리
> - `MechaEntity.steal_part_at_slot(slot)`: 슬롯 파츠 제거 + EventBus 발행
> - 강탈한 파츠의 `SkillData`를 가져와 새 `CollectorArmEntity` 인스턴스 생성
> - 데모 우선순위에 따라 **심플 버전** 먼저 구현 가능:  
>   팔 탈취 성공 시 플레이어 ARM 슬롯 비우기만 하고, 새 팔 생성은 생략 (단순 빈 슬롯 처리)

#### 재수집 페이즈 (Re-collection Phase)

모든 팔이 파괴되었을 때 발동.

```
[재수집 발동 조건] 팔 HP == 0인 팔의 수 == 현재 장착 팔 수
```

| 단계 | 내용 |
|------|------|
| 발동 | 해당 턴 보스의 팔 스킬을 사용하지 못함 (코어 스킬은 가능) |
| 재장착 | 보스 AP 소모 없이, 팔 **4개**를 랜덤으로 새로 장착 |
| 제한 | **무한 반복 가능** — 팔을 다시 부수면 또 채운다 |

> [!note] 플레이어 판단 포인트
> - 팔을 다 부수면 그 턴 팔 공격은 없지만 보스는 고철 흡수로 회복할 수 있다.
> - 재장착 후 새 팔이 더 위험한 조합일 수 있다 (압착 팔 4개 등).
> - "팔을 제거하는 게 목적"이 아니라 "코어를 빨리 잡는 게 목적".

---

#### 전투 흐름 예시

```
[보스 1턴]  압착 팔 → 유압 압착 (22) + 드릴 팔 → 드릴 돌격 (16 + 쉴드 관통)
[플레이어]  방어 팔 공격 → 방어 팔 HP 45→0 (파괴) / 코어 공격
            ※ 이 순간 활성 팔 3개 → 보스 코어 행동 풀에 '팔 탈취' 추가됨

[보스 2턴]  스윕 팔 → 광역 스윕 (15) + 코어 → 팔 탈취!
            → 플레이어 ARM_L 슬롯의 '고출력 드릴'을 강탈
            → 플레이어 ARM_L 빈 슬롯이 됨
            → 새 보스 팔 '고출력 드릴 팔' 생성 (HP 45, 스킬: 플레이어 드릴 스킬 그대로)

[플레이어]  내 드릴 스킬이 보스 팔로... 압착 팔 집중 공격 → 파괴 / 고출력 드릴 팔 공격
[보스 3턴]  고출력 드릴 팔 → (원래 내 스킬로 나를 공격!) + 등 EMP 충격 → BACK 내구도↓
                                      ↓ 이 턴에 마지막 팔 파괴됨
[재수집]    팔 4개 재장착 (랜덤). 강탈된 '고출력 드릴 팔'은 파괴 시 소멸 — 반환 없음.
```

---

#### 디자인 컨셉

- 외형: 중앙의 거대한 핵심 코어에서 4~6개의 고철 팔이 뻗어 나온 형태.  
  팔마다 형태가 다르다 (집게, 드릴, 압착기, 철판 등). 대칭 없음.
- 팔이 파괴될 때: 분리되어 바닥에 떨어지는 연출. 코어는 그 자리에서 팔꿈치만 남아 덜렁거림.
- 재수집 시: 주변 고철 파편들이 끌려와 새 팔 형태로 결합하는 연출.
- 고철 흡수 시: 주변 잔해들이 빠른 속도로 코어 내부로 흡수. 코어 표면이 잠시 빛남.

#### 구현 메모

> [!warning] 구현 구조
> **팔 = 별도 EnemyEntity 인스턴스** (슬롯 아님)
> - 보스 전투 = `enemies` 배열에 수집가 코어 1개 + 팔 N개가 함께 등록됨
> - 팔도 `EnemyEntity`(또는 경량 서브클래스 `CollectorArmEntity`)로 취급
> - 팔 HP = 0 → 해당 entity가 배열에서 제거됨 (일반 적 사망과 동일 처리)
> - 재수집 발동 시 → 새로운 arm entity들을 `enemies` 배열에 추가
> - 팔 entity의 action: 자신의 스킬 1종을 보유, 해당 턴에 AP 소모해 사용
> - 전투 UI: 코어와 팔이 모두 타겟 선택지로 표시, 각 팔 HP 바 개별 표시

**미결**
- [ ] 팔 파괴 시 드롭 여부 (파츠 드롭? 크레딧 일부? 없음?)
- [ ] 팔 종류별 시각 에셋 (압착 팔·드릴 팔·집게 팔·스윕 팔·방어 팔 각 스프라이트)
- [ ] 보스 BGM / 연출 방향

## 3.4 저격 스킬 명세 (Snipe Skill Specification)

### 개요

적이 플레이어의 **특정 파츠 슬롯을 겨냥**해 내구도를 직접 깎는 스킬.  
일반 데미지와 함께, 해당 슬롯 파츠의 내구도를 1 감소시킨다.

**배정 원칙**
- 매립지 일반 병종 (스크래퍼/러셔/가드유닛): **저격 스킬 없음** — 단순하고 예측 불가한 성격에 맞게
- 수집가 코어: **등 EMP 충격** 1종만 보유
- 구 도시·연구시설 병종: 추후 기획 시 저격 스킬 배정 예정

> [!note] 기존 스킬 수정 필요
> `skill_rapid_fire`, `skill_heavy_punch`, `skill_scatter_shot`, `skill_cannon_shot`, `skill_colossus_strike`에  
> 이전 디자인 잔재로 `target_slot` 값이 세팅되어 있음.  
> 구현 시 이 값들을 **NONE(-1)으로 초기화**해야 함.  
> 그대로 두면 저격 로직 구현 즉시 일반 적도 저격 스킬을 쓰게 됨.

## 4. 구 도시 (Old City) — "위병" ▷추후 예정

### 4.1 지역 설정

Protocol이 닿았을 때 **현역으로 도시를 수비·관리하던** 기계들.  
경비 로봇, 교통 통제 장치, 유지보수 기계 등 시민 생활 밀착형 기계들이다.

원형을 온전히 유지한 채 Protocol에 Interrupt되었다.  
자신이 맡았던 **"임무"를 여전히 수행 중**이라 믿고 있어 행동 패턴이 규칙적이고 조직적이다.  
도시가 파괴된 이후에도 구역을 순찰하고, 침입자를 배제하고, 시설을 "유지"한다.

**공통 특성**
- HP / 쉴드 모두 스크래퍼보다 높음
- AP 2~3
- 스킬 풀에 원래 기능의 잔재가 남아 있음 (순찰·차단·경보 등)
- 슬롯 0~2개
- AI: 완전 랜덤 (단, 방어 스킬 비중이 스크래퍼보다 높음)

## 5. 연구시설 (Lab) — "레플리카" ▷추후 예정

### 5.1 지역 설정

Protocol이 닿았을 때 **개발·실험 중이던 프로토타입** 기계들.  
완성되지 않은 회로, 미조정 출력값, 불안정한 AI가 Protocol과 뒤섞였다.

능력치의 편차가 크고, 경우에 따라 "있어서는 안 될 기능"을 발현하기도 한다.  
"레플리카"라는 별칭은 — 이들이 **무언가를 모방하도록** 설계된 프로토타입이라는 데서 유래.

**공통 특성**
- 스탯 편차 큼 (일부는 매우 공격적, 일부는 매우 방어적)
- AP 3~4
- 스킬 풀에 실험적·불안정한 능력 포함
- 슬롯 2~3개
- AI: 완전 랜덤 (스킬 예측 가장 어려움)

## 6. 적 AI 원칙

### 6.1 기본 원칙 (전 지역 공통)

- 적은 **완전 랜덤으로 스킬 선택**한다. 현재 체력·전황에 따른 조건부 AI 없음.
- **다음 턴 행동을 미리 공개** — 플레이어가 대응 판단 가능.
- 스킬 선택 기준: 현재 AP 합산이 행동력 이하인 스킬 조합 중 랜덤 선택.

> [!note] 지역 보스 AI 고도화 여부는 미결. 체력 페이즈별 패턴 도입 검토 가능.

### 6.2 행동 예고 규칙

- 플레이어 턴 시작 전, 다음 적 행동이 UI에 표시됨.
- 예고 = 실제 실행 스킬 (페이크 없음).
- 파츠 저격 스킬이 있는 경우 타겟 슬롯도 함께 표시.
