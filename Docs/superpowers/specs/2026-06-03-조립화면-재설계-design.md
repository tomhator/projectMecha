# 조립 화면 재설계 + 코어 재정의(파츠가 주인공) — 설계

> office-hours 브레인스토밍(2026-06-03) 산출물.
> UI/UX 개편의 **두 번째 화면(조립)** 설계다. 첫 화면(전투, 2026-06-01)을 잇는 확장.
> **중요:** 이 문서는 게임의 핵심 재미(코어)를 **"교체의 긴장" → "파츠가 주인공"** 으로 전환한다.
> 그 전환의 첫 적용 화면이 조립이다. 코어 변경의 파급(GDD·검증기준 정리)은 §8에 명시.

---

## 0. 배경 / 코어 전환

- **진단:** 핵심 재미 검증 결과 V1~V6 전부 미통과 + 밸런스가 ①(자동 교체)·②(절대 교체) 극단. 사용자 판단: **"교체의 긴장"이 잘못된 코어.** 칼날 위 30~70% 수락률 밴드에서만 재밌는 구조적 취약성이 "밍숭맹숭"의 원인.
- **새 코어 (확정):**
  > **"파츠가 주인공 — 적극적으로 사용·버리기·교체·활용한다."**
  - 옛 코어("교체를 아프게" = 절제 강요)를 **대체**한다.
  - 교체는 *권장*되고, 비용은 *의도적 소비(투척·희생)* 에만 붙는다.
- **효과:** 칼날 밸런스가 사라진다. 더는 "교체를 아프게" 튜닝하지 않고 **"교체가 잘 흐르고 짜릿한가"** 로 튜닝한다. 어제 전투 재설계의 **해체공 축**(파츠=연료)이 이 방향이었음.

---

## 1. 코어가 만든 규칙 변화 (조립 관련)

- **조립 교체 = 자유(무비용).** 다음 전투를 진지하게 준비하라고 마찰을 없앤다.
- **전투 교체 = 손패 뒤로(템포 비용), 무손실.** 갈아낸 파츠는 파괴되지 않고 로테이션 뒤로 가서 잠깐 못 쓴다.
- **소비(투척·희생) = 파괴.** "버리기"라는 동사는 여기 몰아넣어 의도적 선택+페이백으로 만든다.
- **예비 = 하이브리드:** 인벤토리가 부위별 예비로 자동 공급 + 슬롯별 "핀"으로 우선순위. (핀 안 하면 자동, 전투 교체 순간 핀+상위 몇 개만 노출 → 정보 과부하 ④ 방지)
- **파손-점유 인벤토리 압박 제거(옛 GDD §4.5~4.6 무효):** 교체가 무손실이라 파손 파츠 점유가 사라짐. 인벤토리는 "불안 관리"가 아니라 **"예비 자원 풀"** 이 됨.
- **파츠 = 코어의 소모성 장갑 + 화력 (핵심 재프레이밍):**
  - 빈 슬롯이면 그 공격이 **코어 HP로 직격**(기존 `빈 슬롯 저격 = HP 데미지` 규칙의 일반화). 파츠는 갈아끼우면 되지만 코어 HP는 못 바꾼다 → **부위를 빼서 위협을 회피할 수 없다.**
  - 팔 = 화력·AP(GDD §5.2). 팔 빼면 기본공격 2AP만 → 엘리트 못 깸.
  - 따라서 위협은 **"소모전"** 이지 "회피 신호"가 아니다. 새 코어("막 쓰고 버리자")가 *정답 플레이*가 된다.

## 2. 비주얼 규칙 (전투와 공유)

- 전투 화면 문서 **§2 ItB×DD 규칙 그대로 계승.** 무대=무게(DD), 계기판=명료(ItB), 하드 베젤 없음.
- **공통 컴포넌트(전 화면 공유, 컴포넌트화):**
  - `TopStatusBar` — 층/거점명 · 코어 HP · 쉴드 · 크레딧 · 설정. 좌측 슬롯만 가변(던전=층 / 거점=거점명).
  - `TabNav` — 세그먼트 탭. 던전 전투 사이=[맵][조립], 거점=[마당][격납고][코어연구][출격] 등. (공급/허브는 §7 다음 패스)
- 이는 §8 로드맵이 말한 "전 화면 공통 디자인 컴포넌트 추출"의 첫 조각.

## 3. 조립 화면 레이아웃

**이유:** 다음 위협을 보고 → 자유롭게 빌드(장착+핀)를 짜서 다음 전투를 준비. 결정의 무게가 "비용"이 아니라 **"이 위협엔 이 빌드가 답인가"** 에 실린다.

```
+----------------------------------------------------------+
| (A) 상태바[공통]  층4  HP 72/100  쉴드 18/20  크레딧 128 [설정]|
+----------------------------------------------------------+
| (B) 탭[공통]            [ 맵 ]   [ 조립<선택> ]            |
+----------------------------------------------------------+
| (C) 위협[조립전용]  다음=엘리트  Breaker x2 (팔 소모전!)    |
+------------+----------------------------+----------------+
| (D)        |                            | (F)            |
| 파츠 트레이 |   (E) 무대                  | 턴 프리뷰       |
|  팔 [][][] |       내 메카 실루엣         |  (전투 덱 미니)  |
|  등 [][]   |       (끼울때 실루엣 변화)    |  공격 오R 12   |
|  다리 [][] |       위협 부위 경고색        |  막기 왼팔     |
|  --------  |                            |  AP 6 / 행동 4 |
|  분해/버리기 |                            |  ----          |
|            |                            |  예비(핀)팔>고출력|
+------------+----------------------------+----------------+
|              (G) [ 확정 -> 다음 방 ]  (가운데 정렬)        |
+----------------------------------------------------------+
```

| 구역 | 레이어 | 내용 |
|------|--------|------|
| **(A) 상태바** | 계기판(공통) | 층 · HP · 쉴드 · 크레딧 · 설정. 전투와 동일 컴포넌트 |
| **(B) 탭** | 계기판(공통) | 맵 ↔ 조립 전환. 공통 `TabNav` |
| **(C) 위협 readout** | 계기판(조립전용) | 다음 방 종류 + 적 구성 + 위협 태그. 워딩은 **소모형**("팔 소모전 — 예비 권장"), 회피형 금지 |
| **(D) 파츠 트레이** | 계기판 | 보유 파츠를 부위별(팔/등/다리)로. 드래그→슬롯(자유 교체). **하단에 분해/버리기 영역**(드래그=버림+고철 약간 회수) |
| **(E) 무대 — 내 메카** | 무대(DD) | 실루엣. 끼울 때마다 실루엣·무게 변화(5/10 시각적 조립 계승). **위협받는 부위 경고색**. 클릭=탈착 |
| **(F) 턴 프리뷰 + 예비/핀** | 계기판 | 이 빌드의 전투 턴(스킬 아이콘+AP+툴팁, **전투 스킬 덱과 같은 모양**), 실시간 갱신. 슬롯별 예비/핀 |
| **(G) 확정** | 계기판 | 가운데 정렬. → 다음 방 |

**한 번에 읽는 흐름:** (C)위협 본다 → (D)트레이에서 (E)무대로 드래그 → (E)실루엣·(F)턴 프리뷰 즉시 갱신 → 핀 줄세우기(선택) → (G)확정.

**"파츠가 주인공" 장치:** 파츠에 이름·이력·affix를 *성격*으로 표시. 소비 가능 파츠는 "태울 수 있음" 표식.

## 4. 전투 화면과의 연결 조항 (한 몸 보장)

조립과 전투가 맥락·재미·UX로 이어지도록 못박는 4개 조항.

1. **메카 에셋 동일:** 조립 무대(E)의 메카 = 전투 무대의 메카. 같은 에셋·같은 DD 처리. "내가 만든 그 기체가 싸운다"가 깨지지 않게.
2. **언어 동일:** 위협 readout 언어 = 전투 예고 언어(같은 표현·아이콘). 약속(조립)과 지불(전투)이 정확히 맞물림.
3. **턴 프리뷰 = 전투 스킬 덱의 미니:** 조립에서 배치한 스킬 아이콘 배열이 전투 하단 덱에 그대로. "내가 짠 게 곧 내가 쥐는 패."
4. **재미 원칙 — "조립은 태세, 전투는 즉흥":**
   - 조립엔 위협의 **종류(벡터)만** 준다("팔이 갈린다"). *언제·어느 팔·몇 뎀*은 안 준다.
   - 전투에서 **구체**가 닥치고, 갖춰둔 수단 중 무엇을 언제 쓸지 그 순간 고른다.
   - → 조립=패를 유리하게 쌓기(태세), 전투=부서지는 대로 플레이(즉흥). **완벽 카운터 금지**(위협 정보를 너무 완전하게 + 카운터를 너무 확정적으로 주면 전투가 결정론이 됨).

## 6. 고려한 접근 (이번 세션 결정)

- **세션 렌즈:** Builder.
- **코어:** 교체의 긴장 *살림/느끼게 바꿈/버림* → **버림**(파츠가 주인공).
- **전투 교체 비용:** 파손(B) / 완전 무손실(C) / **손패 뒤로+소비 파괴(A) 채택** — 교체(템포)와 소비(파괴)를 분리해 4동사를 각각 살림.
- **예비 구조:** 인벤토리=예비(A) / 큐레이션 덱(C) / **하이브리드 채택**(A 기본 + 슬롯별 핀). 어제 전투 §3.5 "맥락 집중"의 직역.
- **버리기 위치:** **트레이(D) 채택.** 조립=차분한 정리 / 전투=뜨거운 소비, 같은 동사 두 톤으로 분리.
- **개선안:** 4개 중 ①위협→위험부위 하이라이트 ②버리기 동선 **채택**. ③파츠 서사 ④빌드 A/B 비교 **보류**.

## 7. 미결 / Open Questions

### 공급 / prep 허브 (다음 설계 패스 — 사용자 확정)
- 새 코어는 **throughput 게임**이라 공급 < 소비면 옛 실패모드 ⑤(교체할 파츠 고갈)로 회귀. **공급 라인은 떠받치는 기둥.**
- prep 허브 = `[맵][조립][공급]` TabNav. 결정거리:
  - **(a) 가용성:** 매 전투 사이 상시 vs 특정 방만. 추천: *가벼운 상시 공급 + 스파이크는 게이트*(상자·엘리트 드롭·제작대 특정 층). 공급률 ≥ 소비률로 튜닝.
  - **(b) 공급 메커니즘:** 상점 / 제작대 / 기존 드롭 강화. 스캐빈저+"파츠=연료" 픽션엔 **제작·분해 루프 하나** 추천(상인보다 고물상). **상점·제작 동시 구현 금지(스코프).**

### 기타
- 위협 readout 깊이: 적 구성 노출 여부(카운터 빌드 ↔ 깜짝 요소). 맵 분기별 위협 사전공개와 연동(맵↔조립 결합 루프).
- 인벤토리 칸 수: 압박이 죽었으니 넉넉히 / "가득 차면 유실"만 완화.
- 트레이 위치: 좌측 vs 하단(전투 덱 "내 파츠는 아래"와 통일할지).
- 빈슬롯→코어HP 직격 규칙의 일반화 구현 범위(현재 저격만 검증됨).

## 9. office-hours 관찰 (What I noticed)

- "교체의 긴장이 잘못된 코어인 것 같다"며 *스스로 핵심 재미를 뒤집었다.* 어제 자기 작업(전투)을 소비-축으로 재해석하며 일관성을 찾았다.
- "팔이 노려진다 → 그냥 팔 안 붙이고 가면 되는 거 아니야?"로 위협 설계의 구멍을 정확히 찔렀고, 그 덕에 *파츠=코어의 소모성 장갑* 재프레이밍이 나왔다. 빌려온 규칙을 무비판적으로 안 쓴다.
- "막 버리고 던지려면 파츠 수급이 원활해야"로 throughput 의존성을 *먼저* 짚고 공급/허브 확장을 요청했다. 시스템 간 인과를 본다.
- 아스키가 깨지자 "다른 방법으로 보여달라"며 전달 매체를 바로 교정 — 결과물의 가독성을 챙긴다.
