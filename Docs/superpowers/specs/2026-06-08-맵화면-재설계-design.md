# 맵 선택 화면 재설계 — 설계

> UI/UX 개편의 **네 번째 화면(던전 맵)** 설계.
> 전투(2026-06-01)·조립(2026-06-03)·작업대(2026-06-04) 스펙을 잇는 prep 루프 `[맵][조립][공급]`의 **[맵] 탭**.
> **핵심 재미: "파츠가 주인공"** — 경로 선택 = 보급 전략.
>
> ⚠️ 이 문서는 사용자와 대화로 합의한 **화면 UX + 방 분포 의도(§8)** 까지 확정한다. 생성 규칙 수치·교리 boon 세부는 §9 미결(다음 패스).

---

## 0. 재설계 이유

현행 맵(`DungeonMapScene`)은 **층별 2지선다 picker**다: `ChoiceContainer`(HBox)에 80×80 버튼 2개 → 클릭 시 `RoomInfoBody`에 타입+힌트 → "입장" 버튼. 별도로 떠 있는 "조립" 버튼이 `AssemblyScene`로 이동. 방끼리 연결선이 없고 앞 층이 안 보여, 경로 "계획"이 불가능하다.

새 코어("파츠가 주인공")에서 파츠는 **부위별로 소비·고갈**된다. 경로 선택을 "어디가 안전한가"에서 **"내 빌드에 뭘 보급하나"** 로 바꾼다. 경로 선택 = 보급 전략.

prep 루프 통합: 떠 있던 조립 버튼 → **[조립] 탭**, 작업대(공급)는 **[공급] 탭**(상시), 맵은 **[맵] 탭**.

## 1. 비주얼 규칙 (전투·조립·작업대와 공유)

ItB×DD 규칙 계승.

| 레이어 | 담당 | 느낌 |
|--------|------|------|
| **무대** (어두운 던전 배경·경로 분위기) | 감정·무게 | 어둡고 그을림 (DD) |
| **계기판** (노드·툴팁·보급 readout) | 판단·명료 | 밝고 평면적 (ItB) |

색 = 의미, 의미 있을 때만 (예: 보급 readout에서 0인 부위만 경고색). 하드 베젤 없음, 무드+위치로 분리.

## 3. 화면 구성 + 맵 구조

### 3.1 전체 화면 구성

```
┌──────────────────────────────────────────────────────────────┐
│ 층4  HP 72/100 ████░  쉴드18  ¢128  고철 ⚙3 ✦1        [⚙]  │  ← RunStatusStrip(공용)
├──────────────────────────────────────────────────────────────┤
│  〔 맵 ▶ 〕   〔 조립 〕   〔 공급 〕                          │  ← TabNav
├──────────────────────────────────────────────────────────────┤
│                          [ ☠ 보스 ]                      ┊   │
│                             │                            ┊   │
│                       [ ⚔ 엘리트 ]✦✦                     ┊   │
│                        ╱           ╲                     ┊   │
│                  [ ? 조우 ]      [ ⚔ 일반 ]              ┊   │
│  ┌──────────────┐   │   ╲        ╱   │                  ┊   │
│  │ 보급         │ [⚔]   [ ■ 상자 ]✦  [⚔]                ┊   │
│  │ 인벤 12/16   │   ╲       │       ╱                    ┊   │
│  │ 팔3 등1      │ [ ⚔ 일반 ]  [ ⚔ 해체공 ]★             ┊   │
│  │ 다리0(!) 추1 │      ╲         ╱                        ╉   │
│  │ ⚙3✦1 파손2   │    [ ▲ 현재 위치 ]                     ┊   │
│  └──────────────┘                                        ┊   │
│                     ┌────────────────────────────┐           │
│                     │ ⚔ 해체공 전투 ★            │           │
│                     │ 공급: RARE 파츠 0~2         │           │
│                     │ 클리어 → 〔투척 강화〕 boon │           │
│                     │ ⚠ 강적 — 파츠 소비 큼       │           │
│                     └────────────────────────────┘           │
└──────────────────────────────────────────────────────────────┘
```

**구역:**
- **상단바**(`RunStatusStrip`, 공용·불변) + **`TabNav`**(`[맵]` 활성).
- **중앙 = StS 그래프** — 보스(위) → 현재 위치(`▲`, 아래), 연결선으로만 이동(§3.2).
- **우측 세로 눈금(`┊`)** = 층 진행 깊이감. **숫자/텍스트 없음**, 현재 줄만 강조(`╉`).
- **좌측 구석 = 보급 readout**(§5) — 컴팩트 상시, 맵 안 가림.
- **툴팁** = 호버/포커스한 노드 옆에만(§4.3). 평소엔 없음. 노드 등급 점(`✦`)·교리 노드(`★`)는 §4·§8.3.
- 무대(배경) = DD 어둡게, 노드·눈금·툴팁·readout = ItB 밝고 평면(§1).

### 3.2 맵 구조 규칙 (Slay the Spire식)

- **연결 그래프** — 최대 **4칸** 폭. 갈래가 합쳐졌다 갈라진다. (4칸 상한이 순수 이진트리 폭발을 막아 자연스럽게 "합쳐지는 그래프"가 된다.)
- **엣지 제약** — 노드는 아무 데나 못 가고 **연결된 노드로만** 이동. 하나 고르면 다른 길은 포기 → "이 길의 X vs 저 길의 Y" 가 진짜 결정이 됨.
- **전체 맵 처음부터 공개** → 미리 경로 계획.

```
              [☠]                10층  보스
               │
              [⚔]✦✦             9층   엘리트
             ╱    ╲
          [?]      [⚔]           8층   조우 / 일반
          │ ╲     ╱ │
        [⚔]  [■]✦  [⚔]          7층   (최대 4칸 폭)
          ╲   │   ╱
           [⚔] [⚔]              6층
             ...
        (1층에서 갈라져 올라감, 길마다 다른 방 조합)
```

## 4. 노드 + 정보

### 4.1 노드 표시
- 노드엔 **종류 아이콘 + 예상 보상 등급(점)** 만, 깔끔하게. (GDD가 약속한 "종류 + 예상 보상 등급 미리 공개"를 노드 아이콘으로 충족.)

### 4.2 맵 노드 방 종류
일반 전투 · 엘리트 전투 · 상자 · 조우 · 보스.

> **작업대는 맵 노드가 아니다.** prep 루프의 **[공급] 탭**으로 상시 접근(수리·분해·제작). 따라서 맵에서 작업대로 "이동"하지 않는다. → §8 미결 (GDD 정합).

### 4.3 노드 디테일 = 툴팁
- **옆 패널 없음.** 옆 패널은 항상 자리를 차지해 맵을 가린다. 맵 화면에선 맵이 주인공.
- **호버 + 포커스 툴팁** — 마우스 호버, 그리고 키보드/패드 **포커스** 시에도 같은 툴팁(거의 공짜 작업, 데모 단계부터 챙김).
- 툴팁 내용 (예시 — 정확한 필드는 §8 미결): 방 종류, 예상 공급(등급·개수), 소비 위험, 내 현재 보급 대비 코멘트(예: "다리 ×0 → 여기선 보충 안 됨").

## 5. 전역 보급 readout (맵 구석, 상시·컴팩트)

경로를 짜는 내내 봐야 하므로 툴팁이 아니라 **상시 표시**. 단 맵을 안 가리게 컴팩트하게. 공용 상단바(`RunStatusStrip`)와는 별개 — 경로 계획 전용.

```
[보급]  인벤토리 12/16
  팔 ×3    등 ×1    다리 ×0 (!)    추가팔 ×1     ← 멀쩡한 것만, 0인 부위 경고색
  고철 ⚙3 ✦1   ·   파손 2                       ← 파손은 부위 무관 합산
```

- **부위별 멀쩡한 파츠 갯수** (팔/등/다리/추가팔). 총합(12/16)이 아니라 부위별이라야 "다리가 동났다" 같은 진짜 보급 위기가 보인다. 0인 부위는 경고색.
- **고철** ⚙커먼/✦레어, **파손**(부위 무관 합산), **인벤토리** N/16.

## 6. 보급 전략 프레이밍 (핵심)

새 코어에서 파츠는 부위별로 닳는다. 맵은 그 보급을 계획하는 무대다.

- **경로 = 원자재 income.** 전투-위주 길(COMMON·고철 많이, 소비도 큼) vs 상자·조우-위주 길(확정 고급·도박). 길마다 보급 성격이 다르다.
- **작업대(공급)는 [공급] 탭으로 항상 들고 다니는 가공소.** 경로로 도달하는 목적지가 아니라, 들어온 원자재(파츠·고철)를 수리·분해·제작으로 가공하는 상시 도구.
- 이 화면이 방어하는 것: 실패모드 **⑤′(공급<소비)** · 검증기준 **nV5(공급)** ([[superpowers/specs/2026-05-31-검증기준-온보딩-안할것-design]]).

## 8. 방 분포 / 경로 설계 (2026-06-08 합의 — 의도 수준)

> 생성 규칙의 정확한 수치(층별 노드 수·등장 확률)는 §9 미결. 여기선 "길이 어떻게 달라야 하는가"의 의도를 확정한다.

### 8.1 길 성격 축 = 전투밀도
맵 노드는 5종이고 전투가 다수, 비전투 보급은 상자·조우뿐. 길을 가르는 실질 레버는 **전투 vs 비전투 비율 + 엘리트 밀도**.

| 길 성격 | 구성 | 보급 체감 | 교리 색(경향) |
|---------|------|----------|--------------|
| **강행군** | 전투·엘리트 빽빽 | 파츠 많이 들어오나 소비도 큼(물량전). 고철·파츠 income | 해체공 쪽 |
| **비축** | 상자·조우 위주 | 전투 적어 소비 적음. 확정/도박성 고급 파츠 위주(골라먹기). 크레딧 income | 정비공 쪽 |

→ "굶주렸으니 강행군으로 물량" vs "귀한 거 노리고 비축". **경로 선택 = 보급 전략.** (교리 색은 *경향*일 뿐 강제 아님 — 명시적 교리 특화는 §8.3.)

### 8.2 부위 특화 (확률↑ + 희소)
일부 방/길은 특정 부위 보급에 치우친다. 단 **확정 공급이 아니라 그 부위 드롭 확률↑**, 그런 방·길 자체가 흔하지 않다. **다리는 더 드물게**(다리=귀한 기둥 설정 유지, 전투 spec §1.6과 안 부딪히게).

### 8.3 교리 노드 + 런 boon (하데스식)
- 맵에 **가끔 선택지 하나로** `해체공 전투`/`정비공 전투` 특수 노드가 섞인다(길 전체가 아님).
- 클리어 시 그 교리의 **런-한정 boon** — 이번 런만 유효, 런 끝나면 리셋(하데스 신 boon).
- **영구 교리 성장(재화 투자)과는 별개 레이어.** 시간축이 다름(메타 vs 런). 하데스의 "거울 업글(영구) + 신 boon(런)" 구조 그대로 — 둘은 보완 관계.
- 🔒 구현은 교리 시스템과 함께. **검증 게이트는 nV1~nV7(옛 V6 아님), 설계는 지금 자유.**
- ⏳ **boon 세부**(해체공/정비공별 종류, 런 중 누적·콤보 여부)는 **내일 작업**.

## 9. 미결

- **생성 규칙 수치** — 층별 노드 수, 전투/비전투·엘리트·부위 특화·교리 노드의 등장 확률·빈도. (§8 의도를 수치로.)
- **교리 노드 boon 세부** — 해체공/정비공별 boon 종류, 런 중 누적·콤보 여부 (내일 작업).
- **GDD §2.2·§2.3 정합** — 작업대를 "방 종류"에서 빼고 [공급] 탭 상시로. GDD/`RoomData`/`DungeonManager` 반영(대화 후).
- **전투 spec/NOT DOING의 "V6" → nV 재키** — 옛 코어 게이트 문구 정리(doc-debt).
- **툴팁 정확한 필드**, 노드 아이콘·등급·부위/교리 표식 비주얼.
