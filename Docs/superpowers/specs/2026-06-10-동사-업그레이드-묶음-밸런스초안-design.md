# 동사 8종 — 내용 + 업그레이드 방향 정리 (2026-06-10)

> [[superpowers/specs/2026-06-09-코어시스템-재기획-design]] §2.3 동사 8종 로스터의 **후속 패스**.
> 각 동사의 **내용(기본형)** 과 **업그레이드 방향**을 8종 기준으로 정리한다.
>
> ⚠ 직전(유실) 세션의 "6종 표"(복구·개조·연성·방출…)는 검증 결과 **정본이 아니었다.**
> git 이력·문서 대조로 출처를 못 박고 8종으로 재정리했다 (§0).
>
> 상태 표기: ✅ 확정 · 🔴 미정(결정 필요) · ❌ 미정의(후속 패스)

---

## 0. "6종 표" 정리 결과 (검증 완료)

직전 세션 "6종 표"의 명칭들을 정본 8종(§2.3)·git 이력과 대조한 결과:

| 6종 명칭 | 검증 결과 | 처리 |
|---------|----------|------|
| 복구 | 수리 계열. 수리가 덧대기에 흡수되므로 별도 명칭 불필요 | **삭제** → 덧대기 |
| 개조 | "개조"는 사용자 발언 기록 없음(AI 명명). 실제 의도 = *덧대기 시 재료 파츠 affix 이전* | **삭제** → 덧대기 업그레이드로 복원 |
| 연성 | 현재 문서·06-09 기록에 근거 0 (옛 git 매치는 "유연성" 부분문자열) | **폐기** |
| 방출 | 8종 어디에도 대응 없음 | **폐기** |

**8종 대비 확정 변경:**
- **수리 → 덧대기 흡수.** 전투 중 내구도 복구는 덧대기가 수행. 수리는 독립 동사로 두지 않음.
- **투척 = 투척** (불변, 다른 동사와 합치지 않음).
- **교체 = §2.2 상시 기본기** (긴급 장착과 한 쌍, 동사 슬롯 아님).
- **과부하 · 재가동 · 복제 · 임시방편 = 불변.**

## 1. 동사 8종 — 내용

| # | 동사 | 내용(기본형) | 교리 | 지속성 | 슬롯 |
|---|------|------------|------|--------|------|
| 1 | **교체** | 멀쩡한 장착 파츠 떼고 큐 파츠 장착 | 회전/중립 | 영구 대상 | **상시 기본기**(슬롯 ✕) |
| 2 | **수리** | 내구도 복구 | 살린다 | 영구 | **덧대기에 흡수**(독립 ✕) |
| 3 | **덧대기** | 재료 파츠 소모 → 대상 파츠 보강·수명 연장 **+ (전투 중) 내구도 복구** | 살린다 | 영구 | ○ |
| 4 | **투척** | 파츠를 적에게 던져 피해 + 소비 | 태운다 | 영구(소비) | ○ |
| 5 | **과부하** | 한 파츠 스킬 출력 폭주. 손상 대량 소모 / 사용 후 파괴 | 태운다 | 영구(가속 소모) | ○ |
| 6 | **재가동** | 파괴된 파츠를 내구도 50%로 되살림 | 집착→소비 | **휘발/화장** | ○ |
| 7 | **복제** | 파츠 1개의 **1회용** 사본 생성·장착(사용 후 소멸) | 버스트(태운다) | **휘발** | ○ |
| 8 | **임시방편** | 능력 없이 "때리기"만 가진 임시 파츠 생성·장착(사용 후 소멸) | 즉흥/중립 | **휘발** | ○ |

- **휘발 3종**(재가동·복제·임시방편) 규칙 = 06-09 §2.4: 전투 한정·종료 시 소멸·즉시 장착→빈 슬롯 없으면 큐 최상위.
- **실제 슬롯에 넣는 동사 = 6종**(덧대기·투척·과부하·재가동·복제·임시방편). 교체=기본기, 수리=흡수.

## 2. 동사별 업그레이드 방향

> 패턴: 기본형 → 메타 업그레이드. "더 세짐"이 아니라 **"쓰는 법이 갈라짐"** (06-09 §2.3.1·§5).

### 덧대기 — ✅ 확정
- **기본형:** 재료 파츠를 소모해 대상 파츠 보강·수명 연장 (+전투 중 내구도 복구 = 흡수된 수리 역할).
- **업그레이드:** **덧대기에 사용한 재료 파츠의 affix 중 1개를 0~15% 확률로 결과 파츠에 이전.**
- 🔴 미정: 옛 수리 "연료 확장"(장착 중 / 인벤토리 멀쩡 파츠도 재료로 소모 가능)이 덧대기로 승계되는가?

### 투척 — ✅ 확정 (§2.3.1)
- **기본형:** **파손 파츠만** 던짐.
- **업그레이드 (두 갈래 택1):** ⓐ *장착 중* 파츠도 던질 수 있음 / ⓑ *인벤토리의 멀쩡한* 파츠도 던질 수 있음. (둘 다는 불가 — commit)

### 복제 — ✅ (이전 세션 합의 · 재확인 권장)
- **택1:** 강화 사본(원본보다 강함) / 다중 복제(사본 2개) / 영속 사본(N턴·내구도 소진까지 유지).
- ✂️ 제외: "공명"(사본↔원본 연동) 드롭 확정.

### 임시방편 — ✅ (이전 세션 합의 · 재확인 권장)
- **택1:** 즉석 affix(임시 파츠에 랜덤 affix 1개) / 공격형 임시 파츠(때리기 위력↑ + 처치 시 AP 소량 환급) / 긴급 일괄 충전(빈 슬롯 수만큼 동시 생성).

### 과부하 — ❌ 미정의
- 후속 패스에서 기본형→업그레이드 확정.

### 재가동 — ❌ 미정의
- 후속 패스. (휘발/화장 규칙은 §2.4 확정.)

### 교체 / 수리 — 해당 없음
- 교체 = 상시 기본기. 수리 = 덧대기에 흡수. 동사 업그레이드 슬롯 없음.

## 3. 밸런스 — 보류

- 직전 §3 수치표(복구·개조·연성·방출 기준)는 **폐기 명칭 기반이라 삭제.** 8종 밸런스는 내용·업그레이드 확정 후 별도 패스 + headless/플레이테스트로 잡는다.
- **불변 원칙:** AP + 재료 단일축(06-09 §2.1) · "공짜 금지"(인컴뱃 비용은 아프게) · 휘발 동사는 AP 무겁게 · 업그레이드는 수치 inflation 아닌 **운용 분기**(§5.1 대가 원칙).

## 4. 미결 / Open Questions

- 🔴 **덧대기 "연료 확장" 승계** — 옛 수리 업그레이드(장착/인벤토리 멀쩡 파츠 소모)가 덧대기로 넘어오는가 (§2).
- 🔴 **복제·임시방편 묶음 재확인** — 이전 세션 합의분, 이번 정리 후 재확인.
- ❌ **과부하·재가동 업그레이드** — 미정의, 후속 패스.
- **덧대기 affix 이전 vs 작업대 정합** — affix 조작이 작업대(영구 공급, [[superpowers/specs/2026-06-04-작업대-재설계-design]])와 겹침. "전투 즉흥 ↔ 작업대 영구" 시간축 분리(§2.4)로 차별화.
- **교리 태그 매핑** — 8종 각 교리 점수(태운다/살린다/중립).
- 확정분 → 06-09 §2.3/§2.3.1 병합.
