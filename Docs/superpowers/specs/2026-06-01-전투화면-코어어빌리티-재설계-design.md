# 전투 화면 UI/UX · 코어 어빌리티 재설계 — 설계

> office-hours 브레인스토밍(2026-06-01) 산출물.
> **이 문서는 전체 UI/UX 개편의 "첫 화면(전투)" 설계다.**
> 나머지 화면(거점·격납고·코어설계·던전맵·조립·보상·상자·조우·작업대·결과)을
> **같은 규칙(§2 ItB×DD)으로 전부 설계한 뒤** 화면별 상세 구현에 착수한다.
> 5/10 시각적 조립 설계([[theoe-main-design-20260510-173659]])를 잇는 확장이며 대체가 아니다.

## 0. 배경 / 진단

- **불만(사용자):** "지금은 그냥 인벤토리, 그냥 전투화면, 그냥 스킬 화면 같다. 파츠가 파손되면 그 다음은? 그냥 부서진 채로 있어야 한다."
- **진단:**
  1. 긴급수리·교체·파손활용 기믹이 **이미 명세/코드에 존재**(긴급 교체·파손 투척·임시 덧대기 = `AbilityTreeSystem` §파츠 활용 어빌리티 / 현장 수리·드론 정비 = `CombatSpecification` §3.3)하는데 **화면이 기믹처럼 드러내질 못함.**
  2. 통일 아트 디렉션·룩 규칙 부재의 프로그래머 아트 단계 (화면별 폰트·간격·색 제각각, 과밀 vs 여백 혼재).
- **핵심 재미(불변):** "교체의 긴장" — 기존 파츠를 파손시키면서까지 갈아끼울지의 비용-보상 결정.
- **동결 준수:** NOT DOING #5(V6 재미검증 전 신규 기믹 동결). **설계는 지금, 신규 메커니즘 구현은 V6 통과 후.** 비주얼/재배치 슬라이스는 지금 가능.

## 1. 게임 시스템 — 코어 어빌리티 재정의

### 1.1 핵심 이동
파츠를 조작하는 권한(교체·수리·투척·덧대·기본공격)을 **"잘 부서지는 파츠 층" → "안 부서지는 코어 층"** 으로 끌어올린다.
정체성: **스캐빈저 파일럿(상수) + 주워온 파츠(변수).** 파츠는 버려져도 파일럿의 코어 권한은 사라지지 않는다.

이는 기존 3요소(기본 공격 / 어빌리티 트리 / 파츠 활용 어빌리티)의 **재구성**이다. 새 메타 시스템이 아니라 기존 거점-크레딧 성장(어빌리티 트리)의 reshape이므로 동결의 정신에는 걸리지 않으나, 저장 포맷·CoreSelect·CombatUI·GameState를 건드리는 **규모 큰 개편**이라 구현은 V6 게이트.

### 1.2 구조: 전부 오픈 + 교리 성장
- 코어 동사(교체·수리·투척·덧대·기본공격)는 **택1이 아니라 전부 열려 있다.**
- 게임 내 재화로 **"교리 방향"에 투자**하면 그 교리의 동사 묶음이 깊어진다. (교리는 고정이 아니라 투자 비중)

### 1.3 2교리 (데모 범위)

| 교리(가제) | 정체성 | 핵심 동사 | 파손을 대하는 태도 | 전투 체감 |
|------|------|------|------|------|
| **정비공** | "이 팔, 보스까지 데려간다" | 수리·덧대 | **막는다**(오래 살림) | 같은 파츠로 진득하게, 내구도 관리가 곧 실력 |
| **해체공** | "고철은 적 얼굴에 던진다" | 투척·교체·소모 | **이용한다**(태워서 자원화) | 파츠가 연료, 휘발적·폭발적, 손이 바쁨 |

- 교리를 가르는 축은 **단 하나: "파손을 어떻게 대하는가."** (핵심 재미가 파손과의 관계이므로)
- 같은 동사라도 교리가 비용/보상을 반대로 빚는다. 예: **교체** — 정비공은 *마지못해*(비용 아픔), 해체공은 *신나게*(부순 파츠가 다음 화력).
- **3교리 "코어 순수"(자립형)는 보류** — "파츠 중심" 정체성과 충돌 소지. 좋은 아이디어 나오면 부활.

### 1.4 가드레일 (양보 불가)
**두 축을 절대 섞지 않는다.**
- **메타 성장 축**(전부 오픈 + 재화 업그레이드) → 자유.
- **인-컴뱃 사용 비용 축**(전투 중 그 동사를 쓸 때의 값) → **반드시 아프게 유지**(AP·전투당 1회·파손 파츠 소모·HP 등).

> "모두 오픈"이 "공짜"가 되는 순간, 파츠를 잃는 비용이 사라져 **교체의 긴장(V1 수락률 밴드)**이 붕괴한다.
> 업그레이드는 "더 세짐"이 아니라 **"트레이드오프가 깊어짐"** 방향이어야 한다.

### 1.5 부위별 동사 인격 (= "파츠마다 다른 기믹")

| 부위 | 정체성 | 핵심 동사 | 세계관 논리 |
|---|---|---|---|
| **팔** (오R·오L) | 만능 일꾼 · 소모품 | 교체·투척·방어 **다 함**(네로식) | 흔함 → 막 굴린다 |
| **등** | 쓰고 버리는 지원 | 투척, 또는 **자신을** 일회용 방패/버스트로 소모 | 보조 장비 → 한 방 쓰고 끝 |
| **다리** | 지켜야 할 기둥(희귀) | 자신은 잘 안 버림. **다른 파츠(팔)를 덧대 수명 연장**(정비공 +α) / 미끼로 피격 유도 | 귀해서 아껴 쓴다 |

**겹침 방지:** 등과 다리 둘 다 "방패"처럼 보이나 방향이 반대다.
- **등 = 자신을 희생한다**(등을 태워 방패가 됨, 소모되는 쪽).
- **다리 = 남을 희생해 지킨다**(팔을 태워 다리를 보존/미끼, 지켜지는 쪽).
"팔을 태워 다리를 지킨다"가 이 게임만의 내부 환율이 된다.

### 1.6 다리 경제 (확정)
- **드롭률만 낮춤.** LEG 10종은 유지하되 런당 드물게 드롭 → 희귀 체감 + 런마다 다른 다리(리플레이성↑).
- **다리도 교체는 가능.** 단 드물어서 *아껴 쓰는* 내러티브. "못 바꾸는" 게 아니므로 방어 죽음의 나선 없음.
- 보존(팔 덧대 수명 연장)은 *수명 연장*이지 영구는 아님. 정비공이면 +α.

## 2. 비주얼 디자인 규칙 — Into the Breach × Darkest Dungeon

### 2.1 한 줄 비주얼 명제
> **"고철 더미의 어둠 속, 정밀한 조종석 계기판."**
> 무대는 무겁고 어둡게(DD), 계기판은 밝고 정확하게(ItB).

### 2.2 핵심 규칙 = 레이어 분리
둘을 픽셀 단위로 섞으면 진흙이 된다. **역할(레이어)로 가른다.**

| 레이어 | 담당 | 느낌 | 출처 |
|--------|------|------|------|
| **무대** (메카·적·파츠·배경) | 감정·무게 | 어둡고 무겁고 그을림, 강한 실루엣, 극적 명암 | Darkest Dungeon |
| **계기판** (정보·수치·텔레그래프·버튼) | 판단·명료 | 밝고 평면적, 정확, 군더더기 없음, 다 예고됨 | Into the Breach |

> 그을음은 절대 계기판의 가독성을 침범하지 않고, 계기판은 절대 무대의 압박감을 누그러뜨리지 않는다. **분리가 곧 규칙.**
> **하드 베젤(분리선)은 두지 않는다.** 무드 + 위치로만 분리한다(Slay the Spire식 장면 — 배경 가득, 부드럽게 덱으로 이어짐).

### 2.3 철학을 그림으로
- "유저에게 생각할 시간을 주어서는 안된다" = **무대의 압박·무게(DD)**
- "매 순간 최선의 생각을 해야한다" = **계기판의 완벽한 가독성(ItB)**

룩에 *이유*가 생긴다. 검증기준 **V3/V4(한 화면 정보로 교체 판단)**와 직결.

### 2.4 디자인 규칙 스타터 (전 화면 공통 — 추후 정식 디자인 시스템 문서로 확장)
- **팔레트:** 근-검정 베이스 + 녹/갈색 미드(무대·파츠) + 경고 앰버/적색 액센트(계기판, 의미 있을 때만). 색 = 의미.
- **대비/조명:** 키/림 라이트로 전투원에 집중 조명, 비네팅, 강한 실루엣(파츠가 실루엣으로 읽혀야 함 — 시각적 조립과 연속).
- **텍스처:** 무대=텍스처/그을림, 계기판=평면/클린. 절대 혼용 금지.
- **타이포:** 계기판 폰트 1종(고가독), 헤더는 무게 있는 디스플레이체 / 데이터는 클린 가독체. (어제 진단 "화면별 폰트 제각각" 해소)
- **밀도:** 계기판=ItB 여백(차분), 무대=DD 밀도. (Hangar/Workshop "과밀", Combat "여백 과다"가 *레이어 역할 오인*에서 왔음 → 역할별 밀도 부여로 동시 해결)

## 3. 전투 화면 레이아웃

### 3.1 3구역 구성
```
──────────────────────────────────────────────────────────  상단 계기판(ItB)
 층4   HP 72/100 ▰▰▰▰▰▱▱   쉴드 18/20 ▰▱   크레딧 128       ⚙
──────────────────────────────────────────────────────────

      Slay the Spire식 전투 장면 — 배경 가득, DD 무드
                       「턴 1」
                                    ⚔예고      ⚔예고  ← ItB가 무대에 닿는 유일 지점
       ▟███▙                      ┌────┐    ┌────┐
      ██ 내 ██   ← 메카를 크게      │적1 │    │적2 │
       ▜███▛       배경 안에 서서   └────┘    └────┘
   ≈≈≈≈≈ 그을린 바닥·배경 (베젤 없이 부드럽게 덱으로) ≈≈≈≈≈
──────────────────────────────────────────────────────────  컨트롤 덱(ItB)
        (하단 3분할 ↓)
──────────────────────────────────────────────────────────
```

### 3.2 하단 컨트롤 덱 — 3분할(중앙 2분할)

| 좌 — 손상도 요약 | 중-좌 — 기본 + 파츠 스킬 | 중-우 — 코어 어빌리티 ★ | 우 — 예비 파츠(스왑) |
|---|---|---|---|
| 등 ▰▰▰ 정상 | 〔기본공격〕 ⚔ | 〔정비공〕 *특수 프레임* | **DMC5 네로 브레이커식** |
| 오R ▰▱▱ 손상 | 오R ⚔ · 오R ⛨ | 교체 · 수리 | ▸ [고출력팔] 호환 |
| 왼 ▰▰▰ 정상 | 왼 ⚔ · 등 + | 투척 · 덧대 | ▸ [경량팔] 호환 |
| 다 ▰▰▱ 손상 | *(기본/파츠 시각 구분)* | *(글로우·특수감)* | ▸ [방패등] (등 전용) |

- **읽기 순서가 서사:** 좌(내 상태) → 중(내 행동) → 우(내 예비 자원). "지금 어떤가 → 뭘 할까 → 뭐가 남았나."
- **손상 표시 분업(기존 §7 규칙 의도적 반전):** 영구 tint는 안 올림(메카는 StS처럼 크고 깨끗). 대신 **무대는 손상 '사건' 연출**(피격 번쩍·흔들림·팔 스파크 = DD 무게), **정확한 손상도 '상태'는 좌측 요약**(ItB 가독). 사건은 무대, 수치는 덱.
- **3층 중요도(코어 어빌리티 ★★★ > 파츠 스킬 ★★ > 기본 ★)** 는 크기보다 **틀·반응**으로 표현.
- **기본 vs 파츠 스킬 구분:** 기본공격 = 상시·무손상 슬롯(테두리 다름), 파츠 스킬 = 부위 라벨 + 내구도 연동(부서지면 회색).

### 3.3 코어 어빌리티 "특수 프레임" (중-우)
디제틱: 파츠 스킬 = 무기 트리거 / **코어 어빌리티 = 파일럿의 지휘 콘솔.**

| 항목 | 파츠 스킬(중-좌) | 코어 어빌리티(중-우) |
|---|---|---|
| 틀 | 평범한 격자 셀 | **함입 콘솔, 베벨 + 내부 글로우** |
| 색 | 차갑고 중립(작업용) | **따뜻한 앰버/골드(권위·특수)** |
| 정체성 | 부위 라벨만 | **교리 문장**(정비공=렌치/모루, 해체공=고철/투척) |
| 반응 | 부서지면 회색 | **파손 시 모듈 전체 맥동** |
| 비용 표기 | AP 배지 | **AP + 트레이드**("AP2·기존팔 상실") → 가드레일을 눈에 보이게 |

### 3.4 파손 반응 루프 (비-모달) — 이 게임의 시그니처
**비트 0 — 위험 신호(파손 전):** 적 저격 예고 시 좌측 손상도 요약 + 무대 그 팔이 경고 앰버. (기존 저격 예고 시스템 활용)
**비트 1 — 파손(DD 무게):** 0.4초 순간 암전 + 메카 흔들림 + 무거운 타격음. 그 팔 균열·스파크·붉은 ✖. 중-좌 해당 파츠 스킬 그룹 회색 사망, 좌측 요약 [파괴] 빨강.
**비트 2 — "그 다음은?"(ItB 명료):** 중-우 코어 어빌리티 맥동 + 「오른팔 파손 — 대응」. **모달 아님·멈춤 없음** — 무시하고 계속 싸워도 됨. 패널은 *주의만 재배치*. (철학: 멈춤 없이, 선택지는 즉시 읽힘)
**비트 3 — 대응(부위별 세트):**
```
팔 파손  → [교체][투척][덧대]  + 스왑 큐(네로식)
등 소모  → [투척][일회용 방패]   (스왑 큐 약함)
다리 위협 → [팔 덧대 보존][미끼]  (스왑 아님 — 보존이 동사)
```
교체 선택 시 우측 스왑 스택이 타겟 피커가 되고, 호버하면 **미리보기 한 줄**: `공격 12→19 │ 비용: AP2·오른팔 영구 상실 (정비공이라 +1)`. → V3/V4 직격.
**비트 4 — 마무리:** 갱신 후 루프 지속. 미대응 시 한 손으로 계속 싸우되 손상도 요약에 파괴 상태가 남아 "아직 대응 가능"을 상기.

### 3.5 스왑 스택 = B(맥락 집중, 팔 중심)
- 슬롯이 N개(부위별 호환 풀 분리)라 네로식 단일 큐를 *그대로* 못 씀. → **맥락 집중**으로 해결.
- **평상시:** 자주 쓰는 *팔 예비*만 네로식으로 크게, 등·다리는 작은 점.
- **파손/교체 순간:** 그 슬롯만 단일 큐로 **집중**(부위별 행동 세트와 연동).
- 큐 운용은 기본 **자유 선택(가)**, 교리(해체공)일 때 큐가 빨리 도는 **하이브리드(다)** 를 V6 후 얹는다. (네로식 고정 큐(나)는 새 메커니즘이라 보류)

## 6. 고려한 접근 (Approaches Considered)

- **재설계 범위:** A(화면·연출 우선) / B(기믹+화면 동시) / C(디자인 규칙 토대 먼저) → **B 채택**(설계는 동결 무관, 구현만 게이트).
- **코어 어빌리티 보장 범위:** 가(택1 유지+UI통합) / 나(기본 보장+확장) / 다(전면 재정의) → **다 채택**(전부 오픈+재화 성장).
- **교리 구조:** 3축(공/방/유틸) / 행동별 트랙 / 교리 분기 → **교리(C)가 정체성 wrapper, 동사 트랙(B)이 메커니즘**으로 통합. 2교리로.
- **비주얼 톤:** 고철 인더스트리얼 / 하드SF 군용 HUD / 네온 / 레트로 → **ItB×DD 결합**(고철 무대 + 군용 HUD).
- **스왑 조직 원리:** A(부위별 동시 진열) / B(맥락 집중) / E(슬롯 직접) → **B 채택**.

## 9. office-hours 관찰 (What I noticed)

- "그냥 전투화면 같다"의 원인을 *기능 부재*가 아니라 *전달 부재*로 다시 보게 했을 때, 바로 "있는 걸 화면으로 드러내자"로 방향을 틀었다. 없는 걸 더 만들려는 충동을 안 따랐다.
- "코어 순수 교리"처럼 *이름이 더 좋다*고 즉시 집어내고, "아직 좋은 아이디어 없다"며 3교리를 미룬 판단 — 억지로 채우지 않는 절제.
- 네로 브레이커를 끌어왔을 때 *스스로* "근데 여긴 부위별로 나뉘잖아"라며 메타포의 구멍을 먼저 찾았다. 빌려온 참조를 무비판적으로 안 쓴다.
- 내 모호한 라벨("소모형/개조형")의 겹침을 정확히 지적했다. 정체성과 메커니즘을 섞으면 뿌예진다는 걸 체감으로 알고 있다.
