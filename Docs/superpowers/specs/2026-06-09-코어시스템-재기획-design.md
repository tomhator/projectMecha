# 코어 시스템 · 코어 어빌리티 · 교리 재기획 — 설계

> UI/UX 개편의 토대가 되는 **게임 시스템 재기획**. 화면 설계가 아니라 *그 화면들이 담을 시스템*을 확정한다.
> 핵심 재미 **"파츠가 주인공"** 위에서 코어 성장 · 코어 어빌리티 · 교리 · 메타 성장의 척추를 정한다.
> 2026-06-01 전투 화면 스펙의 "코어 어빌리티 재정의"(§1)를 **대체·확정**한다. (그 문서의 레이아웃·비주얼 규칙 §2·§3은 유효.)
>
> 모든 결정은 사용자와의 대화(2026-06-09)로 합의됨. 이 문서는 **코어 기둥**을 확정하며, **파츠 기둥**(덱 구조·affix 재설계)은 후속.

---

## 0. 재기획 이유

- boon 설계가 막힌 근본 원인 = **성장 축 / 코어 / 파츠 모델이 미확정**이었음.
- 던전 이벤트(조우·상자·boon)는 전부 *보상을 분배하는 장치*다. 무엇을 주느냐가 "코어가 어떻게 크고 파츠가 어떻게 도느냐"에 100% 의존 → 토대 없이 이벤트부터 설계하면 모래 위에 짓는 것.
- 따라서 순서: **코어·파츠 먼저 → 그 위에 던전 이벤트.** 이 문서는 그중 코어 기둥을 확정한다.

---

## 1. 코어(본체)

- **단일 주인공.** 스토리상 주인공 = 코어/파일럿(상수). **절대 교체·해금 없음.** 데모~정식 모두 1개 고정.
- **런 중 고정.** HP·하중·AP·슬롯 등 본체 스탯은 런 동안 변하지 않는다. **런 내 파워 변동은 파츠가 담당.**
- **성장 = 메타.** 코어를 바꾸는 게 아니라, *단일 코어의 노드 트리 / 예산 / 슬롯이 게임 전체 진행도로 깊어진다.* (하데스 자그레우스 결.)
- **이중 주인공 구조:** 스토리 주인공 = 코어(상수 파일럿) / 게임플레이 주인공 = 파츠(변수). **파일럿은 무대, 파츠는 배우.** "파츠가 주인공"과 충돌하지 않는다.

---

## 2. 코어 어빌리티

### 2.1 정의

코어 어빌리티 = 파일럿의 **파츠 조작 동사 그 자체.** (옛 3요소 "기본공격 / 어빌리티 트리 / 파츠 활용"의 재구성 — 2026-06-01 §1.1 계승.)

- 출격 전 **3슬롯 로드아웃**으로 동사를 세팅 = 빌드 / 교리 방향. **런 중 고정.**
- 동사는 교리에 **종속되지 않는 자유 부품.** 조합으로 원하는 교리 쪽으로 기운다.
- 표시 위치 = 전투 화면 **중-우 콘솔**(2026-06-01 §3.2·§3.3). 상시 3동사 표시.
- **메타 진행:** 동사 로스터 확장 + 슬롯 **1→2→3**, 4번째는 후반 특별 해금.

### 2.2 상시 기본기 (콘솔 밖 · 슬롯 무관 · 항상 사용)

| 기본기 | 내용 |
|--------|------|
| **기본공격** | 출격 전 **노드로 1택.** 각 기본공격은 **교리를 가짐** → 키스톤 타이브레이커(§4). 전투 중 상시 사용. |
| **긴급 장착** | 파손돼 *떨어져 나간 빈 슬롯*에 큐의 파츠를 장착(손실 복구). |

> **교체 ↔ 긴급 장착 구분:** 교체 = *멀쩡한* 파츠를 능동적으로 떼고 큐 파츠 장착(전략). 긴급 장착 = *파손으로 빈* 슬롯 채우기(복구). 긴급 장착이 상시 기본기라, 교체를 슬롯에 안 넣어도 손실 복구는 항상 가능 → "교체 없는 빌드"가 답답하지 않다.

### 2.3 동사 로스터 (데모 8종 → 3택)

| 동사 | 효과 개요 | 교리 친화 | 지속성 |
|------|----------|----------|--------|
| **교체** | 멀쩡한 장착 파츠를 떼고 큐 파츠 장착 | 회전/중립 | 영구 대상 |
| **수리** | 파츠 내구도 복구 | 살린다 | 영구 |
| **투척** | 파츠를 적에게 던져 피해 + 소비 | 태운다 | 영구(소비) |
| **덧대** | 파츠 보강·수명 연장 | 살린다 | 영구 |
| **과부하** | 한 파츠 스킬 출력 폭주. 손상도 대량 소모 / 사용 후 파괴 | 태운다 | 영구(가속 소모) |
| **재가동** | 파괴된 파츠를 내구도 50%로 되살림 | 집착→소비 | **휘발/화장**(§2.4) |
| **복제** | 파츠 1개 사본 생성. 사본은 **1회용** | 버스트(태운다) | **휘발** |
| **임시방편** | 능력 없이 "때리기" 공격만 가진 임시 파츠 생성·장착 | 즉흥/중립 | **휘발** |

- **임시방편 레퍼런스:** 제이슨 본이 수건·잡지를 말아 싸우는 즉흥 무장. "무에서 연료/몸뚱이를 만든다."
- **백로그(추후 추가):** 분해(파츠 → 즉석 자원), 강탈(적 파츠 탈취) 등. 더 추가 예정.
- **로스터 불균형 메모:** 새 코어는 *태운다(해체공)* 쪽에 기본 무게. 추가 4종으로 태운다 계열(투척·과부하·복제·임시방편)이 채워짐.

**자연 발생 콤보 (= "원하는 교리로 세팅"의 증거):**
- 복제 → 과부하(사본) → 투척: 원본 안전, 사본으로 폭딜.
- 임시방편 → 투척/과부하: 쓰레기 찍어내 태우는 *연료 엔진*.
- 과부하(파괴) → 재가동(50%) → 마지막 한 방 → 재: *끝까지 우려먹기*.

### 2.4 휘발 동사 규칙 (재가동 · 복제 · 임시방편)

- **전투 한정.** 전투 종료 시 산출물 소멸.
- **작업대(공급탭) = 영구 공급 ↔ 휘발 동사 = 전투 중 즉흥 공급.** 같은 환상(만들고 고친다)을 **시간축**으로 분리 — 계획적 정비(작업대) vs 급박한 임기응변(전투 동사).
- **재가동 = 화장:** 50%로 무리하게 부팅 → 내구도 소진 시 **완전 소멸**(파손 더미로도 복귀 없음). 전투 끝까지 살아남아도 휘발 → "쓰거나 잃거나", 그 전투 안에 다 짜내야 한다.
- **남용 차단:** 명품 파츠 영구 복제 불가, 임시방편으로 인벤토리 불리기 불가. 전부 그 전투용.

## 3. 교리

- **정체:** 상자(클래스)가 아니라 **축/지향점.** 플레이어가 *겨냥하는 방향*. 확장형 — 현재 **태운다 / 살린다**, 추후 추가.
- **형식 추적 (가):** 게임이 방향을 인식·보상한다 → 유저가 플레이 방향을 *설정* 가능.
- **backbone = 태그:** 빌드 재료(동사 · affix · 교리 패시브 · boon)마다 0~1개 교리 태그. 게임이 활성 태그 분포(=정렬)를 읽는다.
- **(C) 두 층:**
  - **출격 전(노드):** 점수 합산 → 키스톤 확정(런 중 고정). = 선언형.
  - **런 중(파츠·affix·boon 태그):** 키스톤 방향에 맞춰 모으면 soft 시너지. = 누적형. 키스톤이 "이 방향 재료를 모아라"는 **나침반**.
- **"안 가둔다" 원칙:** 동사·노드는 자유 조합, 교리는 *결과로* 결정. 수리 노드 낀 해체공도 가능(점수만 분산).
- **affix 재설계 방향:** affix를 *교리 태그 붙은 시너지 조각*으로 전면 재설계. (상세는 파츠 기둥.)

## 4. 키스톤 결정

코어 설계 화면 = **노드 빌드.** 노드마다 교리 점수(태운다 +N / 살린다 +N / 중립 0).

**결정 규칙 (최종)**

| 순서 | 규칙 |
|------|------|
| 1 | 노드 교리 점수 **합산** *(기본공격 제외)* |
| 2 | **동점이면** → 기본공격 교리 점수 발동·가산 → 타이 해소 |
| 3 | 선두 교리가 **문턱(최소 점수) ≥** → 그 교리 **강한 키스톤** 발동 / **문턱 미달** → 키스톤 없음(제너럴리스트) |

- **기본공격 = 순수 타이브레이커.** 점수는 평소 잠자고, *동점일 때만* 발동해 자기 교리에 가산. (합산 = 노드로 쌓은 방향 / 기본공격 = 타고난 바닥, 역할 분리.)
- **키스톤 = 빌드 일관성으로 "버는" 것.** 해체공 키스톤을 원하면 해체공 노드를 충분히 모아야 함 → commit이 곧 보상.
- **문턱 미달 = 트레이드오프.** 어정쩡하면 유연하지만 키스톤 파워 없음. "확 commit → 강한 키스톤 / 어정쩡 → 유연하지만 밋밋"이 곧 빌드 결정.
- **UX:** 노드 조정 시 교리 점수 바 실시간 + "현재 키스톤: ___" 표시. → nV4(한 화면 판단).

## 5. 메타 성장

- **성격:** 스탯 뻥튀기 ❌ → **규칙을 비트는 키스톤 + 트레이드오프** ⭕. (2026-06-01 §1.4 가드레일 정합 — "더 세짐 ❌ / 트레이드오프 깊어짐 ⭕".)
- **예:** "재가동 파츠 내구 소진돼도 소멸 안 함 (대신 ___ 포기)". 기본 규칙은 빡세게(재가동=화장), 메타로 *특정 규칙만* 풀되 *다른 걸 내준다.* (PoE 키스톤 결.)
- **코어(본체) 업그레이드 + 교리 패시브** = 둘 다 이 "키스톤" 형식으로 통일.
- 해금 = 거점 재화 투자 / 전체 진행도 (영구). 노드 트리·슬롯·키스톤이 여기서 깊어진다.

## 6. 코어 설계 화면 (노드 빌드) — 개요

- 단일 코어의 **노드 트리.** 노드 = 동사 / 기본공격(택) / 코어 업그레이드 / 교리 패시브. 각 노드 교리 점수 보유.
- 출격 전 여기서 확정: **3 동사 슬롯 + 기본공격 택 + 키스톤(노드 점수 결과).**
- 상세 레이아웃은 **화면 설계 패스**에서 (기존 [[AbilityTreeSystem]] / 코어 설계 화면 재설계와 통합).
