# 작업대 화면 재설계 — 설계

> UI/UX 개편의 **세 번째 화면(작업대)** 설계.
> 전투(2026-06-01) · 조립(2026-06-03) 스펙을 잇는 확장.
> **핵심 재미: "파츠가 주인공"** — 적극 사용·버리기·교체·활용.

---

## 0. 재설계 이유

새 코어("파츠가 주인공")에서 전투 중 투척·희생이 일상적 동사가 된다.
파츠 소비 속도가 빨라지면 전투 보상 드롭만으로는 인벤토리 throughput이 부족하다.
조립 화면 스펙(§7)이 지목한 "공급 라인은 떠받치는 기둥" 문제를 작업대가 정면으로 해결한다.

**작업대의 새 역할: 파츠 공급 허브 (던전 중 공급 탭).**

두 교리를 동시에 떠받친다:
- **정비공** — 수리(파츠를 살린다).
- **해체공** — 분해+제작(파츠를 태우고 다시 만든다).

---

## 1. 비주얼 규칙 (전투·조립과 공유)

전투 화면 스펙 §2 **ItB×DD 규칙** 그대로 계승.

| 레이어 | 담당 | 느낌 |
|--------|------|------|
| **무대** (배경·작업장 기계·고물 더미) | 감정·무게 | 어둡고 그을림, 폐공장 분위기 (DD) |
| **계기판** (슬롯·버튼·수치·재고 표시) | 판단·명료 | 밝고 평면적, 군더더기 없음 (ItB) |

하드 베젤 없음. 위치+무드로만 분리.

---

## 2. 공통 컴포넌트

조립 화면 스펙 §2에서 확정한 전 화면 공통 컴포넌트를 그대로 사용한다.

| 컴포넌트 | 위치 | 내용 |
|---------|------|------|
| `RunStatusStrip` | 최상단 | 층 · 코어 HP · 쉴드 · 크레딧 · 설정 |
| `TabNav` | RunStatusStrip 바로 아래 | [맵] [조립] [공급 ▶] — 작업대는 **공급 탭** |

## 3. 화면 레이아웃

### 3.1 전체 구조

```
┌─────────────────────────────────────────────────────────────┐
│  RunStatusStrip  층4  HP 72/100 ████░  쉴드 18  ¢128  [⚙]  │
├─────────────────────────────────────────────────────────────┤
│  TabNav  [ 맵 ]  [ 조립 ]  [ 공급 ▶ ]                       │
├────────┬────────────────────────────────┬───────────────────┤
│  좌    │          중앙 (주 작업 영역)      │       우           │
│  기능  │                                │                   │
│  탭    │  ══ 수리 ═══════════════════  │  인벤토리           │
│        │                                │  ┌──┐┌──┐┌──┐   │
│  🔧   │  ┌──────────────────────┐    │  │  ││  ││  │   │
│  수리  │  │                      │    │  └──┘└──┘└──┘   │
│        │  │   파츠 슬롯 (1)       │    │  ┌──┐┌──┐┌──┐   │
│  🔨   │  │                      │    │  │  ││  ││  │   │
│  분해  │  └──────────────────────┘    │  └──┘└──┘└──┘   │
│        │                                │  ...              │
│  ⚗️   │  [수리하기]        ¢ 30        │                   │
│  제작  │  [전체 수리]       ¢ 128       │  ────────────     │
│        │                                │  [⛏ 고철더미]    │
│        │                                │   잔여 5 / 5회    │
└────────┴────────────────────────────────┴───────────────────┘
```

### 3.2 좌 — 기능 탭 (세로 아이콘)

3개 기능을 세로로 나열. 텍스트 라벨은 아이콘 아래 소자로.
선택된 탭은 액센트 앰버/밝은 색으로 하이라이트 (계기판 ItB 규칙).

| 탭 | 아이콘 | 기능 |
|----|--------|------|
| 수리 | 🔧 | 파츠 내구도 복구 |
| 분해 | 🔨 | 파츠 → 등급별 스크랩 |
| 제작 | ⚗️ | 스크랩 → 새 파츠 |

### 3.3 우 — 인벤토리 + 고철더미

**인벤토리:** 부위별 파츠 카드 그리드. 조립 화면의 파츠 트레이와 동일한 카드 컴포넌트.
파츠를 중앙 슬롯으로 드래그해 수리·분해 작업에 올린다.

**소재 재고 표시 (인벤토리 상단):**
```
⚙ 커먼 스크랩  N개   ✦ 레어 스크랩  N개
```
항상 노출. 제작 탭에서 비용 판단에 사용.

**고철더미 버튼 (인벤토리 하단):**
```
[⛏ 고철더미]
잔여 N / 5회
```
- 층당 5회 한정. 클릭 1회 = 랜덤 파츠 1개 드로우.
- 드로우 결과는 중앙 영역에 잠깐 표시 후 인벤토리에 추가.
- 0회 남으면 버튼 비활성화 + "이번 층 소진" 표시.

## 4. 탭별 중앙 작업 영역

### 4.1 수리 탭

디아블로 4 수리 화면 참고. 슬롯 1개 + 버튼 2개 구성.

```
══ 수리 ═════════════════════════════

┌──────────────────────────────────┐
│                                  │
│        파츠 슬롯 (1)              │
│    [인벤토리에서 파츠를 올려둬]    │
│                                  │
└──────────────────────────────────┘

  손상도: ██░░░  2 / 5

  [수리하기]          ¢ 30
  [전체 수리]         ¢ 128
```

- **수리하기:** 슬롯에 올린 파츠 1개만 수리. 비용은 파츠 손상 정도 기반.
- **전체 수리:** 현재 장착 중인 전 파츠 일괄 수리. 합산 비용 표시.
- 비용은 실시간 계산 — 파츠를 슬롯에 올리는 순간 갱신.
- 파츠가 없으면 두 버튼 모두 비활성화.

### 4.2 분해 탭

수리 탭과 동일한 1-슬롯 레이아웃. 결과 예측 표시가 추가됨.

```
══ 분해 ═════════════════════════════

┌──────────────────────────────────┐
│                                  │
│        파츠 슬롯 (1)              │
│    [인벤토리에서 파츠를 올려둬]    │
│                                  │
└──────────────────────────────────┘

  예상 획득:
    ⚙ 커먼 스크랩  × 2
    ✦ 레어 스크랩  × 1

  [분해하기]
```

- 예상 획득 스크랩은 파츠 등급 기반 고정 산출. 슬롯에 올리면 즉시 표시.
- COMMON 파츠 → 커먼 스크랩만. RARE → 커먼+레어 혼합. EPIC → 레어 중심.
- 분해는 파괴. 인벤토리에서 파츠가 사라지고 스크랩이 재고에 더해짐.

### 4.3 제작 탭

부위 선택 → Affix 개수 조절 → 비용 확인 → 제작.

```
══ 제작 ═════════════════════════════

부위 선택:
  [ARM_L]  [ARM_R]  [BACK]  [LEG]   ← 라디오 아이콘 (택 1)

Affix 개수:
  [ ‹ ]  3  [ › ]  →  RARE
  ※ 0-1 = COMMON   2-3 = RARE   4-5 = EPIC
  코어 티어 최대: 4

소모 비용:
  ⚙ 커먼 스크랩  × 2
  ✦ 레어 스크랩  × 1

  [제작하기]
```

- **부위 선택:** ARM_L / ARM_R / BACK / LEG 4종. 라디오 아이콘, 택 1.
- **Affix 개수:** 스텝퍼 인풋, 범위 0~5. 옆에 등급 레이블 실시간 갱신.
  - 코어 어빌리티 티어 이상의 개수는 잠금 표시 (코어 설계 화면 재설계 시 연결).
- **제작 결과:** 선택 부위의 파츠 중 랜덤 1종. Affix 종류는 파츠별 affix_pool에서 랜덤 선택.
- 재료 부족 시 `[제작하기]` 비활성화 + 부족한 스크랩 강조 표시.

## 5. 소재(스크랩) 체계

| 등급 | 이름 | 출처 | 색상 |
|------|------|------|------|
| 커먼 스크랩 ⚙ | COMMON급 분해 산출 + 고철더미 하위 드롭 | 회색 |
| 레어 스크랩 ✦ | RARE/EPIC 분해 산출 + 고철더미 상위 드롭 | 파랑 |

스크랩은 런 내 자원. 거점 이월 여부는 추후 결정.

---

## 6. 고철더미 상세

- **위치:** 우측 인벤토리 하단 고정 버튼.
- **횟수:** 층당 5회. 방문 횟수 아닌 층(floor) 기준으로 초기화.
- **1회 결과:** 랜덤 파츠 1개. 등급은 현재 층 드롭 테이블 하위 등급 기반.
- **결과 연출:** 클릭 → 중앙 영역에 드로우된 파츠 카드 1초간 표시 → 인벤토리 추가.
- **소진 시:** 버튼 비활성화. "이번 층 소진" 레이블.

## 8. 다른 화면과의 연결

| 화면 | 연결 지점 |
|------|----------|
| **조립 화면** | 분해 기능 작업대로 이전. 조립은 순수 교체만. |
| **전투 화면** | 투척·희생으로 소비된 파츠 → 작업대에서 분해·제작으로 재생산. |
| **코어 설계 화면** | 코어 어빌리티 티어 → 제작 탭 Affix 최대 개수 제한에 반영. |
