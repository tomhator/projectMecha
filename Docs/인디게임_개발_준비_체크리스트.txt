# 인디게임 생존키트 - 개발 준비 상태 점검 체크리스트

이 체크리스트는 내 프로젝트가 실제 개발에 착수해도 되는 수준인지 판단하기 위한 도구입니다.
각 항목을 점검하고, 부족한 부분은 보완한 후 개발을 시작하세요.

---

## 1단계: 마인드셋 점검

### 시작 전 마음가짐
- [ ] 완성할 수 있다는 믿음이 있는가?
- [ ] 게임이 완성된 모습(완주의 이미지)을 선명하게 그릴 수 있는가?
- [ ] "까짓거 해보지 뭐"라는 마음으로 시작할 준비가 되었는가?

### 버려야 할 것들
- [ ] 조바심을 내려놓았는가?
- [ ] 남들과 비교하는 마음을 버렸는가?
- [ ] "대박 아니면 의미 없다"는 생각을 버렸는가?

---

## 2단계: 기초 기획 점검

### 장르/테마/규모 (3-1)
- [ ] 장르가 명확하게 정해졌는가? (액션, 퍼즐, 시뮬레이션, RPG 등)
- [ ] 테마가 정해졌는가? (SF, 판타지, 현대, 호러 등)
- [ ] 테마가 내 역량으로 구현 가능한 범위인가?
- [ ] 개발 기간과 예산을 기준으로 적정 규모를 계산했는가?
- [ ] 목표 플레이타임이 정해졌는가?

### 핵심 경험 정의 (3-4)
- [ ] **핵심 경험을 한 문장으로 정의할 수 있는가?**
  - 예: "플레이어가 _____ 상황에서 _____ 를 통해 _____ 을 느끼게 한다"
- [ ] 이 핵심 경험이 무너지는 상황 3가지를 상상해보았는가?
- [ ] 각 상황을 방지하기 위한 유도 장치를 설계했는가?

---

## 3단계: 코어루프 점검 (3-2)

### 코어루프 3요소
- [ ] **행동**: 플레이어가 가장 자주 하게 될 조작이 정의되어 있는가?
- [ ] **행동**: 그 행동 자체가 재미있는가? (타격감, 피드백 등)
- [ ] **성장**: 성장 요소가 3가지 이상 기획되어 있는가?
- [ ] **고난**: 고난 요소가 3가지 이상 기획되어 있는가?
- [ ] 행동-성장-고난의 순환 구조가 명확한가?

### 밸런싱 기준
- [ ] 새로운 기믹 : 합성 기믹 : 익숙한 기믹의 비율이 대략 1:2:7 인가?
- [ ] 자원/스킬의 "환율" 개념이 정리되어 있는가?
- [ ] 실패 시 조정 기준이 미리 정해져 있는가?

---

## 4단계: 경험 흐름 설계 점검 (3-3)

### 도파민 루프 5단계
- [ ] **상황**: 플레이어가 처한 맥락이 명확한가?
- [ ] **결핍**: 플레이어에게 부족한 무언가가 설계되어 있는가?
- [ ] **욕망**: 플레이어가 갖고 싶어할 보상이 눈에 보이게 배치되어 있는가?
- [ ] **도전**: 보상을 얻기 위해 넘어야 할 벽이 설계되어 있는가?
- [ ] **보상**: 도전의 크기에 비례하는 보상이 설계되어 있는가?

### 시간 단위 경험 설계
- [ ] **첫 30초**: 플레이어를 매료시킬 경험이 설계되어 있는가?
- [ ] **첫 3분**: 30초 경험의 바리에이션으로 3분을 채울 수 있는가?
- [ ] **첫 10분**: 긴장이 올라가는 순간과 내려가는 순간이 설계되어 있는가?

---

## 5단계: 검증 기준 점검 (5-1)

### 검증의 3층위
- [ ] **기능 검증**: 특정 행동에 기대한 결과가 나오는지 확인할 수 있는가?
- [ ] **전달력 검증**: 기획 의도대로 경험이 전달되는지 측정할 기준이 있는가?
- [ ] **재미 검증**: 코어루프의 반복 빈도/재플레이 의향을 측정할 기준이 있는가?

### 수치화된 검증 기준 (최소 3개 이상)
- [ ] 검증 기준이 구체적인 숫자로 정해져 있는가?
  - 예: "플레이어가 3분 안에 메카닉 응용 구간을 넘어간다"
  - 예: "첫 번째 보스에서 평균 3회 재도전한다"
  - 예: "회피/패링 성공률이 전투당 평균 70%에 도달한다"
- [ ] 검증 실패 시 어디로 돌아갈지 정해져 있는가?
- [ ] 몇 회 실패 시 기능을 축소/포기할지 정해져 있는가?

---

## 6단계: 개발 구조 점검 (2-1, 2-2)

### 게임의 4요소 (Elemental Tetrahedron)
- [ ] **메카닉**: 규칙과 시스템이 문서화되어 있는가?
- [ ] **스토리**: 플레이어에게 동기를 주는 요소가 정리되어 있는가?
- [ ] **미학**: 시각/청각 스타일이 결정되어 있는가?
- [ ] **기술**: 사용할 엔진/도구가 결정되어 있는가?
- [ ] 네 요소 간의 충돌이나 모순은 없는가?

### 개발의 4요소
- [ ] **기획**: 무엇을 만들지 문서로 정리되어 있는가?
- [ ] **구현**: 어떻게 만들지 기술적 방향이 정해졌는가?
- [ ] **리소스**: 필요한 그래픽/사운드/텍스트 목록이 있는가?
- [ ] **파이프라인**: 작업 흐름이 정리되어 있는가?

### 퀄리티 체크
- [ ] **게임 퀄리티**: "지루한 구간은 없는가?"를 체크했는가?
- [ ] **개발 퀄리티**: "빠뜨린 단계는 없는가?"를 체크했는가?
- [ ] 역량을 벗어난 계획은 없는가?
- [ ] 동시에 만족할 수 없는 조건이 충돌하지 않는가?
- [ ] 의도에 맞는 수단이 사용되고 있는가?

---

## 7단계: 협업/작업 관리 점검 (4-1, 4-2)

### AI/협업자를 위한 스펙 준비
- [ ] 핵심 기능의 스펙 문서가 작성되어 있는가?
  - [ ] **상황**: 현재 프로젝트 상태가 기술되어 있는가?
  - [ ] **의도**: 왜 이 작업을 하려는지 설명되어 있는가?
  - [ ] **기대**: 완성 시 어떤 모습이어야 하는지 묘사되어 있는가?

### 작업 분담
- [ ] 직접 할 작업과 맡길 작업이 구분되어 있는가?
- [ ] "하지 않을 것" 목록이 5개 이상 정해져 있는가?
- [ ] 작업이 최소 단위로 쪼개져 있는가?
- [ ] 작업 간 의존 관계와 순서가 정리되어 있는가?

### 관리 문서
- [ ] **규칙 문서**: 작업 방식에 대한 약속이 정해져 있는가?
- [ ] **기획서**: 게임의 모습이 문서화되어 있는가?
- [ ] **체크리스트**: 작업 진행 상황을 추적할 도구가 준비되어 있는가?
- [ ] Git 등 버전 관리 도구가 설정되어 있는가?

---

## 8단계: 프로젝트 환경 점검 (5-2, 5-3)

### 시간 관리
- [ ] 개발에 쓸 수 있는 시간이 현실적으로 계산되어 있는가?
- [ ] 정해진 작업 시간이 있는가?
- [ ] 예상 소요 시간에 2~3배 여유를 두었는가?
- [ ] 한 달 단위 목표가 설정되어 있는가?

### 외부 환경
- [ ] 피드백을 받을 채널이 준비되어 있는가? (SNS, 커뮤니티, 지인 등)
- [ ] 일주일에 한 번 이상 진행 상황을 공유할 계획이 있는가?
- [ ] 마감일이 정해져 있는가?
- [ ] 마감을 누군가에게 선언했는가?

### 결정 피로 관리
- [ ] 핵심 원칙 외의 판단은 외부 피드백으로 대체할 준비가 되어 있는가?
- [ ] 반복적인 결정을 줄일 수 있는 규칙이 정해져 있는가?

## 최종 점검: 개발 착수 가능 여부

### 필수 항목 (이것들이 없으면 개발 착수 금지)
- [ ] 핵심 경험 한 문장 정의
- [ ] 코어루프 3요소 (행동/성장/고난) 설계
- [ ] 첫 30초/3분 경험 설계
- [ ] 수치화된 검증 기준 3개 이상
- [ ] 기획서 또는 스펙 문서

### 권장 항목 (있으면 훨씬 수월함)
- [ ] 도파민 루프 5단계 설계
- [ ] "하지 않을 것" 목록
- [ ] 작업 체크리스트
- [ ] 피드백 채널 및 마감일
- [ ] 버전 관리 도구 설정

---

## 점검 결과 해석

**필수 항목 5개 중:**
- 5개 완료: 개발 착수 가능
- 3~4개 완료: 부족한 부분 보완 후 착수
- 2개 이하: 기획 단계로 돌아가기

**권장 항목 5개 중:**
- 4개 이상: 프로젝트 완주 가능성 높음
- 2~3개: 개발 중 보완 필요
- 1개 이하: 중도 포기 위험 있음
