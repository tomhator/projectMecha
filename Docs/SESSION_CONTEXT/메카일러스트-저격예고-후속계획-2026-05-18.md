# 메카 일러스트 교체 및 후속 계획

## 요약
> 전투 UI의 메카 표시를 `ColorRect`에서 `TextureRect` 기반으로 전환했고, HUD 타입 오류를 수정했다.  
> 이어서 오늘 기획 기준 남은 코딩 작업(파츠 저격 예고 연동, affix 전투 효과 런타임 적용)을 식별하고 우선순위 계획을 정리했다.

## 작업 내용

### 논의된 사항
- 메카 일러스트 실아트 교체는 코드 레벨 전환을 우선 완료하고, 아트 리소스 부재 시 fallback 텍스처로 동작 유지.
- HUD 중앙 코어 셀 처리에서 `int`와 `String` 비교가 섞여 런타임 에러 발생, sentinel 상수 기반으로 정리.
- 오늘 기획 문서 기준 남은 주요 코딩 작업은 2개:
  - 파츠 저격 예고 연동 (`SkillData.target_slot`, `EventBus.enemy_snipe_preview_changed`, `CombatUI` 하이라이트)
  - affix 전투 효과 런타임 적용 (`MechaEntity`/`TurnManager` 중심)

### 구현/변경된 파일
- `Scenes/UI/CombatUI.gd` — 메카 레이어를 `TextureRect`로 교체, 텍스처 경로/폴백 생성 로직 추가
- `Scenes/UI/CombatUi.tscn` — `MechIllustration` 최소 크기 상향 조정
- `Scenes/UI/CombatUI.gd` — `_build_parts_hud()`의 코어 셀 식별 로직 타입 안정화 (`HUD_CORE_CELL`)

### 결정 사항
- 다음 구현 우선순위는 **파츠 저격 예고 연동 먼저**로 확정.
- affix 런타임은 후속 작업으로 분리하되, AP/데미지/트리거를 한 번에 넣지 않고 단계적으로 적용.
