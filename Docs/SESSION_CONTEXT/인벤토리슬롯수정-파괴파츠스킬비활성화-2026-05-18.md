# 인벤토리 슬롯 4×4 수정 + 파괴 파츠 스킬 버튼 비활성화

## 요약
> PR #5 머지 이후 다음 세션 TODO 두 항목을 처리했다.  
> 조립 씬 인벤토리 슬롯을 GDD §4.6에 맞게 5×6=30 → 4×4=16으로 수정하고,  
> 전투 화면에서 파괴된 파츠 소속 스킬 버튼을 `disabled = true`로 비활성화했다.

---

## 작업 내용

### 논의된 사항

1. **브랜치 rebase**
   - `claude/review-context-next-task-5E5op`가 PR #5 머지 전 상태였음.
   - `git rebase origin/main`으로 main(PR #5 포함) 위에 재배치 후 작업 진행.

2. **인벤토리 슬롯 수 불일치**
   - `AssemblyScene.gd`의 `INVENTORY_COLS=5`, `INVENTORY_ROWS=6` 이 GDD와 불일치.
   - GDD §4.6: 16슬롯(4×4) 시작, 코어 업그레이드로 24→28 확장.
   - `AssemblyScene.tscn`의 `InventoryGrid columns=5`도 함께 수정 필요.

3. **파괴 파츠 스킬 버튼 처리 부재**
   - 파츠가 파괴(`durability == 0`)되어도 소속 스킬 버튼이 활성 상태로 남아있었음.
   - `_set_skill_buttons_disabled(false)` 호출 시 파괴 스킬 버튼도 재활성화되는 문제.
   - 해결: `_broken_skills` 딕셔너리로 파괴 스킬을 추적, 재활성화 단계에서 skip.

### 구현/변경된 파일

- **`Scenes/Assembly/AssemblyScene.gd`**
  - `INVENTORY_COLS: int = 5` → `4`
  - `INVENTORY_ROWS: int = 6` → `4`
  - `INVENTORY_SLOTS` = 30 → 16 (자동 계산)

- **`Scenes/Assembly/AssemblyScene.tscn`**
  - `InventoryGrid` `columns = 5` → `4`

- **`Scenes/UI/CombatUI.gd`**
  - 멤버 변수 `_broken_skills: Dictionary` 추가 (SkillData → true)
  - `_rebuild_skill_buttons()`:
    - `GameState.equipped_parts` 순회 → `is_broken()` 파츠의 스킬을 `_broken_skills`에 등록
    - 해당 스킬 버튼: `disabled = true`, `tooltip_text = "파츠 파괴됨"`, pressed 시그널 미연결
  - `_set_skill_buttons_disabled()`:
    - `_broken_skills.has(s)` 체크 → 파괴 스킬은 `continue` (재활성화 방지)

### 결정 사항

- 파괴 파츠 스킬은 버튼을 **숨기지 않고 disabled로 표시** — 플레이어가 손실을 인지하도록.
- 파괴 스킬 추적은 `_skill_to_part`(MechaEntity)에 접근하지 않고,  
  CombatUI 내에서 `GameState.equipped_parts`를 직접 순회하는 방식으로 구현.  
  MechaEntity의 내부 딕셔너리를 외부에 노출하지 않아도 됨.
