## 작업 내용

### 논의된 사항
- 구현은 plan 순서(저위험 UI 가독성 → 구조 개선 → 툴팁 → 타겟팅 피드백 → affix 계산식 정합성)로 진행
- plan 파일은 수정하지 않고 코드/문서만 반영

### 구현/변경된 파일
- `Scenes/UI/CombatUI.gd` — 적 카드 예고/호버 라벨 2줄 분리, 줄바꿈 적용, 스킬 비활성 사유 툴팁, SELF 타겟팅 시 PlayerColumn 강조
- `Scenes/Entities/MechaEntity.gd` — preview/실사용 공통 계산식에 affix 보정 반영, 반격 본능 활성/소모 및 진심펀치 대기 상태 처리
- `Docs/UI/CombatScreen.md` — 예고/호버 라벨 구조, 툴팁 정책, SELF 컬럼 강조, affix 예상 수치 반영 규칙으로 문서 갱신
- `Docs/WorkNote/TODO-2026-05-18.md` — 구현 리스트 5개 완료 체크

### 결정 사항
- 적 카드 라벨은 단일 라벨 점유 방식 대신 `예고`/`호버 예상` 2라벨 병행 표시로 확정
- 스킬 버튼 비활성 사유는 최소 `파츠 파괴`/`AP 부족`을 툴팁으로 표준화
- 예상 수치 계산은 `use_skill` 경로와 동일 계수 체계를 사용해 UI-실제 값 불일치 최소화
