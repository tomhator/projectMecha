# 전투 UI 타겟팅·예상 수치 표시 + 조립 드래그 하이라이트

## 요약
> 적 카드에 HP/쉴드 숫자를 바 위에 겹쳐 표시하고, 스킬을 고른 뒤 적·내 메카에 호버하면 예상 피해/회복량이 뜨도록 했다. 자기 대상 스킬도 이제 「스킬 클릭 → 내 메카 클릭」으로 발동된다. 조립 화면에서는 인벤토리 카드를 드래그 시작한 순간부터 호환 슬롯이 호버 색으로 강조된다.

## 작업 내용

### 논의된 사항

1. **적 카드 숫자 표시 방식**
   - 처음에는 HBox로 바 옆에 두었으나, 사용자 요청에 따라 `Control` 한 칸 안에 `ProgressBar` + `Label`을 `PRESET_FULL_RECT`로 겹치도록 변경.
   - 라벨 정렬은 가운데, 텍스트는 접두어 없이 `현재 / 최대` 숫자만 표시 (`42 / 100`).

2. **스킬 → 타겟 흐름 통일**
   - 적 대상: 기존대로 스킬 → 적 카드 클릭.
   - **자기 대상: 즉시 발동을 제거하고 스킬 → 내 메카(왼쪽 패널) 클릭**으로 통일.
   - `PlayerColumn` 하단에 투명 `Button`(`_player_target_catcher`)을 두고, 대기 중에만 `mouse_filter = STOP`으로 클릭을 받는다.

3. **호버 시 예상 수치**
   - 적 호버: 적 카드 상단 주황 라벨에 `예상 피해 N (쉴드 A · HP B) · 나 HP +x · 나 쉴드 +y`.
   - 자기 호버: 좌측 `PlayerColumn`에 새로 추가한 주황 라벨에 `예상 HP +x · 쉴드 +y`.
   - 회복량은 `MechaEntity.use_skill`과 동일한 상한(코어 max HP/쉴드)을 반영한 "실제로 오를 양".

4. **조립 화면 드래그 하이라이트**
   - 기존: 슬롯에 마우스를 올려야만 `_can_drop_data`에서 색이 바뀜.
   - 변경: 카드 `_get_drag_data` 시 `EventBus.assembly_drag_inventory_part`에 저장 + `part_socket_drag_highlight` 그룹에 알림 → 호환 슬롯은 드래그 내내 호버 색을 유지.
   - 드래그 종료 시 `AssemblyScene._notification(NOTIFICATION_DRAG_END)`가 그룹을 청소.

### 결정 사항

- **자기 대상 스킬 발동 정책**: 기존 즉시 발동 방식을 폐기. 모든 스킬이 「스킬 선택 → 대상 클릭」 2단계로 통일된다. 이후 패시브/광역 스킬이 추가되면 별도 분기 필요.
- **숫자 표시 위치**: 적 HP/쉴드 바에 오버레이. `outline_size = 2`로 가독성 확보. 라벨은 `mouse_filter = IGNORE`로 적 카드 `Button` 클릭을 방해하지 않게 한다.
- **드래그 하이라이트 전파 방법**: 시그널 대신 그룹(`part_socket_drag_highlight`) + `EventBus` 변수 1개를 채택. 소켓을 동적으로 추가/제거해도 자동으로 따라온다.

## 후속 작업 — 2026-05-14 (밤)

### 보고된 문제와 해결

1. **스킬 하나를 고르면 다른 스킬이 잠겨 교체 불가**
   - 원인: `_set_skill_buttons_disabled(true, skill)`이 모든 버튼을 `disabled = true`로 잠그고, 펜딩 강조는 `disabled` 스타일박스에만 적용되어 있었다.
   - 해결: `_set_skill_buttons_disabled` 동작을 「버튼은 항상 활성 유지, 펜딩 스킬만 시각 강조」 토글로 변경. `_style_skill_button_pending`을 `normal`/`hover`/`pressed` 스타일박스에 적용. `_clear_skill_button_decoration`에서 `font_hover_color`까지 정리.
   - 결과: 펜딩 중에도 다른 스킬을 자유롭게 다시 클릭해서 펜딩을 교체할 수 있다. ENEMY↔SELF 모드 전환 정리는 기존 `_on_skill_button_pressed` 분기에서 처리 (적 타겟팅 OFF / 자기 캐처 OFF·ON).

2. **회복 스킬 사용 시 아무 반응 없음**
   - 원인: 자기 대상용 `_player_target_catcher`가 `flat = true` + 빈 텍스트 + 스타일 없음 상태로, 펜딩 SELF여도 시각적으로 전혀 보이지 않아 사용자가 어디를 클릭해야 할지 알 수 없었다. (`MechaEntity.use_skill`의 `skill_heal > 0.0 → GameState.heal_hp` 로직 자체는 정상.)
   - 해결: `_set_player_self_target_pending(on)`을 확장해 `on=true`일 때 `flat = false`, 텍스트 `"내 메카 클릭"`, 점선 박스 스타일(`_style_player_target_catcher_active`)을 적용. `on=false`에서 `_clear_player_target_catcher_style`로 복귀.
   - 결과: 회복 스킬 선택 시 왼쪽 패널 하단에 녹색 점선 박스가 보이고, 클릭하면 회복이 정상 발동한다.

### 전투 화면 상단 중앙 턴 표시 추가

- 사용자 요청: "전투화면일 때 상단 중앙에 현재 몇 턴인지 보여줘."
- 구현:
  - `Scripts/Autoload/EventBus.gd` — `signal combat_turn_changed(turn: int)` 추가.
  - `Scenes/Combat/TurnManager.gd` — `current_turn: int` 카운터. `start_combat`에서 0으로 초기화, `start_player_turn` 시작 시 +1 + `EventBus.combat_turn_changed.emit(current_turn)`. 콘솔 로그도 `[턴 N · 플레이어] …` 형태로 갱신.
  - `Scenes/UI/CombatUi.tscn` — `RunStatusStrip`(0~56px) 바로 아래 60~92px에 `TurnLabel`(가로 풀폭, 중앙 정렬, font 18, 옅은 노란빛). `BattleMargin.offset_top` 64 → 96으로 내려 겹침 방지.
  - `Scenes/UI/CombatUI.gd` — `@onready var turn_label = %TurnLabel`, `_on_combat_turn_changed(turn)` 핸들러 연결, 라벨 텍스트 `"턴 N"`.

### 결정 사항 (보강)

- **스킬 버튼 정책**: 타겟 확정 전까지는 모든 스킬 버튼이 활성 유지된다. 펜딩 강조는 시각만 담당하고 클릭은 항상 가능. AP 부족이나 손상 잠금처럼 "근본적으로 못 쓰는" 상태일 때만 진짜 `disabled`를 사용한다.
- **자기 대상 클릭 영역**: 별도 캐처 박스(점선)로 표시. 향후 캐릭터 일러스트가 들어가면 `PlayerColumn` 자체에 테두리를 두르는 방식으로 재배치 검토.
- **턴 카운터 위치**: `RunStatusStrip` 안이 아니라 `CombatUI` 루트 자식으로 분리. 던전맵·조립 등 다른 화면에 표시할 일이 없는 전투 전용 위젯이기 때문.
