# 던전 맵·전투 HUD·런 스트립 UI 레이아웃 (2026-05-11)

## 요약

이번 세션에서는 던전 맵 방 타일의 세로 과대 표시를 바로잡고, 상단 `RunStatusStrip`의 HP/쉴드 표기를 바 위 숫자로 정리했으며, 던전 방 정보를 **호버·모달 대신 클릭 + 우측 고정 패널**로 바꿨다. 전투 화면은 적을 **가로 배치**하고 본전 영역을 **좌≈40% / 우≈60%(우측 너비 200~500px 클램프)** 로 나누었다.

## 작업 내용

### 논의된 사항

- 던전 맵: 방 선택 타일이 세로로 늘어나 보임 → `ChoiceContainer`의 세로 EXPAND 제거 + 타일 `custom_minimum_size` 유지로 정사각형 유지.
- 런 HUD: HP·쉴드 숫자를 프로그레스 바 **위**에 두고, `현재 / 최대` 텍스트로 가독성 확보.
- 던전 맵: 정보는 **클릭 시에만** 갱신, **우측 정보 패널** 고정, 맵 콘텐츠는 좌측 `BodyHBox` 레이아웃으로 이동.
- 전투: 적 목록 **세로 → 가로**, 좌/우 **4:6 의도**(우측 목표 폭의 60%를 200~500으로 클램프, 플레이어 열은 나머지 가로 확장).

### 결정 사항

- 던전 방 정보: 모달 대신 **항상 보이는 우측 패널**(약 300px) + 진입은 패널 버튼만.
- 전투 우측 열 너비: **목표 = 가용 가로의 60%**, **최소 200 · 최대 500**; 화면이 매우 좁으면 플레이어 최소 여유(약 64px)를 위해 우측이 200 미만으로 줄 수 있음.
- 적 다수 시 카드는 `HBox` 내에서 `SIZE_EXPAND_FILL`로 균등 분배, 카드 최소 가로 72px.
