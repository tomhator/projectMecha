# ⚙️ Project Mecha — 기획 정본 정리 (재빌드 기준)

> **작성일:** 2026-06-11
> **목적:** 여러 시점의 GDD 패치·spec·세션 기록에 흩어진 기획을 **하나의 기준 문서**로 통합한다.
> 새 프로젝트는 이 문서 하나에서 출발할 수 있어야 한다.
>
> **왜 만들었나:** 게임의 *토대*(정체성·루프·파츠·전투)는 이미 구현·검증까지 끝나 안정적이다.
> 그러나 *코어 기둥*(코어 어빌리티·동사·교리·키스톤·메타·boon)이 수렴하지 못하고 여러 번 재설계되면서,
> 옛 모델(GDD 본문)과 최신 결정(spec)이 충돌하고 "확정 → 재논의" 루프가 반복됐다.
> 이 문서는 **무엇이 굳었고 / 무엇이 아직 정하는 중이고 / 무엇이 폐기됐는지**를 분리해 그 혼란을 끝낸다.

## 범례 (상태 태그)

| 태그 | 의미 |
|------|------|
| ✅ **확정·구현** | 코드로 존재하고 headless/플레이로 검증됨. 새 프로젝트가 **그대로 가져간다.** |
| 🟡 **확정·미구현** | 기획은 굳었으나 아직 코드 없음. 그대로 구현하면 됨. |
| 🔧 **재설계 중** | 최신 합의는 있으나 **미수렴**. 새 프로젝트가 **이어서 확정해야 할 영역.** |
| ❌ **폐기** | 과거 결정이지만 무효. **다시 끌어오지 말 것** (§11). |

## 0. 한 줄 정체성

> 고철 부품을 주워 메카를 조립하고, 그 메카로 적과 싸우는 **턴제 메카 빌더 로그라이크.** (2D 픽셀 아트, Godot 4.x, 런 ≈15분)

**핵심 철학:** "유저에게 생각할 시간을 주어서는 안된다. 하지만, 매 순간 유저는 최선의 생각을 해야 한다." — 불필요한 마찰은 줄이되, 매 선택은 의미 있게.

**핵심 재미 (✅ 2026-06-03 확정, 이후 일관):**
> **"파츠가 주인공 — 적극적으로 사용·버리고·교체하고 활용한다."**
> 파츠는 코어를 지키는 **소모성 장갑 + 화력**이다. 교체는 권장되고(조립=무비용, 전투=손패 뒤로 무손실), 파괴 비용은 *의도적 소비(투척·희생)* 에만 붙는다.

## 1. 런 구조 (✅ 확정·구현)

**핵심 루프:** 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투

```
거점(외곽 은신처) → 출격 게이트 → 던전 특성 확인 → 출격 빌드 설계 → 던전 진입
던전 내: [전투 클리어 → 보상 → (재조립 가능) → 다음 방 2갈래 선택] 반복
런 종료: 보스 처치(성공) / 코어 HP 0(실패) → 결과 화면 → 거점 복귀
```

**던전:** 총 **10층**, 각 층 **2갈래 분기**(방 종류 + 예상 보상 등급 미리 공개).

| 층 | 구성 |
|----|------|
| 1~3 | 일반 전투 |
| 4~5 | 분기 선택 (작업대 1회 보장) |
| 6~8 | 일반 + 엘리트 전투 |
| 9 | 작업대 보장 (보스 전 정비) |
| 10 | 보스 전투 (클리어 시 런 종료) |

**방 종류:** 일반 전투 / 엘리트 전투 / 상자(RARE 75%·EPIC 25%) / 조우 이벤트 / 작업대 / 보스.

**재화 (✅):**
- **크레딧** — 전투 클리어 시 고정 지급(일반 15~25 / 엘리트 45~65 / 보스 85~105). 수리·거점 코어 연구에 사용.
- **고철** — 커먼(⚙)/레어(✦) 2등급. 분해·고철더미·드롭으로 수급. 작업대 제작 재료 + 거점 정비.

**보상 드롭:** 격파 적 1기당 0개 40% / 1개 50% / 2개 10%. 드롭은 선택 없이 전부 인벤토리로. (`RewardManager` → `PartsFactory`)

**결과 화면:** 성공/실패 공통으로 도달 층·소요 시간·크레딧 순이익·획득/파괴 파츠 수·희귀도별 집계 등 표시.

## 2. 거점 시스템 (✅ 확정·구현)

오염 구역 밖 **외곽 은신처.** 런 사이 다음 출격을 준비한다.

| 구역 | 기능 |
|------|------|
| 은신처 마당 | 메인, 재화/상태 요약, 각 구역 진입 |
| 격납고 | 영구 파츠 창고, 장착 준비, 수리, 분해 |
| 코어 연구대 | (🔧 재설계 대상 — §8) 코어 노드 연구/성장 |
| 작전 단말 | 퀘스트, 업적, 기록, 세계관 로그 (스텁) |
| 출격 게이트 | 던전 선택, 특성 확인, 출격 빌드 확정 |
| 시스템 콘솔 | 저장/불러오기/옵션 (스텁) |

**정산 규칙 (✅):** 거점 창고와 런 인벤토리 분리. 성공 시 장착·런 인벤토리 파츠 전부 회수, **실패 시 장착 파츠 영구 손실** + 런 인벤토리 회수 + 재화 50% 회수. (`ConfigFile` 저장/복원 구현)

## 3. 파츠 시스템 (✅ 구현 / 단 affix는 🔧 재설계 예정)

**슬롯 구조 (✅):** 코어 중심 기본 4슬롯 + 조건부 추가 팔.

| 부위 | 슬롯 | 성향 | 비고 |
|------|------|------|------|
| 팔 | 2 | 공격·방어 | 능동 행동 |
| 등 | 1 | 방어·회복·버프 | 지원형 |
| 다리 | 1 | 방어·회피·반격 | **피격 시 발동** 조건 |
| 추가 팔 | 조건부 1 | 공격·방어 | `evolution_lord` affix 보유 시 개방 |

**파츠 풀 (✅):** 고유 34종 (ARM_L 8 / ARM_R 8 / BACK 8 / LEG 10). 스킬은 파츠에 고정.

**등급 (✅):** 파츠 고정 속성이 아니라 **드롭 시 굴린 affix 개수**로 결정.

| 등급 | affix 개수 | 초기 손상도 |
|------|-----------|-----------|
| COMMON | 0~1 | 3 |
| RARE | 2~3 | 5 |
| EPIC | 4~5 | 7 |

**파손 규칙 (✅ — "파츠가 주인공" 정합):**
- 적 공격으로 내구도 0 → **파손**(장착 불가, 슬롯 점유, 스킬/affix 비활성)
- 의도적 소비(투척·희생) → **파괴**(소멸)
- **던전 중 교체는 파손을 만들지 않는다** (조립=무비용)
- 처분: 거점 수리(고철) / 분해(슬롯 해방) / 전투 중 투척·덧대기

**인벤토리 (✅):** 기본 16칸(4×4). 가득 차면 드롭 유실. (확장은 추후 코어 업그레이드)

**조립 UX (🟡 설계 완료, 구현 단계 A 대기):** 드래그앤드롭으로 소켓에 파츠를 붙이면 **메카 실루엣이 변한다**는 시각적 조립 경험이 차별점. 단계 A(직사각형 블록 프로토타입) → B(Skeleton2D 스프라이트) → C(공간형 조립실, 장기 비전).

> ⚠️ **affix는 🔧 재설계 예정:** 현재 affix 풀은 구현돼 있으나, 코어 재기획(§8)에서 **교리 태그가 붙은 시너지 조각**으로 전면 재설계하기로 함. 이것이 "파츠 기둥" 재기획의 핵심이며 boon의 선행 의존성이다.

## 4. 전투 시스템 (✅ 확정·구현·검증)

- **구도:** 내 메카 1 vs 적 메카 1~3
- **방식:** 교대식 턴제 (내 턴 → 적 턴 반복)
- **승패:** 적 코어 HP 0 승리 / 내 코어 HP 0 패배(런 종료)

**내 턴 (✅):** 턴당 행동력 = 코어 기본 2 + 파츠 AP 기여 + 코어 성장 보정. 스킬마다 AP 비용 지불, 타겟 직접 지정(적 여럿일 때). **기본 공격은 파츠가 다 파손돼도 쓰는 안전판.**

**적 턴 (✅):** 랜덤 스킬 선택, 단 **다음 턴 행동을 미리 공개**(예고)해 플레이어가 대응 판단.

**구현·검증된 메커니즘 (✅ headless):** 멀티타겟 분배, affix 보너스 계약(productive·overload·gambler·lifedrain·momentum·evolution_lord 등), 버프/디버프 상태(BURN·ATTACK_DOWN·AP_DOWN·방어막·나노수복·시즈모드·회피/차단/반격), 현장 수리 타겟팅, `EXTRA_ARM` 조건부 슬롯 저장/복원.

> 상세 단일 기준: [[CombatSpecification]]. (단, 코어 어빌리티 관련 §은 §8 최신안으로 갱신 필요)

## 5. 적 시스템 (✅ 매립지만 구현 / 타 지역 P4)

적은 Protocol Interrupt 당시 **처해 있던 환경(지역)** 으로 분류된다. 적 슬롯은 스킬 풀 다양성만 결정하며 타겟·파괴 불가.

| 지역 | 별칭 | 병종 방향 | 보스 | 상태 |
|------|------|----------|------|------|
| **매립지** | 스크래퍼 | 공격/파츠 압박/지원/증원 (역할 튜토리얼) | 수집가 | ✅ 일반·엘리트·보스 v1 구현·검증 |
| 구 도시 | 위병 | 차단·경보·제압 편대 | 미정 | P4 (데모 이후) |
| 연구시설 | 레플리카 | 복제·출력 폭주·오류 주입 | 미정 | P4 (데모 이후) |

**수집가 보스 (✅ 검증):** 코어 1 + 팔 4 = 5체, 팔 독립 행동, 팔 전멸 시 코어 노출/재수집, 팔 탈취(플레이어 ARM 강탈 후 적으로 사용), 코어 HP 0 시 즉시 승리. **저격(EMP)**: BACK 슬롯 내구도 표적 메커니즘.

## 6. 세계관 (✅ 데모 스코프)

인류 기술이 정점에 달한 미래, 기계들이 발신원 불명의 **명령어**를 수신해 폭주, 문명을 쑥대밭으로 만들었다. 살아남은 인간 일부는 **스캐빈저**가 되어 오염 구역에서 파츠를 골라내 생존한다. 플레이어는 그중 한 명이고, **한 런 = 오염 구역에 들어가 파츠를 모아 나오는 한 번의 출격.**

> 명령어의 정체는 데모에서 의도적으로 비공개. 인간 진영(밴딧·기계교단 등)은 데모 이후. 상세: [[WorldSetting]]

## 8. 🔧 재설계 중 영역 — 코어 기둥 (새 프로젝트가 확정해야 할 곳)

> **여기가 "방향 상실"의 진원지였다.** 06-01 → 06-09 → 06-10에 걸쳐 코어/동사/교리가 반복 재설계됐다.
> 아래는 **2026-06-10 시점의 최신 합의**다. 개념은 상당히 굳었으나 **수치·개별 노드·구현은 전부 미결**이며, 코드에는 거의 0건이다.
> 정합성 원칙: **"코어·파츠 먼저 → 그 위에 던전 이벤트(조우·상자·boon)."** 토대 없이 이벤트부터 설계하면 모래 위에 짓는 것.
> 출처: [[superpowers/specs/2026-06-09-코어시스템-재기획-design]] (코어 기둥) · [[superpowers/specs/2026-06-10-동사-업그레이드-묶음-밸런스초안-design]] (동사).

### 8.1 코어 본체 (개념 확정)
- **단일 주인공.** 절대 교체·해금 없음, 데모~정식 1개 고정.
- **런 중 고정.** HP/하중/AP/슬롯 등 본체 스탯은 런 동안 불변. **런 내 파워 변동은 파츠가 담당.**
- **성장 = 메타.** 단일 코어의 노드 트리/예산/슬롯이 전체 진행도로 깊어진다. (하데스 자그레우스 결.)
- **이중 주인공:** 스토리 주인공=코어(상수 파일럿) / 게임플레이 주인공=파츠(변수). "파일럿은 무대, 파츠는 배우."

### 8.2 코어 어빌리티 / 동사 모델 (최신 = 2026-06-10)
코어 어빌리티 = 파일럿의 **파츠 조작 동사 그 자체.** 출격 전 **3슬롯 로드아웃**(런 중 고정), 메타로 슬롯 **시작 1 → 2 → 3 (→후반 4)**.
- **공통 비용 = AP + 재료** 단일축 (새 자원 0, "공짜 금지" 가드레일).

**상시 기본기 (콘솔 밖·슬롯 무관·항상 사용):**
| 기본기 | 내용 |
|--------|------|
| 기본공격 | 출격 전 노드로 1택. 각각 교리 보유 → 키스톤 타이브레이커 |
| 긴급 장착 | 파손돼 빈 슬롯에 큐 파츠 자동 장착 (손실 복구) |

> 기본공격 3종 + 교리 태그: 입자 광선=**태운다** / 간이 방벽=**살린다** / 속사 사격=**중립**. (콘셉트·수치는 기존 유지, 태그만 추가)

**슬롯에 넣는 동사 7종** (수리는 덧대기에 흡수되어 독립 동사 아님):

| 동사 | 기본형 | 교리 | 지속성 | 업그레이드 방향 (택1, 수치 미결) |
|------|--------|------|--------|----------------------------------|
| 교체 | 멀쩡 장착 파츠를 큐로·AP만 | 중립/회전 | 영구 | ⓐAP 0 무료회전 / ⓑ새 파츠 첫 사용 강화 |
| 덧대기 | 재료 파츠 소모 → 대상 내구도 복구 | 살린다 | 영구 | ⓐaffix 이전(0~15%) / ⓑ최대 내구도↑ / ⓒ능력치↑ |
| 투척 | 파손 파츠 던져 피해+소비 | 태운다 | 영구(소비) | ⓐ장착 파츠도 / ⓑ멀쩡 파츠도 (둘 다 불가) |
| 과부하 | 한 파츠 출력 폭주 → 사용 후 파괴 | 태운다 | 영구(가속) | ⓐ파괴 대신 파손 / ⓑ피해↑+코어 반동 |
| 재가동 | 파괴 파츠 50%로 부활 | 집착→소비 | **휘발/화장** | ⓐ부활 내구↑(75%) / ⓑ대상 +1 |
| 복제 | 1회용 사본 생성·장착 | 태운다 | **휘발** | ⓐ강화 사본 / ⓑ다중(2) / ⓒ영속 |
| 임시방편 | "때리기"만 가진 임시 파츠 생성 | 즉흥/중립 | **휘발** | ⓐ즉석 affix / ⓑ공격형 / ⓒ긴급 일괄 |

- **휘발 3종(재가동·복제·임시방편):** 전투 한정, 종료 시 소멸. **작업대=영구 공급 ↔ 휘발 동사=전투 중 즉흥 공급**(시간축 분리). 재가동=화장(50% 부팅→내구 소진 시 완전 소멸). 산출물은 즉시 장착(빈 슬롯 없으면 큐 최상위).
- **자연 발생 콤보:** 복제→과부하(사본)→투척 / 임시방편→투척·과부하(연료 엔진) / 과부하(파괴)→재가동→마지막 한 방.

### 8.3 교리 / 키스톤 (개념 확정)
- **교리 = 축/지향점**(클래스 아님). 현재 **태운다 / 살린다** (+중립), 확장형.
- **backbone = 태그:** 빌드 재료(동사·affix·교리 패시브·boon)마다 0~1개 교리 태그. 게임이 활성 태그 분포(=정렬)를 읽는다.
- **키스톤 결정:** ① 노드 교리 점수 합산(기본공격 제외) → ② 동점 시 기본공격 교리로 타이브레이크 → ③ 선두가 **문턱 ≥** 이면 강한 키스톤 / 미달이면 제너럴리스트(유연하나 밋밋).
- **"안 가둔다" 원칙:** 동사·노드는 자유 조합, 교리는 *결과로* 결정. commit이 곧 보상.

### 8.4 메타 성장 / 코어 업그레이드 / 노드 빌드 (개념 확정)
- **메타 = 스탯 뻥튀기 ❌ → 규칙 비트는 키스톤 + 트레이드오프 ⭕.** (예: "재가동 파츠 소멸 무효 / 대신 ___ 포기")
- **코어 업그레이드 노드 = 규칙 + 스탯(대가 필수).** 순수 +스탯 노드는 없음. 모든 노드가 교리 점수 보유 → 키스톤에 기여.
- **노드 빌드 = 평면 노드망.** 옛 5티어 게이팅·노드 레벨(1~5) 폐지. 진행 곡선 = 재화 예산 + 해금 순서.
- 출격 전 코어 설계 화면에서 확정: **3 동사 슬롯(시작 1) + 기본공격 1택 + 키스톤(합산 결과).**

### 8.5 boon (방향만, 미설계)
- 하데스 풀카피 아님. 맵 갈림길 중 일부가 **교리 패시브 부여**(교리 노드 = 전투/조우/상자). 같은 교리 반복 시 **약한 시너지.**
- boon = **미리 세팅해온 코어 어빌리티 강화**(새 인터랙트 없음). 런 한정 교리 태그로 키스톤/정렬 방향 증폭.
- **선행 의존성:** affix 교리 태그(§3 파츠 기둥)가 먼저 확정돼야 함.

### 8.6 이 영역의 미결 쟁점 (새 프로젝트가 정해야 할 것)
- [ ] 각 동사 **효과 수치** + 동사별 재료 종류/양
- [ ] 키스톤 **문턱 최소 점수** + 교리 점수 스케일
- [ ] 노드망 **재화 예산·해금 곡선** 수치
- [ ] **코어 외장 ↔ 노드 매핑 재설계** (옛 티어별 외장 슬롯 폐기 여파)
- [ ] 개별 노드 로스터(동사/기본공격/코어 업그레이드/교리 패시브) 목록·수치·대가
- [ ] **파츠 기둥:** 덱 구조(부위 슬롯 4 + 인벤토리 큐, 고정 진열 vs 드로우 미결) + **affix 전면 재설계**(교리 태그 시너지 글루)
- [ ] **boon 본 설계** (affix 태그 확정 후)
- [ ] 어빌리티 트리(AbilityTreeSystem) 코드 대수술 — 옛 5티어×3축 → 평면 노드망

## 12. 새 프로젝트 착수 권장 순서

1. **이 문서 §1~§7 (✅/🟡)를 그대로 이식** — 정체성·루프·거점·파츠·전투·적·세계관. 검증된 코드는 도메인 단위로 옮긴다.
2. **§8 코어 기둥을 마저 확정** — 순서: **(a) 파츠 기둥(덱 구조 + affix 교리 태그) → (b) 동사 수치·노드 로스터 → (c) 코어 설계 화면 → (d) 던전 이벤트·boon.** 토대(코어·파츠)부터, 이벤트는 그 위에.
3. **§11 폐기 목록을 가져오지 않는다** — 옛 트리·옛 동사표·옛 검증 기준 코드/문서 제외.
4. **나머지 화면 설계 완료**(§7) 후 디자인 규칙 정식화 → 화면별 구현. 신규 시스템은 **nV 검증 게이트** 통과 후.
