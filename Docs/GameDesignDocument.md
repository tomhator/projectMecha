## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 메카 빌더 로그라이크 |
| **엔진** | Godot 4.x (GDScript) |
| **타겟** | 데모 버전 (2026년 내 출시 목표) |
| **런 길이** | 약 15분 |
| **핵심 루프** | 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투 |

### 한 줄 소개

고철 부품을 주워 메카를 조립하고, 그 메카로 적과 싸우는 턴제 로그라이크.

---

## 2. 런 구조 (Run Structure)

### 2.1 전체 흐름

```
코어 선택 → 던전 탐색 (10층) → 보스 전투 → 런 종료
```

### 2.2 던전 구성

- 총 **10층** 구조
- 각 층마다 **분기 선택** 존재
- 다음 방 정보 미리 공개: `등급 + 부위 또는 특징`

**층 구성 예시**

| 층 | 내용 |
|----|------|
| 1~3층 | 일반 전투 + 부품 획득 |
| 4~5층 | 분기 선택 |
| 6~8층 | 일반 전투 + 부품 획득 |
| 9층 | 엘리트 전투 |
| 10층 | 보스 전투 |

### 2.3 방 종류

| 방 종류 | 설명 |
|--------|------|
| **일반 전투** | 적 처치 시 일반 등급 부품 보상 (가끔 드롭) |
| **엘리트 전투** | 강한 적, 희귀/에픽 부품 보상 |
| **상자** | 희귀 75% / 에픽 25% |
| **조우 이벤트** | 텍스트 이벤트, 보상 + 리스크 |
| **작업대** | 크레딧 소모 → 회복 or 부품 스킬 강화 |
| **보스** | 10층, 클리어 시 런 종료 |

### 2.4 재화 시스템

- 재화명: **크레딧 (Credits)**
- 획득처: 전투 보상
- 사용처: 작업대 (회복 / 부품 업그레이드)

## 3. 코어 시스템 (Core System)

런 시작 전 코어를 선택한다. 코어는 메카의 방향성을 결정하는 특성값을 가진다.

### 3.1 코어 종류 (데모: 3종)

| 코어 | 특성 | 플레이 스타일 |
|------|------|-------------|
| **범용** | 평균적인 스탯, 제약 없음 | 입문용, 밸런스형 |
| **경량** | 공격력 x0.6 / 매 턴 스킬 2회 선택 / 하중 제한↓ | 빠르고 유연한 빌드 |
| **방어** | 체력↑↑ / 공격력↓ / 공격횟수↓ | 버티면서 반격하는 빌드 |

> [!note]
> 경량 코어의 공격력 x0.6은 2회 선택의 유연성을 상쇄하기 위한 밸런스 수치. 플레이테스트 후 조정 예정.

## 4. 파츠 시스템 (Parts System)

> [!info] 상세 명세
> [[PartsSystem]] 참고

### 4.1 슬롯 구조

메카는 코어를 중심으로 4개의 부품 슬롯을 가진다.

| 부위 | 슬롯 수 | 스킬 성향 | 특징 |
|------|--------|----------|------|
| **팔** | 2 | 공격, 방어 | 능동적 행동 |
| **등** | 1 | 방어, 회복, 패시브 | 지원형 |
| **다리** | 1 | 방어, 패시브 | 피격 시 발동 조건 (회피, 반격) |

### 4.2 스킬 종류

| 스킬 | 설명 |
|------|------|
| **공격** | 적 코어 또는 부품에 피해 |
| **방어** | 피해 감소, 쉴드 부여 |
| **회복** | 코어 HP 회복 |
| **패시브** | 조건 충족 시 자동 발동 |

> [!note]
> 버프/디버프는 독립 스킬이 아닌 각 스킬의 **부가 효과**로 처리.
> 다리 스킬은 **피격 시 발동** 조건을 공통으로 가짐 (회피, 반격 등).

### 4.3 부품 풀 및 등급

고유 파츠 34종. ARM·BACK 슬롯 8종, LEG 슬롯 10종.

| 슬롯 | 파츠 수 |
|------|--------|
| ARM_L | 8 |
| ARM_R | 8 |
| BACK | 8 |
| LEG | 8 |
| **합계** | **32** |

등급은 파츠 자체에 고정된 속성이 아니라 드롭 시 굴린 **affix 개수**로 결정된다.

| 등급 | affix 개수 | 초기 손상도 | 주요 획득처 |
|------|-----------|-----------|-----------|
| **COMMON** | 0~1개 | 3 | 일반 전투 |
| **RARE** | 2~3개 | 5 | 엘리트 전투, 상자 |
| **EPIC** | 4~5개 | 7 | 엘리트/보스, 상자 |

> [!note] 랜덤 롤 시스템
> 파츠를 루팅할 때마다 수치·affix 개수·affix 종류가 달라진다.  
> 스킬은 파츠에 고정. 자세한 사항은 [[PartsSystem]], [[PartsCatalog]], [[AffixSystem]] 참고.

### 4.4 조립 UX — 시각적 조립 시스템

**핵심 설계 의도**

파츠가 스탯 + 슬롯으로만 표현되면 카드 게임과 다를 게 없다.  
차별점은 **파츠를 붙일 때마다 메카 실루엣이 바뀌는 시각적 조립 경험**에 있다.

ARM_L을 끼우면 왼팔이 생긴다. BACK에 대형 캐논을 달면 등이 무거워 보인다.  
4개 다 채우면 — 내 메카가 서 있다. 그 실루엣이 던전으로 걸어 들어간다.

**조립 방식**

- 런 시작 전 1회 조립. 던전 중 재조립 없음.
- 드래그앤드롭: 파츠를 소켓에 끌어다 붙인다.
- 호환 소켓: 초록 테두리 + 맥박 애니메이션 (장착 가능)
- 비호환 소켓: 빨간 테두리, 드롭 거부
- 장착 성공: 섀시 shake 피드백
- 탈착: 장착된 소켓 클릭 → 파츠가 트레이로 복귀

**소켓과 허용 파츠**

| 소켓 | 위치 | 허용 타입 |
|------|------|---------|
| ARM_L | 섀시 왼쪽 | arm만 |
| ARM_R | 섀시 오른쪽 | arm만 |
| BACK | 섀시 등 | back만 |
| LEG | 섀시 하단 | leg만 |

**구현 단계**

| 단계 | 설명 | 상태 |
|------|------|------|
| **A — 실루엣 먼저** | 직사각형 블록으로 드래그앤드롭 느낌 먼저 검증. `AssemblyPrototype.tscn` (독립 씬) | 🔜 다음 작업 |
| **B — Skeleton2D 모듈** | A 검증 후 Skeleton2D + Bone2D로 실제 파츠 스프라이트 교체 | 미정 |
| **C — 공간형 조립실** | 데모 이후 비전 — 던전 방 안에서 섀시에 끌어다 붙이는 경험 | 장기 비전 |

> [!note]
> A 단계 성공 기준: 4개 소켓을 전부 채웠을 때 "완성됐다"는 느낌이 오는지 본인이 3번 해보고 판단.  
> 느낌이 오면 B로, 안 오면 재진단.

## 5. 전투 시스템 (Combat System)

> [!info] 상세 명세
> [[CombatSpecification]] 참고

### 5.1 전투 구조

- **구도:** 내 메카 1 vs 적 메카 1~3
- **방식:** 교대식 턴제 (내 턴 → 적 턴 → 반복)
- **승리 조건:** 적 코어 HP 0
- **패배 조건:** 내 코어 HP 0 → 런 종료

### 5.2 내 턴

| 코어 | 턴당 스킬 선택 횟수 |
|------|----------------|
| 범용 / 방어 | 1회 |
| 경량 | 2회 (공격력 x0.6 적용) |

- 스킬 선택 후 **타겟 지정** (적이 여럿일 경우 플레이어가 직접 선택)
- 타겟 리스트가 비면 자동으로 적 코어 공격

### 5.3 적 턴

- 적은 **랜덤으로 스킬 선택**
- 단, **다음 턴 행동을 미리 공개** → 플레이어가 대응 판단 가능

### 5.4 전투 보상

- 크레딧 획득
- 일반 부품 **가끔** 드롭

## 7. 이벤트 시스템 (Event System)

### 7.1 상자

| 결과 | 확률 |
|------|------|
| 희귀 부품 | 75% |
| 에픽 부품 | 25% |

### 7.2 조우 이벤트

텍스트로 상황을 제시하고 플레이어가 선택한다.

**선택지**
- `한다` → 결과 발생 (보상 + 리스크)
- `지나간다` → 아무 일 없음

**결과 종류**

| 결과 | 내용 |
|------|------|
| A | 부품 획득 + 코어 HP 소량 손실 |
| B | 부품 획득 + 해당 부품 손상도 -1 |
| C | 부품 획득 실패 + 코어 HP 소량 손실 |
| D | 코어 HP 회복 + 공격력↓ |
| E | 공격력↑ + 코어 HP 손실 |

### 7.3 작업대

크레딧을 소모해 하나의 행동을 선택한다.

| 선택 | 효과 |
|------|------|
| **회복** | 코어 HP 회복 |
| **부품 업그레이드** | 장착된 부품 스킬 수치 강화 |

> [!note]
> 작업대에서는 반드시 하나만 선택 가능. 둘 다 사용 불가.

## 9. 세계관 (World Setting)

> [!info] 상세 설정
> [[WorldSetting]] 참고. 아래는 데모 범위의 최소 스케치.

인류의 기술이 정점에 달한 미래, 어느 날 기계들이 **명령어**를 수신하기 시작했다. 발신원은 불명. 명령어를 받은 메카들은 자의 없이 움직이며 인류 문명을 쑥대밭으로 만들었다.

살아남은 인간들은 외곽으로 흩어졌고, 일부는 **스캐빈저**가 되었다 — 오염 구역에 들어가 멀쩡한 파츠를 골라내 팔거나 재활용하며 생존하는 사람들.

플레이어는 그 중 한 명이다. 게임의 한 런 = 오염 구역에 들어가 파츠를 모아 나오는 한 번의 출격.

> [!note] 명령어의 정체는 데모에서 의도적으로 공개하지 않는다.
