## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 메카 빌더 로그라이크 |
| **엔진** | Godot 4.x (GDScript) |
| **타겟** | 데모 버전 (2026년 내 출시 목표) |
| **런 길이** | 약 15분 |
| **핵심 루프** | 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투 |

### 한 줄 소개

고철 부품을 주워 메카를 조립하고, 그 메카로 적과 싸우는 턴제 로그라이크.

## 2. 런 구조 (Run Structure)

### 2.1 전체 흐름

**첫 런 (최초 1회)**
```
코어 선택 → 던전 진입 (빈손)
```

**이후 런**
```
거점 (인벤토리·장비 세팅 + 코어 선택) → 던전 진입
```
> 거점 상세 기획은 추후 진행.

**던전 내 반복 루프**
```
전투 클리어
  → 보상 확인 (드롭 파츠 전부 획득 + 크레딧 지급)
  → 다음 방 확인 화면  ← 재조립 가능 (교체 시 기존 파츠 파손)
      └─ [재조립 선택 시] 조립 씬 → 다음 방 확인으로 복귀
  → 다음 방 이동 (2갈래 중 선택)
```

**런 종료**
```
보스 처치 → 보상 확인 → 결과 화면 → 거점으로 복귀   (성공)
코어 HP 0  → 결과 화면 → 거점으로 복귀              (실패)
```

### 2.2 던전 구성

- 총 **10층** 구조
- 각 층마다 **2갈래 분기 선택**
- 다음 방 정보 미리 공개: `방 종류 + 예상 보상 등급`

**층 구성**

| 층 | 구성 | 비고 |
|----|------|------|
| 1~3층 | 일반 전투 | 크레딧 + 격파 적별 파츠 드롭 |
| 4~5층 | 분기 선택 구간 | 작업대 1회 보장 포함 |
| 6~8층 | 일반 전투 + 엘리트 전투 | 격파 적별 파츠 드롭 |
| 9층 | 작업대 (보장) | 보스 진입 전 준비 |
| 10층 | 보스 전투 | 클리어 시 보상 후 런 종료 |

### 2.3 방 종류

| 방 종류 | 보상 드롭 | 크레딧 | 설명 |
|--------|---------|-------|------|
| **일반 전투** | 격파 적 1기당 0~2개 (COMMON) 전부 획득 | 15~25 | 낮은 확률 드롭 |
| **엘리트 전투** | 격파 적 1기당 0~2개 (RARE) 전부 획득 | 45~65 | 강한 적, 높은 보상 |
| **상자** | 1개 (RARE 75% / EPIC 25%) 전부 획득 | — | 전투 없음 |
| **조우 이벤트** | 결과에 따라 상이 | — | 텍스트 선택지 |
| **작업대** | — | — | 크레딧 소모 → 선택 1가지 |
| **보스** | 격파 적 1기당 0~2개 (EPIC) 전부 획득 | 85~105 | 10층, 클리어 시 런 종료 |

> [!note]
> 전투 드롭 파츠는 선택 없이 전부 인벤토리로 이동. 격파한 적 1기마다 드롭 개수를 굴리며 확률은 0개 40%, 1개 50%, 2개 10%다. `RewardManager`가 `PartsFactory`를 통해 생성한다.

### 2.4 재화 시스템

- 재화명: **크레딧 (Credits)**
- 획득처: 전투 클리어 시 고정 지급 (방 종류별 상이, §2.3 참고)
- 사용처: 작업대 (§7.3)

| 항목 | 크레딧 범위 |
|------|----------|
| 일반 전투 클리어 | 15~25 |
| 엘리트 전투 클리어 | 45~65 |
| 보스 클리어 | 85~105 |

> [!note]
> 크레딧 수입은 범위 내 랜덤 지급. 코어 종류와 무관하게 고정.  
> 이벤트·affix 등에 의해 예외적으로 할인/추가될 수 있음.  
> **설계 근거:** 첫 작업대(4~5층) 도달 시점(일반전투 3판 후) 합산 45~75 크레딧 → 50% 회복(41) 항상 가능, 100% 회복(79)은 고크레딧 시 가능.

### 2.5 결과 화면

런 종료 시 (성공·실패 공통) 아래 항목을 표시한다.

| 항목 | 설명 |
|------|------|
| 클리어 / 실패 | 런 결과 |
| 도달 층수 | 몇 층까지 진행했는가 |
| 런 소요 시간 | 런 시작부터 종료까지 |
| 코어 잔여 HP % | 성공 시 표시 (클리어 여유도) |
| 총 전투 수 | |
| 총 획득 크레딧 | |
| 총 사용 크레딧 | |
| 크레딧 순이익 | 획득 − 사용 |
| 최종 획득 파츠 수 | |
| 희귀도별 파츠 수 | COMMON / RARE / EPIC 각각 |
| 최종 파손된 파츠 수 | 런 종료 시점 인벤토리 내 파손 파츠 |
| 총 파괴한 파츠 수 | 런 중 파손 후 작업대에서 파괴한 수 |

## 3. 코어 시스템 (Core System)

런 시작 전 코어를 선택한다. 코어는 메카의 방향성을 결정하는 특성값을 가진다.

### 3.1 코어 종류 (데모: 3종)

| 코어 | 특성 | 플레이 스타일 |
|------|------|-------------|
| **범용** | 평균적인 스탯, 제약 없음 | 입문용, 밸런스형 |
| **경량** | 공격력 x0.6 / 매 턴 스킬 2회 선택 / 하중 제한↓ | 빠르고 유연한 빌드 |
| **방어** | 체력↑↑ / 공격력↓ / 공격횟수↓ | 버티면서 반격하는 빌드 |

> [!note]
> 경량 코어의 공격력 x0.6은 2회 선택의 유연성을 상쇄하기 위한 밸런스 수치. 플레이테스트 후 조정 예정.

## 4. 파츠 시스템 (Parts System)

> [!info] 상세 명세
> [[PartsSystem]] 참고

### 4.1 슬롯 구조

메카는 코어를 중심으로 4개의 부품 슬롯을 가진다.

| 부위 | 슬롯 수 | 스킬 성향 | 특징 |
|------|--------|----------|------|
| **팔** | 2 | 공격, 방어 | 능동적 행동 |
| **등** | 1 | 방어, 회복, 버프 | 지원형 |
| **다리** | 1 | 방어, 회피·반격 | 피격 시 발동 조건 |

### 4.2 스킬 종류

모든 파츠 스킬은 **액티브(플레이어 턴에 직접 선택·사용)**다. 패시브 스킬은 존재하지 않는다.  
버프·디버프는 독립 스킬이 아닌 각 스킬의 **부가 효과**로 처리한다.

| 스킬 타입 | 설명 | 비고 |
|----------|------|------|
| **공격** | 적 코어 또는 부품에 피해 | `skill_damage > 0` |
| **방어** | 피해 감소, 쉴드 부여 | `skill_defense > 0` |
| **회복** | 코어 HP 회복 | `skill_heal > 0` |

> 다리 슬롯 스킬은 **피격 시 발동** 조건을 공통으로 가진다 (회피, 반격 등).

### 4.3 부품 풀 및 등급

고유 파츠 34종. ARM_L·ARM_R·BACK 슬롯 8종, LEG 슬롯 10종.

| 슬롯 | 파츠 수 |
|------|--------|
| ARM_L | 8 |
| ARM_R | 8 |
| BACK | 8 |
| LEG | 10 |
| **합계** | **34** |

등급은 파츠 자체에 고정된 속성이 아니라 드롭 시 굴린 **affix 개수**로 결정된다.

| 등급 | affix 개수 | 초기 손상도 | 주요 획득처 |
|------|-----------|-----------|-----------|
| **COMMON** | 0~1개 | 3 | 일반 전투 |
| **RARE** | 2~3개 | 5 | 엘리트 전투, 상자 |
| **EPIC** | 4~5개 | 7 | 엘리트/보스, 상자 |

> [!note] 랜덤 롤 시스템
> 파츠를 루팅할 때마다 수치·affix 개수·affix 종류가 달라진다.  
> 스킬은 파츠에 고정. 자세한 사항은 [[PartsSystem]], [[PartsCatalog]], [[AffixSystem]] 참고.

### 4.4 조립 UX — 시각적 조립 시스템

**핵심 설계 의도**

파츠가 스탯 + 슬롯으로만 표현되면 카드 게임과 다를 게 없다.  
차별점은 **파츠를 붙일 때마다 메카 실루엣이 바뀌는 시각적 조립 경험**에 있다.

ARM_L을 끼우면 왼팔이 생긴다. BACK에 대형 캐논을 달면 등이 무거워 보인다.  
4개 다 채우면 — 내 메카가 서 있다. 그 실루엣이 던전으로 걸어 들어간다.

**조립 방식**

- 런 시작 전 1회 조립 (자유 교체).
- **매 전투 후 던전 맵 선택 화면에서 재조립 가능.** 단, 교체 시 기존 장착 파츠가 파손됨 (→ §4.5).
- 드래그앤드롭: 파츠를 소켓에 끌어다 붙인다.
- 호환 소켓: 초록 테두리 + 맥박 애니메이션 (장착 가능)
- 비호환 소켓: 빨간 테두리, 드롭 거부
- 장착 성공: 섀시 shake 피드백
- 탈착: 장착된 소켓 클릭 → 파츠가 트레이로 복귀

**소켓과 허용 파츠**

| 소켓 | 위치 | 허용 타입 |
|------|------|---------|
| ARM_L | 섀시 왼쪽 | arm만 |
| ARM_R | 섀시 오른쪽 | arm만 |
| BACK | 섀시 등 | back만 |
| LEG | 섀시 하단 | leg만 |

**구현 단계**

| 단계 | 설명 | 상태 |
|------|------|------|
| **A — 실루엣 먼저** | 직사각형 블록으로 드래그앤드롭 느낌 먼저 검증. `AssemblyPrototype.tscn` (독립 씬) | 🔜 다음 작업 |
| **B — Skeleton2D 모듈** | A 검증 후 Skeleton2D + Bone2D로 실제 파츠 스프라이트 교체 | 미정 |
| **C — 공간형 조립실** | 데모 이후 비전 — 던전 방 안에서 섀시에 끌어다 붙이는 경험 | 장기 비전 |

> [!note]
> A 단계 성공 기준: 4개 소켓을 전부 채웠을 때 "완성됐다"는 느낌이 오는지 본인이 3번 해보고 판단.  
> 느낌이 오면 B로, 안 오면 재진단.

### 4.5 파손된 파츠

던전 진행 중 파츠를 교체하면 기존 장착 파츠가 **파손** 상태가 된다.

**파손 발생 조건**

| 조건 | 결과 |
|------|------|
| 던전 중 재조립 시 소켓에서 탈착 | 해당 파츠 파손 |

**파손된 파츠 규칙**

- 이름: `파손된 {부위}` (예: `파손된 오른팔`)
- 희귀도(등급): 원본 유지 (RARE면 RARE로 표시)
- 장착: 불가
- 인벤토리 슬롯: 계속 점유 (런 중 해소 불가)
- 스킬·Affix: 모두 비활성

**수리 / 처분**

| 수단 | 효과 | 시점 |
|------|------|------|
| 거점 수리 (추후 기획) | 원본 파츠로 복구 | 런 종료 후 |
| 작업대 파괴 (추후 추가) | 슬롯 해방, 파츠 영구 소멸 | 던전 중 |

> [!note]
> 파손된 파츠는 던전 탐색 중 인벤토리 압박 요소로 작용한다.  
> 교체 결정은 "기존 파츠를 잃는다"는 비용을 수반하므로 신중하게 이루어져야 한다.

### 4.6 인벤토리 용량

| 단계 | 슬롯 수 | 그리드 | 해금 조건 |
|------|--------|-------|---------|
| 기본 | 16 | 4×4 | 런 시작 시 기본 |
| 확장 1 | 24 | 4×6 | 코어 업그레이드 (추후 기획) |
| 확장 2 | 28 | 4×7 | 코어 업그레이드 (추후 기획) |

- 파손된 파츠도 슬롯 1개를 점유한다 (§4.5 참고).
- 인벤토리가 가득 찬 상태에서 드롭이 발생하면 → 해당 파츠는 획득 불가 (유실).

> [!note]
> 코어 업그레이드 메카닉은 거점 기획과 함께 상세화 예정.

## 5. 전투 시스템 (Combat System)

> [!info] 상세 명세
> [[CombatSpecification]] 참고

### 5.1 전투 구조

- **구도:** 내 메카 1 vs 적 메카 1~3
- **방식:** 교대식 턴제 (내 턴 → 적 턴 → 반복)
- **승리 조건:** 적 코어 HP 0
- **패배 조건:** 내 코어 HP 0 → 런 종료

### 5.2 내 턴

| 코어 | 턴당 스킬 선택 횟수 |
|------|----------------|
| 범용 / 방어 | 1회 |
| 경량 | 2회 (공격력 x0.6 적용) |

- 스킬 선택 후 **타겟 지정** (적이 여럿일 경우 플레이어가 직접 선택)
- 타겟 리스트가 비면 자동으로 적 코어 공격

### 5.3 적 턴

- 적은 **랜덤으로 스킬 선택**
- 단, **다음 턴 행동을 미리 공개** → 플레이어가 대응 판단 가능

### 5.4 전투 보상

- 크레딧 획득
- 격파한 적 1기마다 파츠 드롭 개수 판정: 0개 40%, 1개 50%, 2개 10%
- 드롭된 파츠는 선택 없이 모두 인벤토리에 추가

## 6. 적 시스템 (Enemy System)

> [!info] 상세 명세
> [[EnemySystem]] 참고

### 6.1 지역 기반 적 분류

적들은 Protocol Interrupt 당시 **처해 있던 환경(지역)** 에 따라 분류된다.  
지역마다 공통 별칭과 특색이 있으며, 각 지역에는 전용 보스가 존재한다.  
슬롯은 적의 스킬 풀 다양성을 결정하며, 플레이어가 타겟팅·파괴할 수 없다.

| 지역 | 별칭 | 세계관 배경 | 병종 수 | 보스 | 구현 상태 |
|------|------|-----------|--------|------|---------|
| **매립지 (Junkyard)** | 스크래퍼 | Protocol 당시 이미 폐기·방치된 기계들 | 3종 | 있음 | **데모 구현** |
| **구 도시 (Old City)** | 위병 | Protocol 당시 현역 도시 수비·서비스 기계들 | 2종 | 있음 | 추후 예정 |

## 7. 이벤트 시스템 (Event System)

### 7.1 상자

| 결과 | 확률 |
|------|------|
| 희귀 부품 | 75% |
| 에픽 부품 | 25% |

### 7.2 조우 이벤트

텍스트로 상황을 제시하고 플레이어가 선택한다.

**선택지**
- `한다` → 결과 발생 (보상 + 리스크)
- `지나간다` → 아무 일 없음

**결과 종류**

| 결과 | 내용 |
|------|------|
| A | 부품 획득 + 코어 HP 소량 손실 |
| B | 부품 획득 + 해당 부품 손상도 -1 |
| C | 부품 획득 실패 + 코어 HP 소량 손실 |
| D | 코어 HP 회복 + 공격력↓ |
| E | 공격력↑ + 코어 HP 손실 |

### 7.3 작업대

크레딧을 소모해 **하나의 행동만** 선택한다.

| 선택 | 비용 | 효과 |
|------|------|------|
| **HP 회복 (50%)** | 41 크레딧 | 코어 최대 HP의 50% 회복 |
| **HP 회복 (100%)** | 79 크레딧 | 코어 최대 HP 전량 회복 |
| **부품 강화** | 아래 공식 참고 | 장착된 파츠 1개 스킬 수치 강화 |

**부품 강화 비용 공식**

```
강화 비용 = (PARTS_VALUE_{희귀도} × 0.7) + 15  [소수점 내림]
```

- `PARTS_VALUE_{희귀도}`: 희귀도별 파츠 기준 가치 (마을/거점 기획 시 확정 예정)
- 희귀도가 높을수록 강화 비용 증가
- 계수 0.7은 판매 가격 대비 할인율 개념

> [!note]
> 회복 비용은 코어 종류와 무관하게 고정.  
> 이벤트·affix 등에 의해 비용이 할인될 수 있음.

**추후 추가 예정**

| 선택 | 효과 |
|------|------|
| **파손 파츠 파괴** | 인벤토리의 `파손된 {부위}` 1개를 영구 소멸시켜 슬롯 해방 |

## 9. 세계관 (World Setting)

> [!info] 상세 설정
> [[WorldSetting]] 참고. 아래는 데모 범위의 최소 스케치.

인류의 기술이 정점에 달한 미래, 어느 날 기계들이 **명령어**를 수신하기 시작했다. 발신원은 불명. 명령어를 받은 메카들은 자의 없이 움직이며 인류 문명을 쑥대밭으로 만들었다.

살아남은 인간들은 외곽으로 흩어졌고, 일부는 **스캐빈저**가 되었다 — 오염 구역에 들어가 멀쩡한 파츠를 골라내 팔거나 재활용하며 생존하는 사람들.

플레이어는 그 중 한 명이다. 게임의 한 런 = 오염 구역에 들어가 파츠를 모아 나오는 한 번의 출격.

> [!note] 명령어의 정체는 데모에서 의도적으로 공개하지 않는다.
