## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 메카 빌더 로그라이크 |
| **엔진** | Godot 4.x (GDScript) |
| **타겟** | 데모 버전 (2026년 내 출시 목표) |
| **런 길이** | 약 15분 |
| **핵심 루프** | 던전 탐색 → 부품 획득 → 메카 조립 → 턴제 전투 |

### 한 줄 소개

고철 부품을 주워 메카를 조립하고, 그 메카로 적과 싸우는 턴제 로그라이크.

---

## 2. 런 구조 (Run Structure)

### 2.1 전체 흐름

```
코어 선택 → 던전 탐색 (10층) → 보스 전투 → 런 종료
```

### 2.2 던전 구성

- 총 **10층** 구조
- 각 층마다 **분기 선택** 존재
- 다음 방 정보 미리 공개: `등급 + 부위 또는 특징`

**층 구성 예시**

| 층 | 내용 |
|----|------|
| 1~3층 | 일반 전투 + 부품 획득 |
| 4~5층 | 분기 선택 |
| 6~8층 | 일반 전투 + 부품 획득 |
| 9층 | 엘리트 전투 |
| 10층 | 보스 전투 |

### 2.3 방 종류

| 방 종류 | 설명 |
|--------|------|
| **일반 전투** | 적 처치 시 일반 등급 부품 보상 (가끔 드롭) |
| **엘리트 전투** | 강한 적, 희귀/에픽 부품 보상 |
| **상자** | 희귀 75% / 에픽 25% |
| **조우 이벤트** | 텍스트 이벤트, 보상 + 리스크 |
| **작업대** | 크레딧 소모 → 회복 or 부품 스킬 강화 |
| **보스** | 10층, 클리어 시 런 종료 |

### 2.4 재화 시스템

- 재화명: **크레딧 (Credits)**
- 획득처: 전투 보상
- 사용처: 작업대 (회복 / 부품 업그레이드)

## 3. 코어 시스템 (Core System)

런 시작 전 코어를 선택한다. 코어는 메카의 방향성을 결정하는 특성값을 가진다.

### 3.1 코어 종류 (데모: 3종)

| 코어 | 특성 | 플레이 스타일 |
|------|------|-------------|
| **범용** | 평균적인 스탯, 제약 없음 | 입문용, 밸런스형 |
| **경량** | 공격력 x0.6 / 매 턴 스킬 2회 선택 / 하중 제한↓ | 빠르고 유연한 빌드 |
| **방어** | 체력↑↑ / 공격력↓ / 공격횟수↓ | 버티면서 반격하는 빌드 |

> [!note]
> 경량 코어의 공격력 x0.6은 2회 선택의 유연성을 상쇄하기 위한 밸런스 수치. 플레이테스트 후 조정 예정.

## 4. 부품 시스템 (Piece System)

> [!info] 상세 명세
> [[PieceSystem]] 참고

### 4.1 슬롯 구조

메카는 코어를 중심으로 4개의 부품 슬롯을 가진다.

| 부위 | 슬롯 수 | 스킬 성향 | 특징 |
|------|--------|----------|------|
| **팔** | 2 | 공격, 방어 | 능동적 행동 |
| **등** | 1 | 방어, 회복, 패시브 | 지원형 |
| **다리** | 1 | 방어, 패시브 | 피격 시 발동 조건 (회피, 반격) |

### 4.2 스킬 종류

| 스킬 | 설명 |
|------|------|
| **공격** | 적 코어 또는 부품에 피해 |
| **방어** | 피해 감소, 쉴드 부여 |
| **회복** | 코어 HP 회복 |
| **패시브** | 조건 충족 시 자동 발동 |

> [!note]
> 버프/디버프는 독립 스킬이 아닌 각 스킬의 **부가 효과**로 처리.
> 다리 스킬은 **피격 시 발동** 조건을 공통으로 가짐 (회피, 반격 등).

### 4.3 부품 등급 및 수량

| 등급 | 부위당 수량 | 총 수량 | 획득처 |
|------|-----------|--------|--------|
| **일반** | 3개 | 12개 | 일반 전투 보상 |
| **희귀** | 2개 | 8개 | 엘리트 전투, 상자 이벤트 |
| **에픽** | 1개 | 4개 | 엘리트/보스 전투, 상자 이벤트 |
| **합계** | | **24개** | |

## 5. 전투 시스템 (Combat System)

> [!info] 상세 명세
> [[CombatSpecification]] 참고

### 5.1 전투 구조

- **구도:** 내 메카 1 vs 적 메카 1~3
- **방식:** 교대식 턴제 (내 턴 → 적 턴 → 반복)
- **승리 조건:** 적 코어 HP 0
- **패배 조건:** 내 코어 HP 0 → 런 종료

### 5.2 내 턴

| 코어 | 턴당 스킬 선택 횟수 |
|------|----------------|
| 범용 / 방어 | 1회 |
| 경량 | 2회 (공격력 x0.6 적용) |

- 스킬 선택 후 **타겟 지정** (적이 여럿일 경우 플레이어가 직접 선택)
- 타겟 리스트가 비면 자동으로 적 코어 공격

### 5.3 적 턴

- 적은 **랜덤으로 스킬 선택**
- 단, **다음 턴 행동을 미리 공개** → 플레이어가 대응 판단 가능

### 5.4 전투 보상

- 크레딧 획득
- 일반 부품 **가끔** 드롭

## 7. 이벤트 시스템 (Event System)

### 7.1 상자

| 결과 | 확률 |
|------|------|
| 희귀 부품 | 75% |
| 에픽 부품 | 25% |

### 7.2 조우 이벤트

텍스트로 상황을 제시하고 플레이어가 선택한다.

**선택지**
- `한다` → 결과 발생 (보상 + 리스크)
- `지나간다` → 아무 일 없음

**결과 종류**

| 결과 | 내용 |
|------|------|
| A | 부품 획득 + 코어 HP 소량 손실 |
| B | 부품 획득 + 해당 부품 능력치↓ |
| C | 부품 획득 실패 + 코어 HP 소량 손실 |
| D | 코어 HP 회복 + 공격력↓ |
| E | 공격력↑ + 코어 HP 손실 |

### 7.3 작업대

크레딧을 소모해 하나의 행동을 선택한다.

| 선택 | 효과 |
|------|------|
| **회복** | 코어 HP 회복 |
| **부품 업그레이드** | 장착된 부품 스킬 수치 강화 |

> [!note]
> 작업대에서는 반드시 하나만 선택 가능. 둘 다 사용 불가.
