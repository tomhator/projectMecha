## 개요

코어 설계는 `영구 연구(메타)`와 `출격 로드아웃(런)`을 분리한다.

- 단일 코어(주인공·상수)의 **평면 노드망**을 거점 재화로 해금한다. **노드 레벨·티어 게이팅 없음.** 진행 곡선 = 재화 예산 + 해금 순서(하데스 거울 결).
- 던전 시작 전 던전 특성 요약을 읽고 이번 런의 **코어 빌드**를 고른다.
- 출격 빌드 = **동사 슬롯 N개(시작 1, 메타로 1→2→3→4)** + **기본공격 1택** + 그 결과로 정해지는 **키스톤**.
- 노드 선택은 코어 외장 외형에도 반영한다(우세 교리 variant). 팔·등·다리 파츠 외형과는 별개다.

> **메타 vs 런:** 코어 본체 스탯(HP·쉴드·AP·하중·슬롯)은 런 중 고정 — 런 내 파워 변동은 **파츠**가 담당. 노드망은 *메타*로만 깊어진다. (코어 스펙 §1.)

## 출격 선택 영역

| 영역 | 선택 | 전투 역할 |
|------|------|----------|
| **동사 슬롯** | 시작 1 (메타 1→2→3→4) | 파츠 조작 동사(교체·수리·투척·덧대·과부하·재가동·복제·임시방편) 중 슬롯 수만큼 |
| **기본공격** | 택 1 | 파츠가 파손돼도 남는 상시 코어 행동. 교리 태그 보유(키스톤 타이브레이커) |
| **상시 기본기** | 고정 | 긴급 장착(파손 빈 슬롯 복구) — 슬롯 무관 항상 사용 |
| **키스톤** | 노드 점수 결과 | 동사·코어 업그레이드·교리 패시브 노드의 교리 점수 합산 → 문턱 넘으면 발동 |

> 옛 "파츠 활용 어빌리티 택1"은 폐기 — 그 동사들(교체·투척·덧대)은 이제 **동사 슬롯**으로 통합. 긴급 장착은 상시 기본기.

### 기본공격 (교리 태그)

3종 유지(콘셉트·수치 유지), 교리 태그만 입힘. 태그는 평소 잠자고 **키스톤 동점 시에만** 발동.

| 기본공격 | 콘셉트 | 교리 |
|----------|--------|------|
| 입자 광선 | 고비용·최고 피해·전 AP 소진 | **태운다** |
| 간이 방벽 | 공격하며 쉴드 확보 | **살린다** |
| 속사 사격 | 저비용·저피해·반복 | **중립** |

3종은 처음부터 열려 있다. 기본공격은 파츠 내구도를 소모하지 않는다. (코어 스펙 §2.2.1.)

## 노드 종류

평면 노드망의 노드는 4종. **모든 노드는 교리 점수**(태운다 +N / 살린다 +N / 중립 0)를 가진다.

| 노드 종류 | 내용 | 출격 시 |
|----------|------|--------|
| **동사** | 동사 1종 + 그 동사의 메타 업그레이드(일부 두 갈래 택1) | 슬롯에 장착 |
| **기본공격** | 입자 광선 / 간이 방벽 / 속사 사격(교리 태그) | 1택 |
| **코어 업그레이드** | 규칙 비틀기 + 스탯(대가 필수). 순수 +스탯 금지 | 장착(슬롯 별도 or 상시 — 미결) |
| **교리 패시브** | 키스톤 형식의 메타 패시브(규칙+트레이드오프) | 장착 |

- **동사 노드:** 기본형 → 메타 업그레이드. 상세는 코어 스펙 §2.3(로스터)·§2.3.1~2(투척·수리·나머지 6종). 비용 = AP + 재료.
- **코어 업그레이드:** 규칙형 / 스탯+대가 / 혼합. 예 "재가동 소멸 무효 / 대신 전투당 1회", "HP+20 / 하중 −10". 코어 스펙 §5.1.
- **교리 패시브:** boon(런 한정)과 별개인 *영구* 교리 강화. 같은 키스톤 형식. (코어 스펙 §5.)

## 평면 노드망 구조

### 해금 규칙

- 모든 노드는 처음 잠겨 있다. 거점 재화로 해금한다.
- **티어 게이팅·노드 레벨 없음.** 노드는 해금/미해금 2상태(단일 강도).
- 진행 곡선 = **재화 예산 + 해금 순서**(인접/선행 노드 규칙은 화면 설계에서). 상위 노드일수록 비용↑.
- 해금은 영구. 환불 없음. 실험성은 출격 때 장착 동사·기본공격·키스톤 조합을 바꿔 확보.
- 잠긴 노드도 효과·교리 점수는 거점 화면에서 공개(빌드 계획 가능 — nV4).

### 출격 규칙

- 해금한 노드만 장착할 수 있다.
- 동사 슬롯 수만큼 동사 노드 장착(시작 1). 기본공격 1택.
- 장착 노드들의 **교리 점수 합산** → 키스톤 결정(아래).
- 효과는 노드 단일 강도로 계산(레벨 개념 없음).

### 키스톤 결정 (코어 스펙 §4 요약)

| 순서 | 규칙 |
|------|------|
| 1 | 장착 노드 교리 점수 **합산** *(기본공격 제외)* |
| 2 | **동점이면** → 기본공격 교리 점수 발동·가산 → 타이 해소 |
| 3 | 선두 교리가 **문턱 ≥** → 그 교리 **강한 키스톤** / 미달 → 키스톤 없음(제너럴리스트) |

- 키스톤 = 빌드 일관성으로 "버는" 것. commit이 곧 보상, 어정쩡하면 유연하나 밋밋(트레이드오프).
- UX: 노드 조정 시 교리 점수 바 실시간 + "현재 키스톤: ___" 표시.

## 코어 외장

코어는 중앙 조종석 + 전면 장갑·견부 프레임·상부 센서·후방 장비팩이 붙는 소형 중장비형 동체다.

- **외형 = 우세 교리 variant (재설계, 미결).** 옛 *티어별 외장 슬롯*은 티어 폐지로 무효. 새 규칙 = 이번 출격의 **키스톤/우세 교리**에 맞춰 코어 외형이 태운다/살린다/중립 결로 변형(기존 `visual_variant` 개념 재활용).
- 외형은 영구 누적이 아니라 이번 출격 빌드 기준. 구체 매핑(어떤 노드/정렬이 어떤 외형 파츠를 바꾸나)은 화면 설계 패스에서 확정.

## 구현 경계

> ⚠️ 아래 코드 구조는 **옛 트리 모델 기준** — 새 모델(평면 노드망·교리 점수·레벨 폐지)로 재작업 필요. 구현은 nV 검증 게이트 후(코어 스펙 §9).

| 역할 | 구현 | 재작업 |
|------|------|--------|
| 노드 데이터 | `Resources/AbilityTreeNode.gd`, `Resources/AbilityTree/*.tres` | 티어/레벨 필드 제거, **교리 점수 필드 추가**, 옛 3축 노드(.tres) 폐기·재작성 |
| 기본공격·동사 | `Resources/SkillData.gd`, `Resources/Skills/skill_core_*.tres` | **교리 태그 필드 추가**, 동사에 기본형/업그레이드 구조 |
| 영구 해금·런 로드아웃 | `Scripts/Autoload/GameState.gd` | 레벨 API 제거, 동사 슬롯(가변)·키스톤 합산 로직 추가 |
| 연구/출격 화면 | `Scenes/CoreSelect/CoreSelectScene.gd` | 평면 노드망 UI + 교리 점수 바 + 키스톤 표시 |
| 전투 노출·조건 | `Scenes/Entities/MechaEntity.gd`, `Scenes/UI/CombatUI.gd` | 동사 콘솔(코어 스펙 §2.1) + 맥락 동사 overlay |

### GameState API (재정의 필요)

```gdscript
# 옛 API (레벨/티어 기반 — 폐기 대상)
GameState.unlock_tree_node(node)        # 해금은 유지, 레벨 개념 제거
GameState.level_tree_node(node)         # 폐기 (레벨 없음)
GameState.set_run_tree_node(node)       # → 동사 슬롯/키스톤 모델로 재정의
GameState.set_run_basic_attack(skill)   # 유지 (교리 태그 반영)
GameState.apply_tree_node(node, level)  # 폐기 (level 인자 무효)

# 신규 (방향 — 구현 패스에서 확정)
# GameState.equip_verb(slot, node)      # 동사 슬롯 장착
# GameState.compute_keystone()          # 교리 점수 합산 → 키스톤
```

거점 해금 진행은 `user://core_research.cfg`에 저장(레벨 필드 제거하고 재사용 가능). 런 장착 선택은 출격 준비 상태로만 유지.

## 미결 (코어 스펙 §8 연동)

- 노드망 인접/해금 규칙, 재화 예산·해금 곡선 수치.
- **코어 외장↔노드/교리 매핑** 구체 규칙.
- 코어 업그레이드·교리 패시브 **개별 노드 로스터**.
- 동사 효과 수치·갈래형(택1) 확정값.
- 키스톤 문턱 점수·교리 점수 스케일.
- 동사 슬롯이 코어 업그레이드/교리 패시브와 별도 슬롯인지, 장착 한도.
