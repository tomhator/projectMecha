# 어빌리티 트리 시스템

> [!info] 관련 문서
> - 전체 게임 구조: [[GameDesignDocument]]
> - 파츠 시스템: [[PartsSystem]]

---

## 개요

코어 설계는 `영구 연구`와 `출격 로드아웃`을 분리한다.

- 거점 크레딧으로 어빌리티 트리 노드를 연구하고 최대 5레벨까지 강화한다.
- 던전 시작 전 던전 특성 요약을 읽고 이번 런의 코어 빌드를 고른다.
- 출격 빌드는 `기본 공격 1개`, `티어별 트리 노드 최대 1개`, `파츠 활용 어빌리티 1개`로 구성한다.
- 트리 노드 선택은 코어 외장 슬롯에도 반영한다. 팔·등·다리 파츠 외형과는 별개다.

---

## 출격 선택 영역

| 영역 | 선택 | 전투 역할 |
|------|------|----------|
| 기본 공격 | 택 1 | 파츠가 파손되어도 남는 상시 코어 행동 |
| 어빌리티 트리 | 각 티어 최대 1개 | 코어 기본 성능과 파츠 운영 빌드 |
| 파츠 활용 어빌리티 | 택 1 | 교체·투척·덧대기 같은 조건부 특수 행동 |

### 기본 공격

| 콘셉트 | 스킬 | 설계 축 |
|------|------|--------|
| 화력 | 입자 광선 | 고비용, 최고 피해, 유틸 거의 없음 |
| 민첩 | 속사 사격 | 저비용, 낮은 피해, 반복 사용 |
| 방어 | 간이 방벽 | 공격하며 소량 쉴드 확보 |

세 기본 공격은 처음부터 열려 있다. 기본 공격은 파츠 내구도를 소모하지 않는다.

### 파츠 활용 어빌리티

| 스킬 | 역할 | 조건·대가 | 해금 |
|------|------|----------|------|
| 긴급 교체 | 인벤토리 파츠를 전투 중 장착 | 기존 장착 파츠는 파손 | 시작 기본 |
| 파손 투척 | 파손 파츠를 공격 자원으로 전환 | 파손 파츠 1개 소모 | 첫 보스 전 |
| 임시 덧대기 | 파손 파츠를 방어 자원으로 전환 | 파손 파츠 1개 소모 | 첫 보스 처치 |

선택한 어빌리티가 조건을 만족하지 못하면 전투 UI는 비활성 상태와 사유를 보여준다.

## 트리 구조

트리는 5티어, 티어별 3개 노드다. 세 선택지는 제조사 고유명 확정 전까지 공격·방어·유틸 역할 축으로 관리한다.

| 티어 | 공격 | 방어 | 유틸 |
|------|------|------|------|
| T1 기반 | 출력 증폭 | 보강 프레임 | 확장 하네스 |
| T2 파츠 효율 | 정상 출력 유지 | 완충 장착부 | 모듈 정렬 |
| T3 행동 운영 | 고출력 루프 | 방호 순환 | 전력 분배 |
| T4 위험 대응 | 비상 점화 | 격실 차단 | 잔해 재배치 |
| T5 완성 | 소모성 화력 교리 | 생존 코어 교리 | 현장 개조 교리 |

### 연구 규칙

- 모든 노드는 처음 잠겨 있다.
- 이전 티어에서 노드 하나를 연구하면 다음 티어 연구가 열린다.
- 연구한 노드는 영구 해금된다.
- 노드는 1레벨로 해금되고 최대 5레벨까지 강화한다.
- 연구 비용은 상위 티어일수록 커지고 레벨업 비용도 단계별로 증가한다.
- 구매·레벨업 환불은 없다. 실험성은 출격 때 티어별 장착 노드를 바꾸는 방식으로 확보한다.
- 잠긴 노드도 역할과 효과 축은 거점 연구 화면에서 공개한다.

### 출격 규칙

- 연구한 노드만 장착할 수 있다.
- 티어마다 최대 1개만 장착한다.
- 연구한 노드가 없는 티어는 비운 채 출격할 수 있다.
- 적용 효과는 장착 노드의 현재 레벨을 기준으로 계산한다.
- 현재 수치 구현은 공격 배율, HP, 쉴드, 행동력, 하중 보정부터 제공한다. 5레벨 보너스는 노드 데이터에 효과 축을 먼저 기록하고 후속 밸런스 작업에서 확정한다.

## 코어 외장

각 티어는 코어 전용 외장 슬롯 하나를 담당한다.

| 티어 | 외장 슬롯 |
|------|----------|
| T1 | 상판 덮개 |
| T2 | 전면 후드 |
| T3 | 측면 장갑 |
| T4 | 후방 팩 |
| T5 | 코어 스파인 |

외형은 영구 연구 누적이 아니라 이번 출격 장착 노드를 기준으로 바뀐다. 노드 레벨은 외형에 섞지 않고 UI 수치로 표시한다.

## 구현 경계

| 역할 | 구현 |
|------|------|
| 트리 노드 데이터 | `Resources/AbilityTreeNode.gd`, `Resources/AbilityTree/*.tres` |
| 기본 공격·파츠 활용 역할 | `Resources/SkillData.gd`, `Resources/Skills/skill_core_*.tres` |
| 영구 연구·런 로드아웃 상태 | `Scripts/Autoload/GameState.gd` |
| 연구/출격 화면 | `Scenes/CoreSelect/CoreSelectScene.gd` |
| 전투 스킬 노출·조건 처리 | `Scenes/Entities/MechaEntity.gd`, `Scenes/UI/CombatUI.gd` |

### GameState API

```gdscript
GameState.unlock_tree_node(node)
GameState.level_tree_node(node)
GameState.set_run_tree_node(node)
GameState.set_run_basic_attack(skill)
GameState.set_run_part_ability(skill)
GameState.apply_tree_node(node, level)
```

거점 연구 진행은 `user://core_research.cfg`에 저장한다. 런 장착 선택은 출격 준비 상태로만 유지한다.
